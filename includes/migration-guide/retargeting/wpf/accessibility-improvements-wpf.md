### <a name="accessibility-improvements-in-wpf"></a>Vylepšení přístupnosti v subsystému WPF

|   |   |
|---|---|
|Podrobnosti|<strong>Vysoký kontrast – vylepšení</strong><br><ul><li>Hlavním cílem <xref:System.Windows.Controls.Expander> je nyní ovládací prvek viditelný. V předchozích verzích rozhraní .NET Framework nebylo.</li><li>Text v <xref:System.Windows.Controls.CheckBox> a <xref:System.Windows.Controls.RadioButton> ovládací prvky, pokud je tato možnost vybrána, je teď snazší zjistit než v předchozích verzích rozhraní .NET Framework.</li><li>Ohraničení zakázané <xref:System.Windows.Controls.ComboBox> je teď na stejnou barvu jako neaktivního textu. V předchozích verzích rozhraní .NET Framework nebylo.</li><li>Tlačítka zakázáno a zaměřují se teď použít barvu motivu správné. V předchozích verzích rozhraní .NET Framework nepodporovaly.</li><li>Tlačítko rozevíracího seznamu se teď zobrazují při <xref:System.Windows.Controls.ComboBox> stylu ovládacího prvku nastavená na <xref:System.Windows.Controls.ToolBar.ComboBoxStyleKey?displayProperty=nameWithType>, v předchozích verzích rozhraní .NET Framework nebyla.</li><li>Na šipku indikátor řazení <xref:System.Windows.Controls.DataGrid> ovládací prvek nyní používá barvy motivu. V předchozích verzích rozhraní .NET Framework tomu tak není.</li><li>Výchozí styl hypertextových odkazů nyní změní barvu správné motiv po pozastavení ukazatele myši. V předchozích verzích rozhraní .NET Framework tomu tak není.</li><li>Fokus klávesnice na přepínací tlačítka se teď zobrazují. V předchozích verzích rozhraní .NET Framework nebylo.</li><li><xref:System.Windows.Controls.DataGrid> Ovládacího prvku zaškrtávací políčko sloupec teď používá očekávané barvy pro zpětnou vazbu fokus klávesnice. V předchozích verzích rozhraní .NET Framework tomu tak není.</li><li>vizuály fokus klávesnice jsou nyní viditelné na <xref:System.Windows.Controls.ComboBox> a <xref:System.Windows.Controls.ListBox> ovládací prvky. V předchozích verzích rozhraní .NET Framework nebylo.</p></li></ul><strong>Zlepšení interakce čtečky obrazovky</strong><br><ul><li><xref:System.Windows.Controls.Expander> ovládací prvky jsou nyní správně oznamují jako skupiny (Rozbalit/sbalit) čtečky obrazovky.</li><li><xref:System.Windows.Controls.DataGridCell> ovládací prvky jsou nyní správně uvolněné jako buňka datové mřížky (lokalizované) pomocí čtečky obrazovky.</li><li>Název upravitelné nyní přečtou čtečky obrazovky <xref:System.Windows.Controls.ComboBox>.</li><li><xref:System.Windows.Controls.PasswordBox> ovládací prvky jsou již uvolněné jako &quot;žádné položky v zobrazení&quot; pomocí čtečky obrazovky.</p></li></ul><strong>Podpora LiveRegion</strong><br>Čtečky obrazovky, jako je například Předčítání pomáhají uživatelům vědět obsah uživatelského rozhraní aplikace, obvykle zadáním popisu vašeho nového něco o rozhraní, které je právě fokus, protože to je pravděpodobně elementu většina zajímavé pro uživatele. Pokud ale prvku uživatelského rozhraní se změní někde na obrazovce a nemá fokus, uživatel nemusí být informováni a přijít o důležité informace. LiveRegions jsou určené k vyřešení tohoto problému. Vývojáři mohou využít k informování čtečka obrazovky nebo jiné [automatizace uživatelského rozhraní](~/docs/framework/ui-automation/ui-automation-overview.md) klienta, který byl proveden důležité změny prvku uživatelského rozhraní. Čtečka obrazovky se můžete rozhodnout používat jak a kdy informovat uživatele o této změně. Vlastnost LiveSetting vám také umožní čtečka obrazovky vědět, jak důležitý je uživatel informován o změny v uživatelském rozhraní.|
|Doporučení|<strong>Jak aktivování nebo zrušení těchto změn</strong><br>Aby aplikace využívat tyto změny, musíte spustit na rozhraní .NET Framework 4.7.1 nebo novější. Aplikace mohou těžit z těchto změn v některém z následujících způsobů:<ul><li>Cílové rozhraní .NET Framework 4.7.1. Toto je doporučený postup. Tyto změny usnadnění přístupu se ve výchozím nastavení se na aplikace WPF, které jsou cíleny rozhraní .NET Framework 4.7.1 povolená nebo novější.</li><li>To výslovný chování starších verzí usnadnění přidáním následujícího kódu [přepínač AppContext](~/docs/framework/configure-apps/file-schema/runtime/appcontextswitchoverrides-element.md) v <code>&lt;runtime&gt;</code> oddílu konfiguračního souboru aplikace a nastavení na <code>false</code>, jak ukazuje následující příklad.</li></ul><pre><code class="lang-xml">&lt;?xml version=&quot;1.0&quot; encoding=&quot;utf-8&quot;?&gt;&#13;&#10;&lt;configuration&gt;&#13;&#10;&lt;startup&gt;&#13;&#10;&lt;supportedRuntime version=&quot;v4.0&quot; sku=&quot;.NETFramework,Version=v4.7&quot;/&gt;&#13;&#10;&lt;/startup&gt;&#13;&#10;&lt;runtime&gt;&#13;&#10;&lt;!-- AppContextSwitchOverrides value attribute is in the form of &#39;key1=true/false;key2=true/false  --&gt;&#13;&#10;&lt;AppContextSwitchOverrides value=&quot;Switch.UseLegacyAccessibilityFeatures=false&quot; /&gt;&#13;&#10;&lt;/runtime&gt;&#13;&#10;&lt;/configuration&gt;&#13;&#10;</code></pre>Aplikací určených pro rozhraní .NET Framework 4.7.1 nebo novější a chcete zachovat starší chování usnadnění můžete přejít k používání funkce starší verze usnadnění explicitním nastavením na tento přepínač AppContext <code>true</code>.<br>Přehled automatizace uživatelského rozhraní, naleznete v tématu [Přehled automatizace uživatelského rozhraní](~/docs/framework/ui-automation/ui-automation-overview.md).|
