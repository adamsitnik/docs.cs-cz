## <a name="backwards-compatibility"></a><span data-ttu-id="e1a75-124">Zpětná kompatibilita</span><span class="sxs-lookup"><span data-stu-id="e1a75-124">Backwards compatibility</span></span>

<span data-ttu-id="e1a75-125">Zpětná kompatibilita znamená, že stávajícímu spotřebiteli rozhraní API může běžet na nové verzi, a přitom se chová stejným způsobem.</span><span class="sxs-lookup"><span data-stu-id="e1a75-125">Backwards compatibility refers to the ability of an existing consumer of an API to run against a new version while behaving in the same way.</span></span> <span data-ttu-id="e1a75-126">Změny chování a změny binární kompatibility mají vliv na zpětnou kompatibilitu.</span><span class="sxs-lookup"><span data-stu-id="e1a75-126">Both behavioral changes and changes in binary compatibility affect backwards compatibility.</span></span> <span data-ttu-id="e1a75-127">Pokud se příjemce nedokáže při spuštění v novější verzi rozhraní API spustit jinak, je rozhraní API *zpětně nekompatibilní*.</span><span class="sxs-lookup"><span data-stu-id="e1a75-127">If a consumer is not able to run or behaves differently when running against the newer version of the API, the API is *backwards incompatible*.</span></span>
