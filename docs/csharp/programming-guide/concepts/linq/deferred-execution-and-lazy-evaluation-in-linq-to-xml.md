## <a name="deferred-execution"></a><span data-ttu-id="51088-105">Odložené provedení</span><span class="sxs-lookup"><span data-stu-id="51088-105">Deferred Execution</span></span>  
 <span data-ttu-id="51088-106">Odložené spuštění znamená, že vyhodnocení výrazu je zpožděno, dokud není skutečně vyžadováno jeho *realizovaná* hodnota.</span><span class="sxs-lookup"><span data-stu-id="51088-106">Deferred execution means that the evaluation of an expression is delayed until its *realized* value is actually required.</span></span> <span data-ttu-id="51088-107">Odložené provádění může významně zlepšit výkon, pokud je třeba manipulovat s velkými datovými kolekcemi, zejména v programech, které obsahují sérii zřetězených dotazů nebo manipulace.</span><span class="sxs-lookup"><span data-stu-id="51088-107">Deferred execution can greatly improve performance when you have to manipulate large data collections, especially in programs that contain a series of chained queries or manipulations.</span></span> <span data-ttu-id="51088-108">V nejlepším případě se odložené spouštění povoluje pouze jedna iterace prostřednictvím zdrojové kolekce.</span><span class="sxs-lookup"><span data-stu-id="51088-108">In the best case, deferred execution enables only a single iteration through the source collection.</span></span>  
  
 <span data-ttu-id="51088-109">Technologie LINQ využívají odložené spouštění jak v rámci členů základních <xref:System.Linq?displayProperty=nameWithType> tříd, tak v metodách rozšíření v různých oborech názvů LINQ, <xref:System.Xml.Linq.Extensions?displayProperty=nameWithType>jako je například.</span><span class="sxs-lookup"><span data-stu-id="51088-109">The LINQ technologies make extensive use of deferred execution in both the members of core <xref:System.Linq?displayProperty=nameWithType> classes and in the extension methods in the various LINQ namespaces, such as <xref:System.Xml.Linq.Extensions?displayProperty=nameWithType>.</span></span>  
  
 <span data-ttu-id="51088-110">Odložené provádění je podporováno přímo v C# jazyce pomocí klíčového slova [yield](../../../language-reference/keywords/yield.md) (ve `yield-return` formě příkazu) při použití v rámci bloku iterátoru.</span><span class="sxs-lookup"><span data-stu-id="51088-110">Deferred execution is supported directly in the C# language by the [yield](../../../language-reference/keywords/yield.md) keyword (in the form of the `yield-return` statement) when used within an iterator block.</span></span> <span data-ttu-id="51088-111">Takový iterátor musí vracet kolekci typu <xref:System.Collections.IEnumerator> nebo <xref:System.Collections.Generic.IEnumerator%601> (nebo odvozeného typu).</span><span class="sxs-lookup"><span data-stu-id="51088-111">Such an iterator must return a collection of type <xref:System.Collections.IEnumerator> or <xref:System.Collections.Generic.IEnumerator%601> (or a derived type).</span></span>  
  
## <a name="eager-vs-lazy-evaluation"></a><span data-ttu-id="51088-112">Eager vs. Opožděné hodnocení</span><span class="sxs-lookup"><span data-stu-id="51088-112">Eager vs. Lazy Evaluation</span></span>  
 <span data-ttu-id="51088-113">Když napíšete metodu, která implementuje odložené vykonání, je také nutné rozhodnout, zda implementovat metodu pomocí opožděného vyhodnocení nebo vyhodnocení Eager.</span><span class="sxs-lookup"><span data-stu-id="51088-113">When you write a method that implements deferred execution, you also have to decide whether to implement the method using lazy evaluation or eager evaluation.</span></span>  
  
- <span data-ttu-id="51088-114">V *opožděném vyhodnocení*je během každého volání iterátoru zpracován jediný element zdrojové kolekce.</span><span class="sxs-lookup"><span data-stu-id="51088-114">In *lazy evaluation*, a single element of the source collection is processed during each call to the iterator.</span></span> <span data-ttu-id="51088-115">Toto je typický způsob, jakým jsou implementovány iterátory.</span><span class="sxs-lookup"><span data-stu-id="51088-115">This is the typical way in which iterators are implemented.</span></span>  
  
- <span data-ttu-id="51088-116">Při *vyhodnocování Eager*bude první volání iterátoru mít za následek zpracování celé kolekce.</span><span class="sxs-lookup"><span data-stu-id="51088-116">In *eager evaluation*, the first call to the iterator will result in the entire collection being processed.</span></span> <span data-ttu-id="51088-117">Může být také nutné zadat dočasnou kopii zdrojové kolekce.</span><span class="sxs-lookup"><span data-stu-id="51088-117">A temporary copy of the source collection might also be required.</span></span> <span data-ttu-id="51088-118">Například <xref:System.Linq.Enumerable.OrderBy%2A> metoda musí seřadit celou kolekci před tím, než vrátí první prvek.</span><span class="sxs-lookup"><span data-stu-id="51088-118">For example, the <xref:System.Linq.Enumerable.OrderBy%2A> method has to sort the entire collection before it returns the first element.</span></span>  
  
 <span data-ttu-id="51088-119">Opožděné hodnocení obvykle poskytuje lepší výkon, protože distribuuje režijní náklady rovnoměrně během hodnocení kolekce a minimalizuje používání dočasných dat.</span><span class="sxs-lookup"><span data-stu-id="51088-119">Lazy evaluation usually yields better performance because it distributes overhead processing evenly throughout the evaluation of the collection and minimizes the use of temporary data.</span></span> <span data-ttu-id="51088-120">Samozřejmě není pro některé operace žádná jiná možnost než vyhodnotit mezilehlé výsledky.</span><span class="sxs-lookup"><span data-stu-id="51088-120">Of course, for some operations, there is no other option than to materialize intermediate results.</span></span>
