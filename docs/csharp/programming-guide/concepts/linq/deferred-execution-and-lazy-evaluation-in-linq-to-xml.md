## <a name="deferred-execution"></a><span data-ttu-id="6fce6-105">Odložené provedení</span><span class="sxs-lookup"><span data-stu-id="6fce6-105">Deferred Execution</span></span>  
 <span data-ttu-id="6fce6-106">Odložené provedení znamená, že vyhodnocení výrazu je odložena až do jeho *realizované* hodnota je skutečně nutná.</span><span class="sxs-lookup"><span data-stu-id="6fce6-106">Deferred execution means that the evaluation of an expression is delayed until its *realized* value is actually required.</span></span> <span data-ttu-id="6fce6-107">Odložené provedení může výrazně zlepšit výkon, když máte k manipulaci s kolekcí velkých objemů dat, zejména v aplikacích, které obsahují řadu zřetězených dotazů nebo manipulace.</span><span class="sxs-lookup"><span data-stu-id="6fce6-107">Deferred execution can greatly improve performance when you have to manipulate large data collections, especially in programs that contain a series of chained queries or manipulations.</span></span> <span data-ttu-id="6fce6-108">Odložené provedení v nejlepším případě umožňuje pouze jednu iterace přes zdrojové kolekce.</span><span class="sxs-lookup"><span data-stu-id="6fce6-108">In the best case, deferred execution enables only a single iteration through the source collection.</span></span>  
  
 <span data-ttu-id="6fce6-109">Technologie LINQ využívat rozsáhlé odložené provedení v členy základní <xref:System.Linq?displayProperty=nameWithType> třídy a metody rozšíření v různých oborech názvů LINQ, jako například <xref:System.Xml.Linq.Extensions?displayProperty=nameWithType>.</span><span class="sxs-lookup"><span data-stu-id="6fce6-109">The LINQ technologies make extensive use of deferred execution in both the members of core <xref:System.Linq?displayProperty=nameWithType> classes and in the extension methods in the various LINQ namespaces, such as <xref:System.Xml.Linq.Extensions?displayProperty=nameWithType>.</span></span>  
  
 <span data-ttu-id="6fce6-110">Odložené provedení je podporován přímo v jazyce C# pomocí [yield](../../../../csharp/language-reference/keywords/yield.md) – klíčové slovo (ve formě `yield-return` příkaz) při použití v rámci blok iterátoru.</span><span class="sxs-lookup"><span data-stu-id="6fce6-110">Deferred execution is supported directly in the C# language by the [yield](../../../../csharp/language-reference/keywords/yield.md) keyword (in the form of the `yield-return` statement) when used within an iterator block.</span></span> <span data-ttu-id="6fce6-111">Takové iterátor musí vracet kolekci typu <xref:System.Collections.IEnumerator> nebo <xref:System.Collections.Generic.IEnumerator%601> (nebo odvozeného typu).</span><span class="sxs-lookup"><span data-stu-id="6fce6-111">Such an iterator must return a collection of type <xref:System.Collections.IEnumerator> or <xref:System.Collections.Generic.IEnumerator%601> (or a derived type).</span></span>  
  
## <a name="eager-vs-lazy-evaluation"></a><span data-ttu-id="6fce6-112">Nemůžou dočkat, až vs. Opožděné vyhodnocení</span><span class="sxs-lookup"><span data-stu-id="6fce6-112">Eager vs. Lazy Evaluation</span></span>  
 <span data-ttu-id="6fce6-113">Když napíšete metodu, která implementuje odložené provedení, máte také rozhodnout, jestli se má implementovat metodu pomocí opožděné vyhodnocení nebo nemůžou dočkat, až hodnocení.</span><span class="sxs-lookup"><span data-stu-id="6fce6-113">When you write a method that implements deferred execution, you also have to decide whether to implement the method using lazy evaluation or eager evaluation.</span></span>  
  
-   <span data-ttu-id="6fce6-114">V *opožděné vyhodnocení*, jeden element zdrojové kolekce je zpracovaných za každé volání iterátoru.</span><span class="sxs-lookup"><span data-stu-id="6fce6-114">In *lazy evaluation*, a single element of the source collection is processed during each call to the iterator.</span></span> <span data-ttu-id="6fce6-115">Toto je typická způsob, ve kterém jsou implementovány iterátory.</span><span class="sxs-lookup"><span data-stu-id="6fce6-115">This is the typical way in which iterators are implemented.</span></span>  
  
-   <span data-ttu-id="6fce6-116">V *nemůžou dočkat, až hodnocení*, první volání iterátoru způsobí celou kolekci právě zpracovává.</span><span class="sxs-lookup"><span data-stu-id="6fce6-116">In *eager evaluation*, the first call to the iterator will result in the entire collection being processed.</span></span> <span data-ttu-id="6fce6-117">Dočasná kopie zdrojové kolekce může být také potřeba.</span><span class="sxs-lookup"><span data-stu-id="6fce6-117">A temporary copy of the source collection might also be required.</span></span> <span data-ttu-id="6fce6-118">Například <xref:System.Linq.Enumerable.OrderBy%2A> metoda se má seřadit celou kolekci dříve, než vrátí první prvek.</span><span class="sxs-lookup"><span data-stu-id="6fce6-118">For example, the <xref:System.Linq.Enumerable.OrderBy%2A> method has to sort the entire collection before it returns the first element.</span></span>  
  
 <span data-ttu-id="6fce6-119">Opožděné vyhodnocení obvykle poskytuje lepší výkon, protože distribuuje nároky na zpracování rovnoměrně během vyhodnocení kolekce a minimalizuje využití dočasná data.</span><span class="sxs-lookup"><span data-stu-id="6fce6-119">Lazy evaluation usually yields better performance because it distributes overhead processing evenly throughout the evaluation of the collection and minimizes the use of temporary data.</span></span> <span data-ttu-id="6fce6-120">Samozřejmě pro některé operace, neexistuje žádná možnost než a materializovat mezilehlých výsledků.</span><span class="sxs-lookup"><span data-stu-id="6fce6-120">Of course, for some operations, there is no other option than to materialize intermediate results.</span></span>
