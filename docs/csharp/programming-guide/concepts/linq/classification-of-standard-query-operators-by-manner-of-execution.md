## <a name="manners-of-execution"></a><span data-ttu-id="a0255-108">Způsoby spouštění</span><span class="sxs-lookup"><span data-stu-id="a0255-108">Manners of Execution</span></span>  
  
### <a name="immediate"></a><span data-ttu-id="a0255-109">Okamžité</span><span class="sxs-lookup"><span data-stu-id="a0255-109">Immediate</span></span>  
 <span data-ttu-id="a0255-110">Okamžité spuštění znamená, že je zdroj dat pro čtení a se operace provádí v místě v kódu, ve kterém je deklarována dotazu.</span><span class="sxs-lookup"><span data-stu-id="a0255-110">Immediate execution means that the data source is read and the operation is performed at the point in the code where the query is declared.</span></span> <span data-ttu-id="a0255-111">Všechny operátory standardního dotazu, které vracejí výsledek jeden, vyčíslitelné spustit okamžitě.</span><span class="sxs-lookup"><span data-stu-id="a0255-111">All the standard query operators that return a single, non-enumerable result execute immediately.</span></span>  
  
### <a name="deferred"></a><span data-ttu-id="a0255-112">Odložené</span><span class="sxs-lookup"><span data-stu-id="a0255-112">Deferred</span></span>  
 <span data-ttu-id="a0255-113">Odložené provedení znamená, že operace se neprovádí v místě v kódu, ve kterém je deklarována dotazu.</span><span class="sxs-lookup"><span data-stu-id="a0255-113">Deferred execution means that the operation is not performed at the point in the code where the query is declared.</span></span> <span data-ttu-id="a0255-114">Operace se provádí jenom v případě, že proměnná dotazu je vypočten, například pomocí `foreach` příkazu.</span><span class="sxs-lookup"><span data-stu-id="a0255-114">The operation is performed only when the query variable is enumerated, for example by using a `foreach` statement.</span></span> <span data-ttu-id="a0255-115">To znamená, že výsledky provádění dotazu závisí na obsahu zdroje dat při spuštění dotazu místo Pokud je dotaz definován.</span><span class="sxs-lookup"><span data-stu-id="a0255-115">This means that the results of executing the query depend on the contents of the data source when the query is executed rather than when the query is defined.</span></span> <span data-ttu-id="a0255-116">Pokud proměnná dotazu je vypočten víckrát, výsledky mohou lišit pokaždé, když.</span><span class="sxs-lookup"><span data-stu-id="a0255-116">If the query variable is enumerated multiple times, the results might differ every time.</span></span> <span data-ttu-id="a0255-117">Téměř všechny operátory standardního dotazu, jehož návratový typ je <xref:System.Collections.Generic.IEnumerable%601> nebo <xref:System.Linq.IOrderedEnumerable%601> spustit odložené způsobem.</span><span class="sxs-lookup"><span data-stu-id="a0255-117">Almost all the standard query operators whose return type is <xref:System.Collections.Generic.IEnumerable%601> or <xref:System.Linq.IOrderedEnumerable%601> execute in a deferred manner.</span></span>  
  
 <span data-ttu-id="a0255-118">Operátory dotazů, které pomocí odloženého provedení můžete kromě jsou klasifikovány jako streamování nebo jiných datových proudů.</span><span class="sxs-lookup"><span data-stu-id="a0255-118">Query operators that use deferred execution can be additionally classified as streaming or non-streaming.</span></span>  
  
#### <a name="streaming"></a><span data-ttu-id="a0255-119">Streamování</span><span class="sxs-lookup"><span data-stu-id="a0255-119">Streaming</span></span>  
 <span data-ttu-id="a0255-120">Streamování operátory nemusíte číst všechna zdrojová data před dávají elementy.</span><span class="sxs-lookup"><span data-stu-id="a0255-120">Streaming operators do not have to read all the source data before they yield elements.</span></span> <span data-ttu-id="a0255-121">V době spuštění streamování operátor provádí její činnosti na každý prvek zdroje, jako je čtení a vrací prvku v případě potřeby.</span><span class="sxs-lookup"><span data-stu-id="a0255-121">At the time of execution, a streaming operator performs its operation on each source element as it is read and yields the element if appropriate.</span></span> <span data-ttu-id="a0255-122">Operátor streamování pokračovat ve čtení zdrojové elementy, dokud elementu výsledku je možné vytvořit.</span><span class="sxs-lookup"><span data-stu-id="a0255-122">A streaming operator continues to read source elements until a result element can be produced.</span></span> <span data-ttu-id="a0255-123">To znamená, že může být více než jeden element source číst k vytvoření jednoho elementu výsledku.</span><span class="sxs-lookup"><span data-stu-id="a0255-123">This means that more than one source element might be read to produce one result element.</span></span>  
  
#### <a name="non-streaming"></a><span data-ttu-id="a0255-124">Streamování</span><span class="sxs-lookup"><span data-stu-id="a0255-124">Non-Streaming</span></span>
