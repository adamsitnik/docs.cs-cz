# <a name="how-to-create-unsigned-friend-assemblies-c"></a><span data-ttu-id="3bb48-102">Postupy: Vytvořit nepodepsaná Friend sestaveníC#()</span><span class="sxs-lookup"><span data-stu-id="3bb48-102">How to: Create Unsigned Friend Assemblies (C#)</span></span>
<span data-ttu-id="3bb48-103">Tento příklad ukazuje, jak použít sestavení typu Friend se sestaveními, která jsou bez znaménka.</span><span class="sxs-lookup"><span data-stu-id="3bb48-103">This example shows how to use friend assemblies with assemblies that are unsigned.</span></span>  
  
### <a name="to-create-an-assembly-and-a-friend-assembly"></a><span data-ttu-id="3bb48-104">Vytvoření sestavení a sestavení typu Friend</span><span class="sxs-lookup"><span data-stu-id="3bb48-104">To create an assembly and a friend assembly</span></span>  
  
1. <span data-ttu-id="3bb48-105">Otevřete příkazový řádek.</span><span class="sxs-lookup"><span data-stu-id="3bb48-105">Open a command prompt.</span></span>  
  
2. <span data-ttu-id="3bb48-106">Vytvořte C# soubor s názvem `friend_unsigned_A.` , který obsahuje následující kód.</span><span class="sxs-lookup"><span data-stu-id="3bb48-106">Create a C# file named `friend_unsigned_A.` that contains the following code.</span></span> <span data-ttu-id="3bb48-107">Kód používá <xref:System.Runtime.CompilerServices.InternalsVisibleToAttribute> atribut k deklaraci friend_unsigned_B jako sestavení typu Friend.</span><span class="sxs-lookup"><span data-stu-id="3bb48-107">The code uses the <xref:System.Runtime.CompilerServices.InternalsVisibleToAttribute> attribute to declare friend_unsigned_B as a friend assembly.</span></span>  
  
    ```csharp  
    // friend_unsigned_A.cs  
    // Compile with:   
    // csc /target:library friend_unsigned_A.cs  
    using System.Runtime.CompilerServices;  
    using System;  
  
    [assembly: InternalsVisibleTo("friend_unsigned_B")]  
  
    // Type is internal by default.  
    class Class1  
    {  
        public void Test()  
        {  
            Console.WriteLine("Class1.Test");  
        }  
    }  
  
    // Public type with internal member.  
    public class Class2  
    {  
        internal void Test()  
        {  
            Console.WriteLine("Class2.Test");  
        }  
    }  
    ```  
  
3. <span data-ttu-id="3bb48-108">Zkompilujte a podepište friend_unsigned_A pomocí následujícího příkazu.</span><span class="sxs-lookup"><span data-stu-id="3bb48-108">Compile and sign friend_unsigned_A by using the following command.</span></span>  
  
    ```csharp  
    csc /target:library friend_unsigned_A.cs  
    ```  
  
4. <span data-ttu-id="3bb48-109">Vytvořte C# soubor s názvem `friend_unsigned_B` , který obsahuje následující kód.</span><span class="sxs-lookup"><span data-stu-id="3bb48-109">Create a C# file named `friend_unsigned_B` that contains the following code.</span></span> <span data-ttu-id="3bb48-110">Vzhledem k tomu, že friend_unsigned_A určuje friend_unsigned_B jako sestavení typu Friend, kód v `internal` friend_unsigned_B může přistupovat k typům a členům z friend_unsigned_A.</span><span class="sxs-lookup"><span data-stu-id="3bb48-110">Because friend_unsigned_A specifies friend_unsigned_B as a friend assembly, the code in friend_unsigned_B can access `internal` types and members from friend_unsigned_A.</span></span>  
  
    ```csharp  
    // friend_unsigned_B.cs  
    // Compile with:   
    // csc /r:friend_unsigned_A.dll /out:friend_unsigned_B.exe friend_unsigned_B.cs  
    public class Program  
    {  
        static void Main()  
        {  
            // Access an internal type.  
            Class1 inst1 = new Class1();  
            inst1.Test();  
  
            Class2 inst2 = new Class2();  
            // Access an internal member of a public type.  
            inst2.Test();  
  
            System.Console.ReadLine();  
        }  
    }  
    ```  
  
5. <span data-ttu-id="3bb48-111">Zkompilujte friend_unsigned_B pomocí následujícího příkazu.</span><span class="sxs-lookup"><span data-stu-id="3bb48-111">Compile friend_unsigned_B by using the following command.</span></span>  
  
    ```csharp  
    csc /r:friend_unsigned_A.dll /out:friend_unsigned_B.exe friend_unsigned_B.cs  
    ```  
  
     <span data-ttu-id="3bb48-112">Název sestavení, který je generován kompilátorem, se musí shodovat s názvem sestavení typu Friend, které je předáno <xref:System.Runtime.CompilerServices.InternalsVisibleToAttribute> atributu.</span><span class="sxs-lookup"><span data-stu-id="3bb48-112">The name of the assembly that is generated by the compiler must match the friend assembly name that is passed to the <xref:System.Runtime.CompilerServices.InternalsVisibleToAttribute> attribute.</span></span> <span data-ttu-id="3bb48-113">Je nutné explicitně zadat název výstupního sestavení (. exe nebo. dll) pomocí `/out` možnosti kompilátoru.</span><span class="sxs-lookup"><span data-stu-id="3bb48-113">You must explicitly specify the name of the output assembly (.exe or .dll) by using the `/out` compiler option.</span></span> <span data-ttu-id="3bb48-114">Další informace naleznete v tématu [/out (C# možnosti kompilátoru)](../../../language-reference/compiler-options/out-compiler-option.md).</span><span class="sxs-lookup"><span data-stu-id="3bb48-114">For more information, see [/out (C# Compiler Options)](../../../language-reference/compiler-options/out-compiler-option.md).</span></span>  
  
6. <span data-ttu-id="3bb48-115">Spusťte soubor friend_unsigned_B. exe.</span><span class="sxs-lookup"><span data-stu-id="3bb48-115">Run the friend_unsigned_B.exe file.</span></span>  
  
     <span data-ttu-id="3bb48-116">Program vytiskne dva řetězce: "Class1. test" a "Class2. test".</span><span class="sxs-lookup"><span data-stu-id="3bb48-116">The program prints two strings: "Class1.Test" and "Class2.Test".</span></span>  
  
## <a name="net-framework-security"></a><span data-ttu-id="3bb48-117">Zabezpečení rozhraní .NET Framework</span><span class="sxs-lookup"><span data-stu-id="3bb48-117">.NET Framework Security</span></span>  
 <span data-ttu-id="3bb48-118">Existují podobnosti mezi <xref:System.Runtime.CompilerServices.InternalsVisibleToAttribute> atributem <xref:System.Security.Permissions.StrongNameIdentityPermission> a třídou.</span><span class="sxs-lookup"><span data-stu-id="3bb48-118">There are similarities between the <xref:System.Runtime.CompilerServices.InternalsVisibleToAttribute> attribute and the <xref:System.Security.Permissions.StrongNameIdentityPermission> class.</span></span> <span data-ttu-id="3bb48-119">Hlavním rozdílem je, <xref:System.Security.Permissions.StrongNameIdentityPermission> že může vyžadovat oprávnění zabezpečení ke spuštění konkrétní části kódu, <xref:System.Runtime.CompilerServices.InternalsVisibleToAttribute> zatímco `internal` atribut řídí viditelnost typů a členů.</span><span class="sxs-lookup"><span data-stu-id="3bb48-119">The main difference is that <xref:System.Security.Permissions.StrongNameIdentityPermission> can demand security permissions to run a particular section of code, whereas the <xref:System.Runtime.CompilerServices.InternalsVisibleToAttribute> attribute controls the visibility of `internal` types and members.</span></span>
