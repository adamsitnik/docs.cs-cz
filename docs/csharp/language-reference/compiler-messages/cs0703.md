<span data-ttu-id="1eefa-103">Nekonzistentní dostupnost: typ omezení 'identifier' je méně dostupný než 'identifier'</span><span class="sxs-lookup"><span data-stu-id="1eefa-103">Inconsistent accessibility: constraint type 'identifier' is less accessible than 'identifier'</span></span>  
  
 <span data-ttu-id="1eefa-104">Obecný parametr míň dostupný než samotná obecná třída nemůže vynutit omezení.</span><span class="sxs-lookup"><span data-stu-id="1eefa-104">A constraint may not force the generic parameter to be less accessible than the generic class itself.</span></span> <span data-ttu-id="1eefa-105">V následujícím příkladu, při obecné třídy C\<T > je deklarována jako veřejné, omezení se pokusí vynutit T pro implementaci interní rozhraní.</span><span class="sxs-lookup"><span data-stu-id="1eefa-105">In the following example, while the generic class C\<T> is declared public, the constraint attempts to force T to implement an internal interface.</span></span> <span data-ttu-id="1eefa-106">I když to bylo povoleno, jen pro klienty s interní přístup by bylo možné vytvořit parametr pro třídu, takže ve skutečnosti třídy může využívat pouze klienti mají interní přístup.</span><span class="sxs-lookup"><span data-stu-id="1eefa-106">Even if this were allowed, only clients with internal access would be able to create the parameter for the class, so in effect, the class could be used only by clients with internal access.</span></span>  
  
 <span data-ttu-id="1eefa-107">K odstranění této chyby, ujistěte se, že úroveň přístupu obecná třída není méně omezující než libovolné třídy nebo rozhraní, které se zobrazují v hranice.</span><span class="sxs-lookup"><span data-stu-id="1eefa-107">To get rid of this error, make sure the access level of the generic class is not less restrictive than any classes or interfaces that appear in the bounds.</span></span>
