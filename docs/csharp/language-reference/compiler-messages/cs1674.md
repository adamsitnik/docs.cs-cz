# <a name="compiler-error-cs1674"></a>Chyba kompilátoru CS1674
'T': typ použitý v pomocí příkazu musí být implicitně převeditelný na System.IDisposable.  
  
 [Příkaz using](../../../csharp/language-reference/keywords/using-statement.md) je určena pro použití k zajištění integrity objekt na konci `using` blokovat se proto mohou používat pouze typy, které jsou na jedno použití takových příkazu. Například nejsou uvolnitelné typy hodnot a parametrů typu, které nejsou omezeny na třídy nemusí být předpokládá se, že na jedno použití.  
  
## <a name="example"></a>Příklad  
 Následující ukázka generuje CS1674.  
  
```csharp  
// CS1674.cs  
class C  
{  
   public static void Main()  
   {  
      int a = 0;  
      a++;  
  
      using (a) {}   // CS1674  
   }  
}  
```  
  
## <a name="example"></a>Příklad  
 Následující ukázka generuje CS1674.  
  
```csharp  
// CS1674_b.cs  
using System;  
class C {  
   public void Test() {  
      using (C c = new C()) {}   // CS1674  
   }  
}  
  
// OK  
class D : IDisposable {  
   void IDisposable.Dispose() {}  
   public void Dispose() {}  
  
   public static void Main() {  
      using (D d = new D()) {}  
   }  
}  
```  
  
## <a name="example"></a>Příklad  
 Následující případ znázorňuje nutnost omezení typu třídy k zajištění, že je neznámý typ parametru na jedno použití. Následující ukázka generuje CS1674.  
  
```csharp  
// CS1674_c.cs  
// compile with: /target:library  
using System;  
public class C<T>  
// Add a class type constraint that specifies a disposable class.  
// Uncomment the following line to resolve.  
// public class C<T> where T : IDisposable  
{  
   public void F(T t)  
   {  
      using (t) {}   // CS1674  
   }  
}  
```
