## <a name="example"></a><span data-ttu-id="e3483-110">Příklad</span><span class="sxs-lookup"><span data-stu-id="e3483-110">Example</span></span>  
 <span data-ttu-id="e3483-111">Následující ukázka generuje CS0686:</span><span class="sxs-lookup"><span data-stu-id="e3483-111">The following sample generates CS0686:</span></span>  
  
```csharp  
// CS0686.cs  
interface I  
{  
    int get_P();  
}  
  
class C : I  
{  
    public int P  
    {  
        get { return 1; }  // CS0686  
    }  
}  
// But the following is valid:  
class D : I  
{  
    int I.get_P() { return 1; }  
    public static void Main() {}  
}  
```  
  
## <a name="example"></a><span data-ttu-id="e3483-112">Příklad</span><span class="sxs-lookup"><span data-stu-id="e3483-112">Example</span></span>  
 <span data-ttu-id="e3483-113">K této chybě může dojít také při deklaraci události.</span><span class="sxs-lookup"><span data-stu-id="e3483-113">This error can also occur when declaring events.</span></span>  <span data-ttu-id="e3483-114">Konstrukce události automaticky generuje `add_event` a `remove_event` metody, které může v konfliktu s metod se stejným názvem v rozhraní, jako v následující ukázce:</span><span class="sxs-lookup"><span data-stu-id="e3483-114">The event construct automatically generates the `add_event` and `remove_event` methods, which could conflict with the methods of the same name in an interface, as in the following sample:</span></span>  
  
```csharp  
// CS0686b.cs  
using System;  
  
interface I  
{  
    void add_OnMyEvent(EventHandler e);  
}  
  
class C : I  
{  
    public event EventHandler OnMyEvent  
    {  
        add { }  // CS0686  
        remove { }  
    }

// Correct (using explicit interface implementation):

{  
    void I.add_OnMyEvent(EventHandler e) {}  
    public static void Main() {}  
}  
```
