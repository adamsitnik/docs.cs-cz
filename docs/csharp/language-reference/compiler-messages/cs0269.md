# <a name="compiler-error-cs0269"></a>Chyba kompilátoru CS0269
Použil se nepřiřazený parametr "parametr" out  
  
 Kompilátor nemůže ověřit, že výstupní parametr byla přidělena hodnota předtím, než byl použit; Při přiřazení, jeho hodnota může nedefinovaný. Je potřeba přiřadit hodnotu `out` parametry v metodě volané před přístupem k hodnotě. Pokud je potřeba použít hodnotu proměnné předaný `ref` parametr místo. Další informace najdete v tématu [předání parametrů](../../../csharp/programming-guide/classes-and-structs/passing-parameters.md).  
  
## <a name="example"></a>Příklad  
 Následující ukázka generuje CS0269:  
  
```csharp  
// CS0269.cs  
class C  
{  
    public static void F(out int i)  
    // One way to resolve the error is to use a ref parameter instead  
    // of an out parameter.  
    //public static void F(ref int i)  
    {  
        // The following line causes a compiler error because no value  
        // has been assigned to i.  
        int k = i;  // CS0269  
        i = 1;  
        // The error does not occur if the order of the two previous   
        // lines is reversed.  
    }  
  
    public static void Main()  
    {  
        int myInt = 1;  
        F(out myInt);  
        // If the declaration of method F is changed to require a ref  
        // parameter, ref must be specified in the call as well.  
        //F(ref myInt);  
    }  
}  
```  
  
## <a name="example"></a>Příklad  
 Tato situace může nastat pokud dojde k inicializaci proměnné v bloku try, který je schopen ověřit kompilátor provede úspěšně:  
  
```csharp  
// CS0269b.cs  
class C  
{  
    public static void F(out int i)  
    {  
        try  
        {  
            // Assignment occurs, but compiler can't verify it  
            i = 1;  
        }  
        catch  
        {  
        }  
  
        int k = i;  // CS0269  
        i = 1;  
    }  
  
    public static void Main()  
    {  
        int myInt;  
        F(out myInt);  
    }  
}  
```
