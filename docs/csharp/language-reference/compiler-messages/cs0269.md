# <a name="compiler-error-cs0269"></a><span data-ttu-id="b81c5-102">Chyba kompilátoru CS0269</span><span class="sxs-lookup"><span data-stu-id="b81c5-102">Compiler Error CS0269</span></span>
<span data-ttu-id="b81c5-103">Použil se nepřiřazený parametr "parametr" out</span><span class="sxs-lookup"><span data-stu-id="b81c5-103">Use of unassigned out parameter 'parameter'</span></span>  
  
 <span data-ttu-id="b81c5-104">Kompilátor nemůže ověřit, že výstupní parametr byla přidělena hodnota předtím, než byl použit; Při přiřazení, jeho hodnota může nedefinovaný.</span><span class="sxs-lookup"><span data-stu-id="b81c5-104">The compiler could not verify that the out parameter was assigned a value before it was used; its value may be undefined when assigned.</span></span> <span data-ttu-id="b81c5-105">Je potřeba přiřadit hodnotu `out` parametry v metodě volané před přístupem k hodnotě.</span><span class="sxs-lookup"><span data-stu-id="b81c5-105">Be sure to assign a value to `out` parameters in the called method before accessing the value.</span></span> <span data-ttu-id="b81c5-106">Pokud je potřeba použít hodnotu proměnné předaný `ref` parametr místo.</span><span class="sxs-lookup"><span data-stu-id="b81c5-106">If you need to use the value of the variable passed in, use a `ref` parameter instead.</span></span> <span data-ttu-id="b81c5-107">Další informace najdete v tématu [předání parametrů](../../../csharp/programming-guide/classes-and-structs/passing-parameters.md).</span><span class="sxs-lookup"><span data-stu-id="b81c5-107">For more information, see [Passing Parameters](../../../csharp/programming-guide/classes-and-structs/passing-parameters.md).</span></span>  
  
## <a name="example"></a><span data-ttu-id="b81c5-108">Příklad</span><span class="sxs-lookup"><span data-stu-id="b81c5-108">Example</span></span>  
 <span data-ttu-id="b81c5-109">Následující ukázka generuje CS0269:</span><span class="sxs-lookup"><span data-stu-id="b81c5-109">The following sample generates CS0269:</span></span>  
  
```csharp  
// CS0269.cs  
class C  
{  
    public static void F(out int i)  
    // One way to resolve the error is to use a ref parameter instead  
    // of an out parameter.  
    //public static void F(ref int i)  
    {  
        // The following line causes a compiler error because no value  
        // has been assigned to i.  
        int k = i;  // CS0269  
        i = 1;  
        // The error does not occur if the order of the two previous   
        // lines is reversed.  
    }  
  
    public static void Main()  
    {  
        int myInt = 1;  
        F(out myInt);  
        // If the declaration of method F is changed to require a ref  
        // parameter, ref must be specified in the call as well.  
        //F(ref myInt);  
    }  
}  
```  
  
## <a name="example"></a><span data-ttu-id="b81c5-110">Příklad</span><span class="sxs-lookup"><span data-stu-id="b81c5-110">Example</span></span>  
 <span data-ttu-id="b81c5-111">Tato situace může nastat pokud dojde k inicializaci proměnné v bloku try, který je schopen ověřit kompilátor provede úspěšně:</span><span class="sxs-lookup"><span data-stu-id="b81c5-111">This could also occur if the initialization of a variable occurs in a try block, which the compiler is unable to verify will execute successfully:</span></span>  
  
```csharp  
// CS0269b.cs  
class C  
{  
    public static void F(out int i)  
    {  
        try  
        {  
            // Assignment occurs, but compiler can't verify it  
            i = 1;  
        }  
        catch  
        {  
        }  
  
        int k = i;  // CS0269  
        i = 1;  
    }  
  
    public static void Main()  
    {  
        int myInt;  
        F(out myInt);  
    }  
}  
```
