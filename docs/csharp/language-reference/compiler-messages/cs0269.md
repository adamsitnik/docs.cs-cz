# <a name="compiler-error-cs0269"></a><span data-ttu-id="39241-102">Chyba kompilátoru CS0269</span><span class="sxs-lookup"><span data-stu-id="39241-102">Compiler Error CS0269</span></span>
<span data-ttu-id="39241-103">Použil se nepřiřazený parametr out s parametrem.</span><span class="sxs-lookup"><span data-stu-id="39241-103">Use of unassigned out parameter 'parameter'</span></span>  
  
 <span data-ttu-id="39241-104">Kompilátor nemůže ověřit, zda byl výstupnímu parametru přiřazena hodnota před použitím; jeho hodnota může být při přiřazení nedefinovaná.</span><span class="sxs-lookup"><span data-stu-id="39241-104">The compiler could not verify that the out parameter was assigned a value before it was used; its value may be undefined when assigned.</span></span> <span data-ttu-id="39241-105">Před přístupem k hodnotě se ujistěte, `out` že před přístupem k hodnotě přiřadíte parametrům v volané metodě hodnotu.</span><span class="sxs-lookup"><span data-stu-id="39241-105">Be sure to assign a value to `out` parameters in the called method before accessing the value.</span></span> <span data-ttu-id="39241-106">Pokud potřebujete použít hodnotu předané proměnné, použijte `ref` místo toho parametr.</span><span class="sxs-lookup"><span data-stu-id="39241-106">If you need to use the value of the variable passed in, use a `ref` parameter instead.</span></span> <span data-ttu-id="39241-107">Další informace najdete v tématu [předávání parametrů](../../programming-guide/classes-and-structs/passing-parameters.md).</span><span class="sxs-lookup"><span data-stu-id="39241-107">For more information, see [Passing Parameters](../../programming-guide/classes-and-structs/passing-parameters.md).</span></span>  
  
## <a name="example"></a><span data-ttu-id="39241-108">Příklad</span><span class="sxs-lookup"><span data-stu-id="39241-108">Example</span></span>  
 <span data-ttu-id="39241-109">Následující ukázka generuje CS0269:</span><span class="sxs-lookup"><span data-stu-id="39241-109">The following sample generates CS0269:</span></span>  
  
```csharp  
// CS0269.cs  
class C  
{  
    public static void F(out int i)  
    // One way to resolve the error is to use a ref parameter instead  
    // of an out parameter.  
    //public static void F(ref int i)  
    {  
        // The following line causes a compiler error because no value  
        // has been assigned to i.  
        int k = i;  // CS0269  
        i = 1;  
        // The error does not occur if the order of the two previous   
        // lines is reversed.  
    }  
  
    public static void Main()  
    {  
        int myInt = 1;  
        F(out myInt);  
        // If the declaration of method F is changed to require a ref  
        // parameter, ref must be specified in the call as well.  
        //F(ref myInt);  
    }  
}  
```  
  
## <a name="example"></a><span data-ttu-id="39241-110">Příklad</span><span class="sxs-lookup"><span data-stu-id="39241-110">Example</span></span>  
 <span data-ttu-id="39241-111">K tomu může dojít také v případě, že k inicializaci proměnné dojde v bloku try, který kompilátor není schopen ověřit, zda se spustí úspěšně:</span><span class="sxs-lookup"><span data-stu-id="39241-111">This could also occur if the initialization of a variable occurs in a try block, which the compiler is unable to verify will execute successfully:</span></span>  
  
```csharp  
// CS0269b.cs  
class C  
{  
    public static void F(out int i)  
    {  
        try  
        {  
            // Assignment occurs, but compiler can't verify it  
            i = 1;  
        }  
        catch  
        {  
        }  
  
        int k = i;  // CS0269  
        i = 1;  
    }  
  
    public static void Main()  
    {  
        int myInt;  
        F(out myInt);  
    }  
}  
```
