## <a name="to-correct-this-error"></a><span data-ttu-id="110c5-106">Oprava této chyby</span><span class="sxs-lookup"><span data-stu-id="110c5-106">To correct this error</span></span>

1. <span data-ttu-id="110c5-107">Pokud je proměnná rozsahu explicitně typu, ujistěte se, že typ je stejný jako nebo implicitně převoditelné z typu prvků v kolekci, kterou opakuje.</span><span class="sxs-lookup"><span data-stu-id="110c5-107">If the range variable is explicitly typed, make sure that the type is either the same as, or implicitly convertible from, the type of the elements in the collection it iterates.</span></span> <span data-ttu-id="110c5-108">Pokud proměnná rozsahu předchází klíčovému slovu `var`, odeberte `var`.</span><span class="sxs-lookup"><span data-stu-id="110c5-108">If the range variable is preceded with the `var` keyword, remove `var`.</span></span>  
  
## <a name="example"></a><span data-ttu-id="110c5-109">Příklad</span><span class="sxs-lookup"><span data-stu-id="110c5-109">Example</span></span>

<span data-ttu-id="110c5-110">Následující kód generuje CS1942:</span><span class="sxs-lookup"><span data-stu-id="110c5-110">The following code generates CS1942:</span></span>  

```csharp
// cs1942.cs  
class Program  
    {  
        static void Main(string[] args)  
        {  
            var x = from var i in Enumerable.Range(1, 100) // CS1949  
                    select i; //CS1942  
        }  
    }  
```

<span data-ttu-id="110c5-111">CS1942 se vztahuje k CS1949, protože použití `var` s proměnnou rozsahu způsobí selhání podkladové operace `Cast<T>`, protože `var` není typu.</span><span class="sxs-lookup"><span data-stu-id="110c5-111">CS1942 is related to CS1949 because the use of `var` with a range variable causes the underlying `Cast<T>` operation to fail because `var` is not a type.</span></span>
