# <a name="compiler-error-cs1032"></a><span data-ttu-id="cb3a5-102">Chyba kompilátoru CS1032</span><span class="sxs-lookup"><span data-stu-id="cb3a5-102">Compiler Error CS1032</span></span>
<span data-ttu-id="cb3a5-103">Po prvním tokenu v souboru nejde definovat symboly preprocesoru a zrušit jejich definice.</span><span class="sxs-lookup"><span data-stu-id="cb3a5-103">Cannot define/undefine preprocessor symbols after first token in file</span></span>  
  
 <span data-ttu-id="cb3a5-104">Direktivy `#undef` [preprocesoru](../language-reference/preprocessor-directives/index.md) a musí být použity na začátku programu, před jakýmkoli jiným klíčovým slovem, jako jsou například ty, které se používají v deklaraci oboru názvů `#define` .</span><span class="sxs-lookup"><span data-stu-id="cb3a5-104">The `#define` and `#undef` [preprocessor directives](../language-reference/preprocessor-directives/index.md) must be used at the beginning of a program, before any other keywords, such as those used in the namespace declaration.</span></span>  
  
 <span data-ttu-id="cb3a5-105">Následující ukázka generuje CS1032:</span><span class="sxs-lookup"><span data-stu-id="cb3a5-105">The following sample generates CS1032:</span></span>  
  
```csharp  
// CS1032.cs  
namespace x  
{  
   public class clx  
   {  
      #define a   // CS1032, put before namespace  
      public static void Main()  
      {  
      }  
   }  
}  
```
