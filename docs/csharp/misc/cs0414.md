 <span data-ttu-id="cb9ad-104">Toto upozornění může dojít v několika situacích, ve kterých můžete ověřit kompilátor nikdy odkazován proměnné:</span><span class="sxs-lookup"><span data-stu-id="cb9ad-104">This warning can occur in several scenarios in which the compiler can verify that a variable is never referenced:</span></span>

- <span data-ttu-id="cb9ad-110">Privátní nebo interní pole je konstantní hodnoty přiřazené a použít jenom v blocích, které jsou vyloučeny pomocí direktivy #ifdef.</span><span class="sxs-lookup"><span data-stu-id="cb9ad-110">A private or internal field is assigned constant values and only used in blocks that are excluded by #ifdef directives.</span></span> <span data-ttu-id="cb9ad-111">Zvažte uvedení pole uvnitř bloku #ifdef.</span><span class="sxs-lookup"><span data-stu-id="cb9ad-111">Consider putting the field inside the #ifdef block.</span></span>  
  
- <span data-ttu-id="cb9ad-112">Privátní nebo interní pole je přiřazené hodnoty konstant na několika místech, ale jinak nejsou dostupné.</span><span class="sxs-lookup"><span data-stu-id="cb9ad-112">A private or internal field is assigned constant values in multiple locations but not otherwise accessed.</span></span> <span data-ttu-id="cb9ad-113">Pokud nepotřebujete pole, zvažte jeho odebrání.</span><span class="sxs-lookup"><span data-stu-id="cb9ad-113">If you do not need the field, consider removing it.</span></span> <span data-ttu-id="cb9ad-114">V opačném případě ho pomocí některé vhodným způsobem.</span><span class="sxs-lookup"><span data-stu-id="cb9ad-114">Otherwise, use it in some appropriate way.</span></span>

 <span data-ttu-id="cb9ad-116">Následující příklad ukazuje jeden ze způsobů, ve kterém se vygeneruje CS0414:</span><span class="sxs-lookup"><span data-stu-id="cb9ad-116">The following sample shows one way in which CS0414 will be generated:</span></span>

   private int i = 1;  // CS0414
