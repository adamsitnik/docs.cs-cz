<span data-ttu-id="43fb8-103">Až se LINQ to Entities dotaz převede na stromy příkazů a spustí se, výsledky dotazu se obvykle vrátí jako jedna z následujících:</span><span class="sxs-lookup"><span data-stu-id="43fb8-103">After a LINQ to Entities query is converted to command trees and executed, the query results are usually returned as one of the following:</span></span>

- <span data-ttu-id="43fb8-105">Typy CLR podporované koncepčním modelem.</span><span class="sxs-lookup"><span data-stu-id="43fb8-105">CLR types supported by the conceptual model.</span></span>  
  
- <span data-ttu-id="43fb8-106">Vložené kolekce.</span><span class="sxs-lookup"><span data-stu-id="43fb8-106">Inline collections.</span></span>  
  
- <span data-ttu-id="43fb8-107">Anonymní typy.</span><span class="sxs-lookup"><span data-stu-id="43fb8-107">Anonymous types.</span></span>  
  
 <span data-ttu-id="43fb8-108">Při spuštění dotazu proti zdroji dat jsou výsledky vyhodnoceny do typů CLR a vráceny klientovi.</span><span class="sxs-lookup"><span data-stu-id="43fb8-108">When the query has executed against the data source, the results are materialized into CLR types and returned to the client.</span></span> <span data-ttu-id="43fb8-109">Všechny materializování objektů provádí Entity Framework.</span><span class="sxs-lookup"><span data-stu-id="43fb8-109">All object materialization is performed by the Entity Framework.</span></span> <span data-ttu-id="43fb8-110">Všechny chyby, které jsou výsledkem neschopnosti mapování mezi Entity Framework a CLR, způsobí, že budou výjimky vyvolány během materializace objektu.</span><span class="sxs-lookup"><span data-stu-id="43fb8-110">Any errors that result from an inability to map between the Entity Framework and the CLR will cause exceptions to be thrown during object materialization.</span></span>
  
 <span data-ttu-id="43fb8-111">Pokud spuštění dotazu vrátí primitivní typy konceptuálního modelu, výsledky se skládají z typů CLR, které jsou samostatné a odpojené od Entity Framework.</span><span class="sxs-lookup"><span data-stu-id="43fb8-111">If the query execution returns primitive conceptual model types, the results consist of CLR types that are stand-alone and disconnected from the Entity Framework.</span></span> <span data-ttu-id="43fb8-112">Nicméně pokud dotaz vrátí kolekci typových objektů entit reprezentovaných <xref:System.Data.Objects.ObjectQuery%601>, tyto typy jsou sledovány kontextem objektu.</span><span class="sxs-lookup"><span data-stu-id="43fb8-112">However, if the query returns a collection of typed entity objects, represented by <xref:System.Data.Objects.ObjectQuery%601>, those types are tracked by the object context.</span></span> <span data-ttu-id="43fb8-113">Chování všech objektů (například podřízené/nadřazené kolekce, sledování změn, polymorfismus atd.) je definováno v Entity Framework.</span><span class="sxs-lookup"><span data-stu-id="43fb8-113">All object behavior (such as child/parent collections, change tracking, polymorphism, and so on) are as defined in the Entity Framework.</span></span> <span data-ttu-id="43fb8-114">Tato funkce se dá použít v její kapacitě, jak je definováno v Entity Framework.</span><span class="sxs-lookup"><span data-stu-id="43fb8-114">This functionality can be used in its capacity, as defined in the Entity Framework.</span></span> <span data-ttu-id="43fb8-115">Další informace naleznete v tématu [práce s objekty](../working-with-objects.md).</span><span class="sxs-lookup"><span data-stu-id="43fb8-115">For more information, see [Working with Objects](../working-with-objects.md).</span></span>
  
 <span data-ttu-id="43fb8-116">Typy struktury vrácené dotazy (například anonymní typy a komplexní typy s možnou hodnotou null) můžou `null` mít hodnotu.</span><span class="sxs-lookup"><span data-stu-id="43fb8-116">Struct types returned from queries (such as anonymous types and nullable complex types) can be of `null` value.</span></span> <span data-ttu-id="43fb8-117">Vlastnost vrácené entity může `null` mít také hodnotu. <xref:System.Data.Objects.DataClasses.EntityCollection%601></span><span class="sxs-lookup"><span data-stu-id="43fb8-117">An <xref:System.Data.Objects.DataClasses.EntityCollection%601> property of a returned entity can also be of `null` value.</span></span> <span data-ttu-id="43fb8-118">To může být způsobeno tím, že prochází vlastnost kolekce entity, která `null` má hodnotu, například volání <xref:System.Linq.Queryable.FirstOrDefault%2A> na objekt <xref:System.Data.Objects.ObjectQuery%601> , který nemá žádné prvky.</span><span class="sxs-lookup"><span data-stu-id="43fb8-118">This can result from projecting the collection property of an entity that is of `null` value, such as calling <xref:System.Linq.Queryable.FirstOrDefault%2A> on an <xref:System.Data.Objects.ObjectQuery%601> that has no elements.</span></span>  
  
 <span data-ttu-id="43fb8-119">V některých situacích se může zobrazit dotaz, který během provádění vygeneruje materializující výsledek, ale dotaz se spustí na serveru a v modulu CLR nebude nikdy materializovat objekt entity.</span><span class="sxs-lookup"><span data-stu-id="43fb8-119">In certain situations, a query might appear to generate a materialized result during its execution, but the query will be executed on the server and the entity object will never be materialized in the CLR.</span></span> <span data-ttu-id="43fb8-120">To může způsobit problémy, pokud závisíte na vedlejších účincích objektu materializace.</span><span class="sxs-lookup"><span data-stu-id="43fb8-120">This can cause problems if you are depending on side effects of object materialization.</span></span>  
  
 <span data-ttu-id="43fb8-121">Následující příklad obsahuje vlastní třídu `MyContact` `LastName` s vlastností.</span><span class="sxs-lookup"><span data-stu-id="43fb8-121">The following example contains a custom class, `MyContact`, with a `LastName` property.</span></span> <span data-ttu-id="43fb8-122">Když je `count` vlastnost nastavena, je zvýšena proměnná. `LastName`</span><span class="sxs-lookup"><span data-stu-id="43fb8-122">When the `LastName` property is set, a `count` variable is incremented.</span></span> <span data-ttu-id="43fb8-123">Při spuštění dvou následujících dotazů se první dotaz zvýší `count` , zatímco druhý dotaz nebude.</span><span class="sxs-lookup"><span data-stu-id="43fb8-123">If you execute the two following queries, the first query will increment `count` while the second query will not.</span></span> <span data-ttu-id="43fb8-124">Důvodem je to, že ve druhém dotazu `LastName` se vlastnost prochází z výsledků `MyContact` a třída se nikdy nevytvoří, protože není nutné spouštět dotaz na Storu.</span><span class="sxs-lookup"><span data-stu-id="43fb8-124">This is because in the second query the `LastName` property is projected from the results and the `MyContact` class is never created, because it is not required to execute the query on the store.</span></span>
