# <a name="navigation-property"></a><span data-ttu-id="52497-102">Navigační vlastnost</span><span class="sxs-lookup"><span data-stu-id="52497-102">navigation property</span></span>
<span data-ttu-id="52497-103">A *navigační vlastnost* vlastnost je volitelná na [typ entity](../../../../docs/framework/data/adonet/entity-type.md) , který umožňuje navigace z jednoho [end](../../../../docs/framework/data/adonet/association-end.md) z [přidružení](../../../../docs/framework/data/adonet/association-type.md) do druhém konci.</span><span class="sxs-lookup"><span data-stu-id="52497-103">A *navigation property* is an optional property on an [entity type](../../../../docs/framework/data/adonet/entity-type.md) that allows for navigation from one [end](../../../../docs/framework/data/adonet/association-end.md) of an [association](../../../../docs/framework/data/adonet/association-type.md) to the other end.</span></span> <span data-ttu-id="52497-104">Na rozdíl od jiných [vlastnosti](../../../../docs/framework/data/adonet/property.md), navigačních vlastností není vázané na data.</span><span class="sxs-lookup"><span data-stu-id="52497-104">Unlike other [properties](../../../../docs/framework/data/adonet/property.md), navigation properties do not carry data.</span></span>  
  
 <span data-ttu-id="52497-105">Definice vlastnosti navigace zahrnuje následující položky:</span><span class="sxs-lookup"><span data-stu-id="52497-105">A navigation property definition includes the following:</span></span>  
  
-   <span data-ttu-id="52497-106">Název.</span><span class="sxs-lookup"><span data-stu-id="52497-106">A name.</span></span> <span data-ttu-id="52497-107">(Povinné)</span><span class="sxs-lookup"><span data-stu-id="52497-107">(Required)</span></span>  
  
-   <span data-ttu-id="52497-108">Přidružení, na kterou se přejde.</span><span class="sxs-lookup"><span data-stu-id="52497-108">The association that it navigates.</span></span> <span data-ttu-id="52497-109">(Povinné)</span><span class="sxs-lookup"><span data-stu-id="52497-109">(Required)</span></span>  
  
-   <span data-ttu-id="52497-110">Elementy end přidružení, na kterou se přejde.</span><span class="sxs-lookup"><span data-stu-id="52497-110">The ends of the association that it navigates.</span></span> <span data-ttu-id="52497-111">(Povinné)</span><span class="sxs-lookup"><span data-stu-id="52497-111">(Required)</span></span>  
  
 <span data-ttu-id="52497-112">Všimněte si, že jsou volitelné na oba typy entit na konci asociace navigační vlastnosti.</span><span class="sxs-lookup"><span data-stu-id="52497-112">Note that navigation properties are optional on both entity types at the ends of an association.</span></span> <span data-ttu-id="52497-113">Pokud definujete navigační vlastnost jednoho typu entity na konci asociace, není nutné definovat vlastnost navigace typu entity na druhém konci přidružení.</span><span class="sxs-lookup"><span data-stu-id="52497-113">If you define a navigation property on one entity type at the end of an association, you do not have to define a navigation property on the entity type at the other end of the association.</span></span>  
  
 <span data-ttu-id="52497-114">Datový typ vlastnosti navigace závisí [násobnost](../../../../docs/framework/data/adonet/association-end-multiplicity.md) jeho vzdáleného [end přidružení](../../../../docs/framework/data/adonet/association-end.md).</span><span class="sxs-lookup"><span data-stu-id="52497-114">The data type of a navigation property is determined by the [multiplicity](../../../../docs/framework/data/adonet/association-end-multiplicity.md) of its remote [association end](../../../../docs/framework/data/adonet/association-end.md).</span></span> <span data-ttu-id="52497-115">Předpokládejme například, vlastnost navigace, `OrdersNavProp`, existuje na `Customer` typu entity a přejde na více přidružení mezi `Customer` a `Order`.</span><span class="sxs-lookup"><span data-stu-id="52497-115">For example, suppose a navigation property, `OrdersNavProp`, exists on a `Customer` entity type and navigates a one-to-many association between `Customer` and `Order`.</span></span> <span data-ttu-id="52497-116">Protože ukončení vzdálené přidružení pro navigační vlastnost má násobnost mnoho (\*), je jeho datový typ kolekce (z `Order`).</span><span class="sxs-lookup"><span data-stu-id="52497-116">Because the remote association end for the navigation property has multiplicity of many (\*), its data type is a collection (of `Order`).</span></span> <span data-ttu-id="52497-117">Podobně, pokud vlastnost navigace, `CustomerNavProp`, existuje na `Order` typ entity, jeho datový typ by `Customer`, protože vzdáleným koncem násobnost je jedna (1).</span><span class="sxs-lookup"><span data-stu-id="52497-117">Similarly, if a navigation property, `CustomerNavProp`, exists on the `Order` entity type, its data type would be `Customer`, because the multiplicity of the remote end is one (1).</span></span>
