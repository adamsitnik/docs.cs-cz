# <a name="entity-data-model-inheritance"></a><span data-ttu-id="b99bf-102">Model EDM (Entity Data Model): dědičnost</span><span class="sxs-lookup"><span data-stu-id="b99bf-102">Entity Data Model: Inheritance</span></span>
<span data-ttu-id="b99bf-103">Model EDM (Entity Data Model) (EDM) podporuje dědičnost pro [typy entit](entity-type.md).</span><span class="sxs-lookup"><span data-stu-id="b99bf-103">The Entity Data Model (EDM) supports inheritance for [entity types](entity-type.md).</span></span> <span data-ttu-id="b99bf-104">Dědičnost v EDM je podobná dědičnosti pro třídy v objektově orientovaném programovacím jazyce.</span><span class="sxs-lookup"><span data-stu-id="b99bf-104">Inheritance in the EDM is similar to inheritance for classes in object-oriented programming languages.</span></span> <span data-ttu-id="b99bf-105">Podobně jako u tříd v objektově orientovaném jazyce, v koncepčním modelu můžete definovat typ entity ( *odvozený typ*), který dědí z jiného typu entity ( *základní typ*).</span><span class="sxs-lookup"><span data-stu-id="b99bf-105">Like with classes in object-oriented languages, in a conceptual model you can define an entity type (a *derived type*) that inherits from another entity type (the *base type*).</span></span> <span data-ttu-id="b99bf-106">Nicméně na rozdíl od tříd v objektově orientovaném programování, v koncepčním modelu odvozený typ vždy zdědí všechny [vlastnosti](property.md) a [navigační vlastnosti](navigation-property.md) základního typu.</span><span class="sxs-lookup"><span data-stu-id="b99bf-106">However, unlike classes in object-oriented programming, in a conceptual model the derived type always inherits all the [properties](property.md) and [navigation properties](navigation-property.md) of the base type.</span></span> <span data-ttu-id="b99bf-107">Zděděné vlastnosti nelze přepsat v odvozeném typu.</span><span class="sxs-lookup"><span data-stu-id="b99bf-107">You cannot override inherited properties in a derived type.</span></span>  
  
 <span data-ttu-id="b99bf-108">V koncepčním modelu můžete vytvořit Hierarchie dědičnosti, ve kterých je odvozený typ děděn z jiného odvozeného typu.</span><span class="sxs-lookup"><span data-stu-id="b99bf-108">In a conceptual model you can build inheritance hierarchies in which a derived type inherits from another derived type.</span></span> <span data-ttu-id="b99bf-109">Typ v horní části hierarchie (jeden typ v hierarchii, který není odvozeným typem) se označuje jako *Kořenový typ*.</span><span class="sxs-lookup"><span data-stu-id="b99bf-109">The type at the top of the hierarchy (the one type in the hierarchy that is not a derived type) is called the *root type*.</span></span> <span data-ttu-id="b99bf-110">V hierarchii dědičnosti musí být [klíč entity](entity-key.md) definován u kořenového typu.</span><span class="sxs-lookup"><span data-stu-id="b99bf-110">In an inheritance hierarchy, the [entity key](entity-key.md) must be defined on the root type.</span></span>  
  
 <span data-ttu-id="b99bf-111">Nelze vytvořit Hierarchie dědičnosti, ve kterých je odvozený typ děděn z více než jednoho typu.</span><span class="sxs-lookup"><span data-stu-id="b99bf-111">You cannot build inheritance hierarchies in which a derived type inherits from more than one type.</span></span> <span data-ttu-id="b99bf-112">Například v koncepčním modelu s typem entity `Book` můžete definovat odvozené typy `FictionBook` a `NonFictionBook`, které jednotlivé dědí z `Book`.</span><span class="sxs-lookup"><span data-stu-id="b99bf-112">For example, in a conceptual model with a `Book` entity type, you could define derived types `FictionBook` and `NonFictionBook` that each inherit from `Book`.</span></span> <span data-ttu-id="b99bf-113">Nicméně nelze definovat typ, který dědí z typu `FictionBook` i `NonFictionBook`.</span><span class="sxs-lookup"><span data-stu-id="b99bf-113">However, you could not then define a type that inherits from both the `FictionBook` and `NonFictionBook` types.</span></span>
