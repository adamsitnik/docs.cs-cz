## <a name="connection-string-syntax"></a><span data-ttu-id="a5335-108">Syntaxe připojovacího řetězce</span><span class="sxs-lookup"><span data-stu-id="a5335-108">Connection string syntax</span></span>

<span data-ttu-id="a5335-109">Připojovací řetězec se středníkem oddělený seznam dvojic klíč/hodnota parametru:</span><span class="sxs-lookup"><span data-stu-id="a5335-109">A connection string is a semicolon-delimited list of key/value parameter pairs:</span></span>
  
    keyword1=value; keyword2=value;
  
<span data-ttu-id="a5335-110">Klíčová slova nerozlišují malá a velká písmena.</span><span class="sxs-lookup"><span data-stu-id="a5335-110">Keywords are not case-sensitive.</span></span> <span data-ttu-id="a5335-111">Hodnoty, ale mohou být velká a malá písmena, v závislosti na zdroji dat.</span><span class="sxs-lookup"><span data-stu-id="a5335-111">Values, however, may be case-sensitive, depending on the data source.</span></span> <span data-ttu-id="a5335-112">Klíčová slova a hodnoty mohou obsahovat [prázdné znaky](https://en.wikipedia.org/wiki/Whitespace_character#Unicode).</span><span class="sxs-lookup"><span data-stu-id="a5335-112">Both keywords and values may contain [whitespace characters](https://en.wikipedia.org/wiki/Whitespace_character#Unicode).</span></span> <span data-ttu-id="a5335-113">Počáteční a koncové mezery bude ignorován v klíčových slov a nekotované hodnoty.</span><span class="sxs-lookup"><span data-stu-id="a5335-113">Leading and trailing white space is ignored in keywords and unquoted values.</span></span>

<span data-ttu-id="a5335-114">Obsahuje-li hodnota středník, [řídící znaky Unicode](https://en.wikipedia.org/wiki/Unicode_control_characters), nebo úvodní a koncové prázdné znaky, musí být uzavřen v jednoduchých nebo dvojitých uvozovek.</span><span class="sxs-lookup"><span data-stu-id="a5335-114">If a value contains the semicolon, [Unicode control characters](https://en.wikipedia.org/wiki/Unicode_control_characters), or leading or trailing white space, it must be enclosed in single or double quotation marks.</span></span> <span data-ttu-id="a5335-115">Příklad:</span><span class="sxs-lookup"><span data-stu-id="a5335-115">For example:</span></span>

    Keyword=" whitespace  ";
    Keyword='special;character';

<span data-ttu-id="a5335-116">Nadřazené znak nedojde v rámci hodnoty, který ji obklopuje.</span><span class="sxs-lookup"><span data-stu-id="a5335-116">The enclosing character may not occur within the value it encloses.</span></span> <span data-ttu-id="a5335-117">Proto můžou být uzavřená hodnotu obsahující jednoduchých uvozovek pouze do dvojitých uvozovek a naopak:</span><span class="sxs-lookup"><span data-stu-id="a5335-117">Therefore, a value containing single quotation marks can be enclosed only in double quotation marks, and vice versa:</span></span>

    Keyword='double"quotation;mark';
    Keyword="single'quotation;mark";

<span data-ttu-id="a5335-118">Uvozovky sami, stejně jako znaménko rovná se nevyžadují, aby uvozovací znaky, takže následující připojovací řetězce jsou platné:</span><span class="sxs-lookup"><span data-stu-id="a5335-118">The quotation marks themselves, as well as the equals sign, do not require escaping, so the following connection strings are valid:</span></span>

    Keyword=no "escaping" 'required';
    Keyword=a=b=c

<span data-ttu-id="a5335-119">Protože každá hodnota je pro čtení do další středníkem nebo konec řetězce, je hodnota v druhém příkladu `a=b=c`, a poslední středník je volitelné.</span><span class="sxs-lookup"><span data-stu-id="a5335-119">Since each value is read till the next semicolon or the end of string, the value in the latter example is `a=b=c`, and the final semicolon is optional.</span></span>

<span data-ttu-id="a5335-120">Všechny řetězce připojení sdílet stejnou základní syntaxi popsané výše.</span><span class="sxs-lookup"><span data-stu-id="a5335-120">All connection strings share the same basic syntax described above.</span></span> <span data-ttu-id="a5335-121">Sada rozpoznaný klíčová slova závisí na poskytovateli, ale a průběžně vyvíjel let ze starší rozhraní API, jako *ODBC*.</span><span class="sxs-lookup"><span data-stu-id="a5335-121">The set of recognized keywords depends on the provider, however, and has evolved over the years from earlier APIs such as *ODBC*.</span></span> <span data-ttu-id="a5335-122">*Rozhraní .NET Framework* zprostředkovatel dat pro *systému SQL Server* (`SqlClient`) podporuje mnoho klíčových slov z starší rozhraní API, ale je obecně flexibilnější a přijímá synonyma pro spoustu běžných připojovací řetězec klíčová slova.</span><span class="sxs-lookup"><span data-stu-id="a5335-122">The *.NET Framework* data provider for *SQL Server* (`SqlClient`) supports many keywords from older APIs, but is generally more flexible and accepts synonyms for many of the common connection string keywords.</span></span>

<span data-ttu-id="a5335-123">Překlepů mohou způsobit chyby.</span><span class="sxs-lookup"><span data-stu-id="a5335-123">Typing mistakes can cause errors.</span></span> <span data-ttu-id="a5335-124">Například `Integrated Security=true` je platný, ale `IntegratedSecurity=true` způsobí chybu.</span><span class="sxs-lookup"><span data-stu-id="a5335-124">For example, `Integrated Security=true` is valid, but `IntegratedSecurity=true` causes an error.</span></span>

<span data-ttu-id="a5335-125">Připojovací řetězce s ručně vytvořeným za běhu z neověřený vstup uživatele je ohrožen útoky prostřednictvím injektáže řetězec a by mohly ohrozit zabezpečení ve zdroji dat.</span><span class="sxs-lookup"><span data-stu-id="a5335-125">Connection strings constructed manually at run time from unvalidated user input are vulnerable to string-injection attacks and jeopardize security at the data source.</span></span> <span data-ttu-id="a5335-126">K řešení těchto problémů *ADO.NET* 2.0 zavedl [tvůrci připojovacích řetězců](../../../../docs/framework/data/adonet/connection-string-builders.md) pro každou *rozhraní .NET Framework* poskytovatele dat služeb.</span><span class="sxs-lookup"><span data-stu-id="a5335-126">To address these problems, *ADO.NET* 2.0 introduced [connection string builders](../../../../docs/framework/data/adonet/connection-string-builders.md) for each *.NET Framework* data provider.</span></span> <span data-ttu-id="a5335-127">Tyto tvůrci připojovacích řetězců vystavit parametry jako vlastnosti silného typu a bude možné ověřit připojovací řetězec před odesláním do zdroje dat.</span><span class="sxs-lookup"><span data-stu-id="a5335-127">These connection string builders expose parameters as strongly-typed properties, and make it possible to validate the connection string before it's sent to the data source.</span></span>
