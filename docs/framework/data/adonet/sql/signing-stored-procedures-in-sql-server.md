 <span data-ttu-id="06f4b-106">Uložená procedura s certifikátem nebo asymetrickým klíčem se můžete přihlásit.</span><span class="sxs-lookup"><span data-stu-id="06f4b-106">You can sign a stored procedure with a certificate or an asymmetric key.</span></span> <span data-ttu-id="06f4b-107">To je určen pro scénáře, když nelze dědit oprávnění prostřednictvím řetězení vlastnictví nebo když řetězce vlastnictví bylo přerušeno, jako je například dynamické SQL.</span><span class="sxs-lookup"><span data-stu-id="06f4b-107">This is designed for scenarios when permissions cannot be inherited through ownership chaining or when the ownership chain is broken, such as dynamic SQL.</span></span> <span data-ttu-id="06f4b-108">Pak vytvoříte uživatele mapovat na certifikát, udělení uživatelských oprávnění na objekty, které uloženou proceduru potřebuje přístup k certifikátu.</span><span class="sxs-lookup"><span data-stu-id="06f4b-108">You can then create a user mapped to the certificate, granting the certificate user permissions on the objects the stored procedure needs to access.</span></span>  

 <span data-ttu-id="06f4b-109">Je můžete také vytvořit přihlašovací jméno mapované na stejný certifikát a pak udělit všechna nezbytná oprávnění na úrovni serveru na tomto přihlášení nebo přidat přihlášení na jeden nebo více rolí serveru.</span><span class="sxs-lookup"><span data-stu-id="06f4b-109">You can also create a login mapped to the same certificate, and then grant any necessary server-level permissions to that login, or add the login to one or more of the fixed server roles.</span></span> <span data-ttu-id="06f4b-110">To je navržená tak, aby povolení `TRUSTWORTHY` databáze nastavení pro scénáře, ve kterých jsou potřeba oprávnění vyšší úrovně.</span><span class="sxs-lookup"><span data-stu-id="06f4b-110">This is designed to avoid enabling the `TRUSTWORTHY` database setting for scenarios in which higher level permissions are needed.</span></span>

## <a name="creating-certificates"></a><span data-ttu-id="06f4b-114">Vytváření certifikátů</span><span class="sxs-lookup"><span data-stu-id="06f4b-114">Creating Certificates</span></span>  
 <span data-ttu-id="06f4b-115">Při přihlašování uloženou proceduru s certifikátem nebo asymentrickým klíčem, algoritmus digest dat skládající se z šifrované hodnoty hash uloženou proceduru kódu, společně s execute – jako uživatel, se vytvoří pomocí soukromého klíče.</span><span class="sxs-lookup"><span data-stu-id="06f4b-115">When you sign a stored procedure with a certificate or asymmetric key, a data digest consisting of the encrypted hash of the stored procedure code, along with the execute-as user, is created using the private key.</span></span> <span data-ttu-id="06f4b-116">V době běhu je ověřování algoritmem digest data dešifrovat pomocí veřejného klíče a ve srovnání s hodnotou hash uloženou proceduru.</span><span class="sxs-lookup"><span data-stu-id="06f4b-116">At run time, the data digest is decrypted with the public key and compared with the hash value of the stored procedure.</span></span> <span data-ttu-id="06f4b-117">Změna execute – jako uživatel zruší platnost hodnoty hash tak, aby už neodpovídá digitální podpis.</span><span class="sxs-lookup"><span data-stu-id="06f4b-117">Changing the execute-as user invalidates the hash value so that the digital signature no longer matches.</span></span> <span data-ttu-id="06f4b-118">Úprava uloženou proceduru zahodí podpis zcela, což zabraňuje někoho, kdo nemá přístup k privátnímu klíči možnost měnit kód uloženou proceduru.</span><span class="sxs-lookup"><span data-stu-id="06f4b-118">Modifying the stored procedure drops the signature entirely, which prevents someone who does not have access to the private key from changing the stored procedure code.</span></span> <span data-ttu-id="06f4b-119">V obou případech musíte znovu podepsat postup při každé změně kódu nebo execute – jako uživatel.</span><span class="sxs-lookup"><span data-stu-id="06f4b-119">In either case, you must re-sign the procedure each time you change the code or the execute-as user.</span></span>  
  
 <span data-ttu-id="06f4b-120">Při podepisování modulu jsou dva kroky:</span><span class="sxs-lookup"><span data-stu-id="06f4b-120">There are two required steps involved in signing a module:</span></span>

1.  <span data-ttu-id="06f4b-124">Podepsat certifikát s použitím příkazů jazyka Transact-SQL postupu `ADD SIGNATURE TO [procedureName] BY CERTIFICATE [certificateName]` příkazu.</span><span class="sxs-lookup"><span data-stu-id="06f4b-124">Sign the procedure with the certificate using the Transact-SQL `ADD SIGNATURE TO [procedureName] BY CERTIFICATE [certificateName]` statement.</span></span>  

<span data-ttu-id="06f4b-125">Jakmile modul byl podepsán, jeden nebo více objektů zabezpečení je potřeba vytvořit, aby bylo možné obsahovat další oprávnění, které by měly být přidruženy s certifikátem.</span><span class="sxs-lookup"><span data-stu-id="06f4b-125">Once the module has been signed, one or more principals needs to be created in order to hold the additional permissions that should be associated with the certificate.</span></span>

<span data-ttu-id="06f4b-130">Pokud je nutné modul další oprávnění na úrovni serveru:</span><span class="sxs-lookup"><span data-stu-id="06f4b-130">If the module needs additional server-level permissions:</span></span>  
  
1.  <span data-ttu-id="06f4b-131">Zkopírujte certifikát, který `master` databáze.</span><span class="sxs-lookup"><span data-stu-id="06f4b-131">Copy the certificate to the `master` database.</span></span>

1.  <span data-ttu-id="06f4b-133">Udělte požadovaná oprávnění na úrovni serveru přihlášení certifikátu.</span><span class="sxs-lookup"><span data-stu-id="06f4b-133">Grant the certificate login the required server-level permissions.</span></span>  
  
> [!NOTE]  
>  <span data-ttu-id="06f4b-134">Certifikát nelze udělit oprávnění uživateli, který má určitá oprávnění odvolat ODEPŘÍT příkaz using.</span><span class="sxs-lookup"><span data-stu-id="06f4b-134">A certificate cannot grant permissions to a user that has had permissions revoked using the DENY statement.</span></span> <span data-ttu-id="06f4b-135">ODEPŘÍT vždy přednost udělení, brání volajícímu dědí oprávnění udělená uživatelského certifikátu.</span><span class="sxs-lookup"><span data-stu-id="06f4b-135">DENY always takes precedence over GRANT, preventing the caller from inheriting permissions granted to the certificate user.</span></span>
