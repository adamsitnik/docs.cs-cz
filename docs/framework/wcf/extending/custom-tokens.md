# <a name="custom-tokens"></a>Vlastní tokeny
I když Windows Communication Foundation (WCF) nativně podporuje certifikáty X. 509, tokeny kontextu zabezpečení, lístky protokolu Kerberos a tokeny uživatelského jména jako přihlašovací údaje, je dostatečně flexibilní, aby bylo možné používat vlastní přihlašovací údaje.  
  
## <a name="in-this-section"></a>V tomto oddílu  
 [Postupy: Vytvoření vlastního tokenu](how-to-create-a-custom-token.md)  
 Popisuje, jak vytvořit vlastní token zabezpečení pomocí <xref:System.IdentityModel.Tokens.SecurityToken> třídy a jak ho integrovat s vlastním poskytovatelem a ověřovatelem tokenů zabezpečení.
