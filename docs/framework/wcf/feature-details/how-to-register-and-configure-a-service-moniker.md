# <a name="how-to-register-and-configure-a-service-moniker"></a><span data-ttu-id="5316d-102">Postupy: Registrace a konfigurace monikeru služby</span><span class="sxs-lookup"><span data-stu-id="5316d-102">How to: Register and Configure a Service Moniker</span></span>
<span data-ttu-id="5316d-103">Před použitím monikeru služby Windows Communication Foundation (WCF) v rámci aplikace modelu COM s typem kontraktu, musí zaregistrovat požadované typy s atributy s modelem COM a konfigurace aplikace modelu COM a zástupný název požadované vazby konfigurace.</span><span class="sxs-lookup"><span data-stu-id="5316d-103">Before using the Windows Communication Foundation (WCF) service moniker within a COM application with a typed contract, you must register the required attributed types with COM, and configure the COM application and the moniker with the required binding configuration.</span></span>  
  
### <a name="to-register-the-required-attributed-types-with-com"></a><span data-ttu-id="5316d-104">Zaregistrovat s atributy požadované typy modelu COM</span><span class="sxs-lookup"><span data-stu-id="5316d-104">To register the required attributed types with COM</span></span>  
  
1. <span data-ttu-id="5316d-105">Použití [ServiceModel Metadata Utility Tool (Svcutil.exe)](../../../../docs/framework/wcf/servicemodel-metadata-utility-tool-svcutil-exe.md) nástroj k načtení metadat kontraktu služby WCF.</span><span class="sxs-lookup"><span data-stu-id="5316d-105">Use the [ServiceModel Metadata Utility Tool (Svcutil.exe)](../../../../docs/framework/wcf/servicemodel-metadata-utility-tool-svcutil-exe.md) tool to retrieve the metadata contract from the WCF service.</span></span> <span data-ttu-id="5316d-106">Tím se vytvoří zdrojový kód pro sestavení klienta WCF a konfigurační soubor aplikace klienta.</span><span class="sxs-lookup"><span data-stu-id="5316d-106">This generates the source code for a WCF client assembly and a client application configuration file.</span></span>  
  
2. <span data-ttu-id="5316d-107">Ujistěte se, že typy v sestavení jsou označeny jako `ComVisible`.</span><span class="sxs-lookup"><span data-stu-id="5316d-107">Ensure that the types in the assembly are marked as `ComVisible`.</span></span> <span data-ttu-id="5316d-108">Uděláte to tak, přidejte následující atribut souboru AssemblyInfo.cs projektu sady Visual Studio.</span><span class="sxs-lookup"><span data-stu-id="5316d-108">To do so, add the following attribute to the AssemblyInfo.cs file in your Visual Studio project.</span></span>  
  
    ```

    ```  
  
3. <span data-ttu-id="5316d-109">Kompilaci spravovaného klienta WCF jako sestavení se silným názvem.</span><span class="sxs-lookup"><span data-stu-id="5316d-109">Compile the managed WCF client as a strong-named assembly.</span></span> <span data-ttu-id="5316d-110">Tento postup vyžaduje přihlášení pomocí páru kryptografických klíčů.</span><span class="sxs-lookup"><span data-stu-id="5316d-110">This requires signing with a cryptographic key pair.</span></span> <span data-ttu-id="5316d-111">Další informace najdete v tématu [podepisování sestavení silným názvem](https://go.microsoft.com/fwlink/?LinkId=94874) v příručce pro vývojáře .NET.</span><span class="sxs-lookup"><span data-stu-id="5316d-111">For more information, see [Signing an Assembly with a Strong Name](https://go.microsoft.com/fwlink/?LinkId=94874) in the .NET Developer's Guide.</span></span>  
  
4. <span data-ttu-id="5316d-112">Nástroj Assembly Registration (Regasm.exe) se `/tlb` možnost zaregistrovat typy v sestavení s modelu COM.</span><span class="sxs-lookup"><span data-stu-id="5316d-112">Use the Assembly Registration (Regasm.exe) tool with the `/tlb` option to register the types in the assembly with COM.</span></span>  
  
5. <span data-ttu-id="5316d-113">Použijte nástroj Global Assembly Cache (Gacutil.exe) Chcete-li přidat sestavení do globální mezipaměti sestavení.</span><span class="sxs-lookup"><span data-stu-id="5316d-113">Use the Global Assembly Cache (Gacutil.exe) tool to add the assembly to the global assembly cache.</span></span>  
  
    > [!NOTE]
    >  <span data-ttu-id="5316d-114">Podepisování sestavení a jeho přidání do globální mezipaměti sestavení jsou volitelné kroky, ale jejich může zjednodušit proces načítání sestavení ze správné místo v době běhu.</span><span class="sxs-lookup"><span data-stu-id="5316d-114">Signing the assembly and adding it to the Global Assembly Cache are optional steps, but they can simplify the process of loading the assembly from the correct location at runtime.</span></span>  
  
### <a name="to-configure-the-com-application-and-the-moniker-with-the-required-binding-configuration"></a><span data-ttu-id="5316d-115">Konfigurace aplikace COM a zástupný název s konfigurací požadované vazby</span><span class="sxs-lookup"><span data-stu-id="5316d-115">To configure the COM application and the moniker with the required binding configuration</span></span>  
  
- <span data-ttu-id="5316d-116">Umístit definic vazeb (generovaných [ServiceModel Metadata Utility Tool (Svcutil.exe)](../../../../docs/framework/wcf/servicemodel-metadata-utility-tool-svcutil-exe.md) v konfiguračním souboru aplikace generovaného klienta) v konfiguračním souboru aplikace klienta.</span><span class="sxs-lookup"><span data-stu-id="5316d-116">Place the binding definitions (generated by the [ServiceModel Metadata Utility Tool (Svcutil.exe)](../../../../docs/framework/wcf/servicemodel-metadata-utility-tool-svcutil-exe.md) in the generated client application configuration file) in the client application's configuration file.</span></span> <span data-ttu-id="5316d-117">Například pro Visual Basic 6.0 spustitelný soubor s názvem CallCenterClient.exe, by měl umístit konfiguraci do souboru s názvem CallCenterConfig.exe.config v rámci stejného adresáře jako spustitelný soubor.</span><span class="sxs-lookup"><span data-stu-id="5316d-117">For example, for a Visual Basic 6.0 executable named CallCenterClient.exe, the configuration should be placed in a file named CallCenterConfig.exe.config within the same directory as the executable.</span></span> <span data-ttu-id="5316d-118">Klientská aplikace teď můžete použít zástupný název.</span><span class="sxs-lookup"><span data-stu-id="5316d-118">The client application can now use the moniker.</span></span> <span data-ttu-id="5316d-119">Všimněte si, že konfigurace vazby není povinný, pokud pomocí jedné z standardních vazby typy poskytované službou WCF.</span><span class="sxs-lookup"><span data-stu-id="5316d-119">Note that the binding configuration is not required if using one of the standard binding types provided by WCF.</span></span>  
  
     <span data-ttu-id="5316d-120">Následující typ je registrován.</span><span class="sxs-lookup"><span data-stu-id="5316d-120">The following type is registered.</span></span>  
  
    ```  
    using System.ServiceModel;  
  
    ...  
  
    [ServiceContract]   
    public interface IMathService   
    {  
    [OperationContract]  
    public int Add(int x, int y);  
    [OperationContract]  
    public int Subtract(int x, int y);  
    }  
    ```  
  
     <span data-ttu-id="5316d-121">Aplikace je přístupné přes `wsHttpBinding` vazby.</span><span class="sxs-lookup"><span data-stu-id="5316d-121">The application is exposed using a `wsHttpBinding` binding.</span></span> <span data-ttu-id="5316d-122">Pro daný typ a konfigurace aplikace se používají následující řetězce monikeru příklad.</span><span class="sxs-lookup"><span data-stu-id="5316d-122">For the given type and application configuration, the following example moniker strings are used.</span></span>  
  
    ```  
    service4:address=http://localhost/MathService, binding=wsHttpBinding, bindingConfiguration=Binding1  
    ```  
  
     `or`  
  
    ```  
    service4:address=http://localhost/MathService, binding=wsHttpBinding, bindingConfiguration=Binding1, contract={36ADAD5A-A944-4d5c-9B7C-967E4F00A090}  
    ```  
  
     <span data-ttu-id="5316d-123">Můžete použít kteroukoli z těchto řetězců moniker z aplikace Visual Basic 6.0, po přidání odkazu na sestavení, které obsahuje `IMathService` typů, jak je znázorněno v následujícím ukázkovém kódu.</span><span class="sxs-lookup"><span data-stu-id="5316d-123">You can use either of these moniker strings from within a Visual Basic 6.0 application, after adding a reference to the assembly that contains the `IMathService` types, as shown in the following sample code.</span></span>  
  
    ```  
    Dim MathProxy As IMathService  
    Dim result As Integer  
  
    Set MathProxy = GetObject( _  
            "service4:address=http://localhost/MathService, _  
            binding=wsHttpBinding, _  
            bindingConfiguration=Binding1")  
  
    result = MathProxy.Add(3, 5)  
    ```  
  
     <span data-ttu-id="5316d-124">V tomto příkladu definici pro konfiguraci vazby `Binding1` je uložen v vhodně pojmenované konfigurační soubor pro klientskou aplikaci, jako je například vb6appname.exe.config.</span><span class="sxs-lookup"><span data-stu-id="5316d-124">In this example, the definition for the binding configuration `Binding1` is stored in a suitably named configuration file for the client application, such as vb6appname.exe.config.</span></span>  
  
    > [!NOTE]
    >  <span data-ttu-id="5316d-125">Podobný kód v C#, C++ nebo jakékoli jiné aplikace jazyk .NET můžete použít.</span><span class="sxs-lookup"><span data-stu-id="5316d-125">You can use similar code in a C#, a C++, or any other .NET Language application.</span></span>  
  
    > [!NOTE]
    >  <span data-ttu-id="5316d-126">: Pokud je moniker je poškozený nebo pokud služba není dostupná, volání `GetObject` vrátí chybu "Neplatná syntaxe".</span><span class="sxs-lookup"><span data-stu-id="5316d-126">: If the moniker is malformed or if the service is unavailable, the call to `GetObject` returns an error of "Invalid Syntax".</span></span> <span data-ttu-id="5316d-127">Pokud se zobrazí tato chyba, ujistěte se, že zástupný název, který používáte, je správná a služba není k dispozici.</span><span class="sxs-lookup"><span data-stu-id="5316d-127">If you receive this error, make sure the moniker you are using is correct and the service is available.</span></span>  
  
     <span data-ttu-id="5316d-128">I když toto téma se zaměřuje na použití monikeru služby z kódu jazyka Visual Basic 6.0, je použití monikeru služby z jiných jazyků.</span><span class="sxs-lookup"><span data-stu-id="5316d-128">Although this topic focuses on using the service moniker from VB 6.0 code, you can use a service moniker from other languages.</span></span> <span data-ttu-id="5316d-129">Při použití monikeru z C++ kódu Svcutil.exe generovaného sestavení by měly být naimportovány s "no_namespace named_guids – raw_interfaces_only", jak je znázorněno v následujícím kódu.</span><span class="sxs-lookup"><span data-stu-id="5316d-129">When using a moniker from C++ code the Svcutil.exe generated assembly should be imported with "no_namespace named_guids raw_interfaces_only" as shown in the following code.</span></span>  
  
    ```  
    #import "ComTestProxy.tlb" no_namespace named_guids  
    ```  
  
     <span data-ttu-id="5316d-130">To definice importované rozhraní upraví tak, aby všechny metody vrátit `HResult`.</span><span class="sxs-lookup"><span data-stu-id="5316d-130">This modifies the imported interface definitions so that all methods return an `HResult`.</span></span> <span data-ttu-id="5316d-131">Všechny ostatní vrácené hodnoty se převedou na výstupní parametry.</span><span class="sxs-lookup"><span data-stu-id="5316d-131">Any other return values are converted into out parameters.</span></span> <span data-ttu-id="5316d-132">Celkové provádění metody zůstává stejná.</span><span class="sxs-lookup"><span data-stu-id="5316d-132">The overall execution of the methods remains the same.</span></span> <span data-ttu-id="5316d-133">To umožňuje určit, proč výjimku při volání metody na proxy serveru.</span><span class="sxs-lookup"><span data-stu-id="5316d-133">This allows you to determine the cause of an exception when calling a method on the proxy.</span></span> <span data-ttu-id="5316d-134">Tato funkce je dostupná pouze z kódu jazyka C++.</span><span class="sxs-lookup"><span data-stu-id="5316d-134">This functionality is only available from C++ code.</span></span>
