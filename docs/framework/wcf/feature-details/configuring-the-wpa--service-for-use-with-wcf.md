## <a name="configuring-a-site-with-non-http-bindings"></a><span data-ttu-id="461b8-109">Konfigurace sítě s vazbami jiným protokolem než HTTP</span><span class="sxs-lookup"><span data-stu-id="461b8-109">Configuring a Site with Non-HTTP bindings</span></span>  
 <span data-ttu-id="461b8-110">Pro použití vazby jiným protokolem než HTTP s WAS, musí přidat vazbu webu do konfigurace WAS.</span><span class="sxs-lookup"><span data-stu-id="461b8-110">To use a non-HTTP binding with WAS, the site binding must be added to the WAS configuration.</span></span> <span data-ttu-id="461b8-111">Konfigurace úložiště pro WAS je soubor applicationHost.config umístěný v adresáři %windir%\system32\inetsrv\config.</span><span class="sxs-lookup"><span data-stu-id="461b8-111">The configuration store for WAS is the applicationHost.config file, located in the %windir%\system32\inetsrv\config directory.</span></span> <span data-ttu-id="461b8-112">Tato konfigurace úložiště je sdílen WAS a IIS 7.0.</span><span class="sxs-lookup"><span data-stu-id="461b8-112">This configuration store is shared by both WAS and IIS 7.0.</span></span>  
  
 <span data-ttu-id="461b8-113">soubor applicationHost.config je textový soubor XML, který lze otevřít v libovolném standardního textového editoru (například Poznámkový blok).</span><span class="sxs-lookup"><span data-stu-id="461b8-113">applicationHost.config is an XML text file that can be opened with any standard text editor (such as Notepad).</span></span> <span data-ttu-id="461b8-114">Ale [!INCLUDE[iisver](../../../../includes/iisver-md.md)] preferovaný způsob, jak přidat vazby webu jiným protokolem než HTTP je nástroj příkazového řádku konfigurace (appcmd.exe).</span><span class="sxs-lookup"><span data-stu-id="461b8-114">However, the [!INCLUDE[iisver](../../../../includes/iisver-md.md)] command-line configuration tool (appcmd.exe) is the preferred way to add non-HTTP site bindings.</span></span>  
  
 <span data-ttu-id="461b8-115">Následující příkaz přidá vazbu webu net.tcp výchozí webový server pomocí appcmd.exe (Tento příkaz je zadat jako jeden řádek).</span><span class="sxs-lookup"><span data-stu-id="461b8-115">The following command adds a net.tcp site binding to the default Web site using appcmd.exe (this command is entered as a single line).</span></span>

```console  
appcmd.exe set site "Default Web Site" -+bindings.[protocol='net.tcp',bindingInformation='808:*']  
```  
  
 <span data-ttu-id="461b8-116">Tento příkaz přidá novou vazbu net.tcp výchozí webový server tak, že přidáte řádku níže uvedeného souboru applicationHost.config.</span><span class="sxs-lookup"><span data-stu-id="461b8-116">This command adds the new net.tcp binding to the default Web site by adding the line indicated below to the applicationHost.config file.</span></span>  
  
```xml

    <site name="Default Web Site" id="1">  
        <bindings>  
            <binding protocol="HTTP" bindingInformation="*:80:" />  
            //The following line is added by the command.  
            <binding protocol="net.tcp" bindingInformation="808:*" />  
        </bindings>  
    </site>  
</sites>  
```  
  
## <a name="enabling-an-application-to-use-non-http-protocols"></a><span data-ttu-id="461b8-117">Povolení aplikace pro použití jiných protokolů než HTTP</span><span class="sxs-lookup"><span data-stu-id="461b8-117">Enabling an Application to Use Non-HTTP Protocols</span></span>  
 <span data-ttu-id="461b8-118">Můžete povolit nebo zakázat jednotlivým síťovým protocolsat na úrovni aplikace.</span><span class="sxs-lookup"><span data-stu-id="461b8-118">You can enable or disable individual network protocolsat the application level.</span></span> <span data-ttu-id="461b8-119">Následující příkaz ukazuje, jak povolit protokoly HTTP i net.tcp pro aplikace, která běží v `Default Web Site`.</span><span class="sxs-lookup"><span data-stu-id="461b8-119">The following command illustrates how to enable both the HTTP and net.tcp protocols for an application that runs in the `Default Web Site`.</span></span>

  
 <span data-ttu-id="461b8-120">Seznam Povolené protokoly je možné také nastavit v \<applicationDefaults > element konfigurace XML pro lokalitu uložené v souboru ApplicationHost.config.</span><span class="sxs-lookup"><span data-stu-id="461b8-120">The list of enabled protocols can also be set in the \<applicationDefaults> element of the site’s XML configuration stored in ApplicationHost.config.</span></span>  
  
 <span data-ttu-id="461b8-121">Následující kód XML ze souboru applicationHost.config znázorňuje lokality vázána na HTTP a jiných protokolů než HTTP.</span><span class="sxs-lookup"><span data-stu-id="461b8-121">The following XML code from applicationHost.config illustrates a site bound to both HTTP and non-HTTP protocols.</span></span> <span data-ttu-id="461b8-122">Další konfigurace vyžadovaná pro podporu jiných protokolů než HTTP je vyznačeny pomocí komentářů.</span><span class="sxs-lookup"><span data-stu-id="461b8-122">The additional configuration required to support non-HTTP protocols is called out with comments.</span></span>  
  
```xml

    <application path="/">  
        <virtualDirectory path="/" physicalPath="D:\inetpub\wwwroot" />  
    </application>  
       <bindings>  
            //The following two lines are added by the command.  
            <binding protocol="HTTP" bindingInformation="*:80:" />  
            <binding protocol="net.tcp" bindingInformation="808:*" />  
       </bindings>  
    </site>  
    <siteDefaults>  
        <logFile   
        customLogPluginClsid="{FF160663-DE82-11CF-BC0A-00AA006111E0}"  
          directory="D:\inetpub\logs\LogFiles" />

    </siteDefaults>  
    <applicationDefaults   
      applicationPool="DefaultAppPool"   
      //The following line is inserted by the command.  
      enabledProtocols="http, net.tcp" />  
    <virtualDirectoryDefaults allowSubDirConfig="true" />  
</sites>  
```
