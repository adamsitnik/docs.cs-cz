 <span data-ttu-id="c7b40-106">*Reflexe útoky* jsou riziko zprávy zpět do odesílatele, jako kdyby pocházejí přijímač jako odpověď.</span><span class="sxs-lookup"><span data-stu-id="c7b40-106">*Reflection attacks* are replays of messages back to a sender as if they came from the receiver as the reply.</span></span> <span data-ttu-id="c7b40-107">Standardní *přehrání* ve Windows Communication Foundation (WCF) mechanismus nezpracovává automaticky to.</span><span class="sxs-lookup"><span data-stu-id="c7b40-107">The standard *replay detection* in the Windows Communication Foundation (WCF) mechanism does not automatically handle this.</span></span>  
  
 <span data-ttu-id="c7b40-108">Reflexe útoky jsou ve výchozím nastavení zmírnit, protože model služby WCF přidá podepsanou zprávu ID zpráv požadavků a očekává, že podepsané `relates-to` záhlaví zprávy odpovědi.</span><span class="sxs-lookup"><span data-stu-id="c7b40-108">Reflection attacks are mitigated by default because the WCF service model adds a signed message ID to request messages and expects a signed `relates-to` header on response messages.</span></span> <span data-ttu-id="c7b40-109">V důsledku toho nelze přehrály zprávy s požadavkem jako odpověď.</span><span class="sxs-lookup"><span data-stu-id="c7b40-109">Consequently, the request message cannot be replayed as a response.</span></span> <span data-ttu-id="c7b40-110">Ve scénářích zabezpečené spolehlivé zprávy (SV) jsou útoky reflexe zmírnit, protože:</span><span class="sxs-lookup"><span data-stu-id="c7b40-110">In secure reliable message (RM) scenarios, reflection attacks are mitigated because:</span></span>  
  
- <span data-ttu-id="c7b40-111">Vytvoření pořadí a schémata zpráv odpověď vytvoření sekvence se liší.</span><span class="sxs-lookup"><span data-stu-id="c7b40-111">The create sequence and create sequence response message schemas are different.</span></span>  
  
- <span data-ttu-id="c7b40-112">Vzhledem k tomu, že klient nemůže rozumět takové zprávy, nelze pro nezpracovávají pořadí přehrály zprávy sekvence, které klient odešle zpět do.</span><span class="sxs-lookup"><span data-stu-id="c7b40-112">For simplex sequences, sequence messages the client sends cannot be replayed back to it because the client cannot understand such messages.</span></span>  
  
- <span data-ttu-id="c7b40-113">Pro duplexní pořadí dvě pořadí identifikátory musí být jedinečný.</span><span class="sxs-lookup"><span data-stu-id="c7b40-113">For duplex sequences, the two sequence IDs must be unique.</span></span> <span data-ttu-id="c7b40-114">Odchozí zpráva sekvence nemůže proto znovu přehrát zpět jako příchozí zprávy sekvence (všechny hlavičky pořadí a úřadů, které jsou podepsané, příliš).</span><span class="sxs-lookup"><span data-stu-id="c7b40-114">Thus, an outgoing sequence message cannot be replayed back as an incoming sequence message (all sequence headers and bodies are signed, too).</span></span>  
  
 <span data-ttu-id="c7b40-115">Pouze vazby, které jsou náchylné k útokům reflexe jsou bez WS-Addressing: vlastní vazby, které mají WS-Addressing zakázána a použít symetrický klíč zabezpečení.</span><span class="sxs-lookup"><span data-stu-id="c7b40-115">The only bindings that are susceptible to reflection attacks are those without WS-Addressing: custom bindings that have WS-Addressing disabled and use the symmetric key-based security.</span></span> <span data-ttu-id="c7b40-116"><xref:System.ServiceModel.BasicHttpBinding> Nepoužívá WS-Addressing ve výchozím nastavení, ale nepoužívá symetrický klíč zabezpečení tak, aby díky tomu bude zranitelný vůči útoku.</span><span class="sxs-lookup"><span data-stu-id="c7b40-116">The <xref:System.ServiceModel.BasicHttpBinding> does not use WS-Addressing by default, but it does not use symmetric key-based security in a way that allows it to be vulnerable to this attack.</span></span>  
  
 <span data-ttu-id="c7b40-117">Zmírnění dopadů pro vlastní vazby není vytvoření kontextu zabezpečení nebo tak, aby vyžadovala záhlaví WS-Addressing.</span><span class="sxs-lookup"><span data-stu-id="c7b40-117">The mitigation for custom bindings is to not establish security context or to require WS-Addressing headers.</span></span>
