<span data-ttu-id="54060-103">Můžete použít Windows Communication Foundation (WCF) pro vývoj nových projektů, když budete pokračovat k údržbě stávajících aplikací vyvinutých pomocí technologie ASP.NET.</span><span class="sxs-lookup"><span data-stu-id="54060-103">You can choose to use Windows Communication Foundation (WCF) for new development, while continuing to maintain existing applications developed using ASP.NET.</span></span> <span data-ttu-id="54060-104">Protože WCF má být nejvhodnější volbou pro usnadnění komunikace s aplikací vytvořených pomocí rozhraní .NET Framework ve všech scénářích, může sloužit jako standardní nástroj pro řešení nejrůznější komunikační problémy se softwarem způsobem, že technologie ASP.NET nelze provést.</span><span class="sxs-lookup"><span data-stu-id="54060-104">Because WCF is intended to be the most suitable choice for facilitating communication with applications built with the .NET Framework in any scenario, it can serve as a standard tool for solving a wide variety of software communications problems in a way that ASP.NET cannot.</span></span>

<span data-ttu-id="54060-105">Nové aplikace WCF je nasadit na stejných počítačů jako existující webové služby ASP.NET.</span><span class="sxs-lookup"><span data-stu-id="54060-105">New WCF applications can be deployed on the same machines as existing ASP.NET Web services.</span></span> <span data-ttu-id="54060-106">Pokud existujících webových služeb ASP.NET s používat verzi rozhraní .NET Framework starší než verze 2.0, můžete používat ASP.NET IIS Registration Tool selektivně nasazení do aplikace služby IIS, ve kterých jsou nové aplikace WCF zajistit také jejich hostování rozhraní .NET Framework 2.0.</span><span class="sxs-lookup"><span data-stu-id="54060-106">If the existing ASP.NET Web services use a version of the .NET Framework prior to version 2.0, then you can use the ASP.NET IIS Registration Tool to selectively deploy the .NET Framework 2.0 to IIS applications in which new WCF applications are to be hosted.</span></span> <span data-ttu-id="54060-107">Tento nástroj je popsána v [registrační nástroj služby IIS technologie ASP.NET (Aspnet_regiis.exe)](https://go.microsoft.com/fwlink/?LinkId=94687), a je součástí uživatelského rozhraní konzoly pro správu služby IIS 6.0.</span><span class="sxs-lookup"><span data-stu-id="54060-107">That tool is documented at [ASP.NET IIS Registration Tool (Aspnet_regiis.exe)](https://go.microsoft.com/fwlink/?LinkId=94687), and has a user interface built into the IIS 6.0 management console.</span></span>

<span data-ttu-id="54060-108">WCF slouží k přidání nových funkcí do existující webové služby ASP.NET tak, že přidáte služby WCF, které jsou nakonfigurovány na spouštění v režimu kompatibility ASP.NET do stávajících aplikací technologie ASP.NET webové služby ve službě IIS.</span><span class="sxs-lookup"><span data-stu-id="54060-108">WCF can be used to add new features to existing ASP.NET Web services by adding WCF services configured to run in ASP.NET compatibility mode to existing ASP.NET Web service applications in IIS.</span></span> <span data-ttu-id="54060-109">Z důvodu režim kompatibility ASP.NET, můžete používat a aktualizovat stejné informace o stavu aplikace jako již existující kód technologie ASP.NET, pomocí kódu pro nové služby WCF <xref:System.Web.HttpContext> třídy.</span><span class="sxs-lookup"><span data-stu-id="54060-109">Because of ASP.NET compatibility mode, the code for the new WCF services can access and update the same application state information as the pre-existing ASP.NET code, by using the <xref:System.Web.HttpContext> class.</span></span> <span data-ttu-id="54060-110">Aplikace můžou také sdílet stejné knihovny tříd.</span><span class="sxs-lookup"><span data-stu-id="54060-110">The applications can also share the same class libraries.</span></span>

<span data-ttu-id="54060-111">Klienti WCF pomocí webové služby ASP.NET.</span><span class="sxs-lookup"><span data-stu-id="54060-111">WCF clients can use ASP.NET Web services.</span></span> <span data-ttu-id="54060-112">Služby WCF, které mají nakonfigurované <xref:System.ServiceModel.BasicHttpBinding> mohou využívat klienty webové služby ASP.NET.</span><span class="sxs-lookup"><span data-stu-id="54060-112">WCF services that are configured with the <xref:System.ServiceModel.BasicHttpBinding> can be used by ASP.NET Web service clients.</span></span> <span data-ttu-id="54060-113">Webové služby ASP.NET může existovat vedle sebe s aplikací služby WCF a WCF slouží i k přidání funkcí do existující webové služby ASP.NET.</span><span class="sxs-lookup"><span data-stu-id="54060-113">ASP.NET Web services can co-exist with WCF applications, and WCF can even be used to add features to existing ASP.NET Web services.</span></span> <span data-ttu-id="54060-114">S ohledem všechny z těchto způsobů, jimiž lze společně použít služby WCF a ASP.NET Web, můžete chtít migrace webových služeb ASP.NET na WCF pouze v případě, že vyžadujete funkce, které jsou k dispozici službami WCF a není v prostředí ASP.NET.</span><span class="sxs-lookup"><span data-stu-id="54060-114">Given all of these ways in which WCF and ASP.NET Web services can be used together, you may want to migrate ASP.NET Web services to WCF only if you require features that are provided by WCF and not ASP.NET Web services.</span></span>

<span data-ttu-id="54060-115">I v několika případech, kdy je nezbytné je migrace kódu z jedné technologie do jiného zřídka správný přístup.</span><span class="sxs-lookup"><span data-stu-id="54060-115">Even in the few cases where it is necessary, migrating code from one technology to another is seldom the correct approach.</span></span> <span data-ttu-id="54060-116">Důvod pro přijetí nové technologie je splnit nové požadavky, které nelze splnit dřívějších technologií a v takovém případě je správnou věc udělat pro návrh nové řešení, které vyhoví nově rozšířit sadu požadavků.</span><span class="sxs-lookup"><span data-stu-id="54060-116">The reason for adopting the new technology is to meet new requirements that cannot be met with the earlier technology, and in that case, the correct thing to do is to design a new solution to meet the newly-expanded set of requirements.</span></span> <span data-ttu-id="54060-117">Nových návrhu výhodách z vašich zkušeností s ve stávajícím systému a při získané od takového systému byla navržena.</span><span class="sxs-lookup"><span data-stu-id="54060-117">The new design benefits from your experience with the existing system and from wisdom gained since that system was designed.</span></span> <span data-ttu-id="54060-118">Nový design můžete také použít všechny funkce nové technologie a ne reprodukce původního návrhu na nové platformě.</span><span class="sxs-lookup"><span data-stu-id="54060-118">The new design can also use the full capabilities of the new technologies rather than reproducing the old design on the new platform.</span></span> <span data-ttu-id="54060-119">Po vytváření prototypů klíčové prvky nového návrhu máme snadnější k opakovanému použití kódu z existujících systému v rámci nového.</span><span class="sxs-lookup"><span data-stu-id="54060-119">After prototyping key elements of the new design, it becomes easier to re-use code from the existing system within the new one.</span></span>
