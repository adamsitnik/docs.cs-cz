## <a name="port-sharing-architecture"></a><span data-ttu-id="73e5d-121">Architektura sdílení portu</span><span class="sxs-lookup"><span data-stu-id="73e5d-121">Port Sharing Architecture</span></span>

- <span data-ttu-id="73e5d-124">Přenos WCF TCP: Implementuje protokol net.tcp://.</span><span class="sxs-lookup"><span data-stu-id="73e5d-124">The WCF TCP transport: Implements the net.tcp:// protocol.</span></span>

 <span data-ttu-id="73e5d-126">Služba sdílení portů Net.TCP je služba Windows uživatelského režimu, který přijímá připojení net.tcp:// jménem pracovních procesů, které se připojují přes něj.</span><span class="sxs-lookup"><span data-stu-id="73e5d-126">The Net.TCP Port Sharing Service is a user-mode Windows service that accepts net.tcp:// connections on behalf of the worker processes that connect through it.</span></span> <span data-ttu-id="73e5d-127">Po přijetí připojení soketu služby Sdílení portů kontroluje příchozí datový proud zpráv získat jeho cílové adresy.</span><span class="sxs-lookup"><span data-stu-id="73e5d-127">When a socket connection arrives, the port sharing service inspects the incoming message stream to obtain its destination address.</span></span> <span data-ttu-id="73e5d-128">Na základě této adresy, služby Sdílení portů může směrovat na datový proud aplikace, takže v konečném důsledku zpracovává.</span><span class="sxs-lookup"><span data-stu-id="73e5d-128">Based on this address, the port sharing service can route the data stream to the application that ultimately processes it.</span></span>
