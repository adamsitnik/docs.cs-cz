# <a name="weakly-typed-json-serialization-sample"></a><span data-ttu-id="2a8db-102">Ukázka slabě typované serializace JSON</span><span class="sxs-lookup"><span data-stu-id="2a8db-102">Weakly-typed JSON Serialization Sample</span></span>
<span data-ttu-id="2a8db-103">Při serializaci uživatelem definovaný typ daného přenosový formát nebo deserializaci přenosový formát zpět do uživatelem definovaný typ, daný uživatelský typ musí být k dispozici na službu nebo klienta.</span><span class="sxs-lookup"><span data-stu-id="2a8db-103">When serializing a user-defined type to a given wire format, or deserializing a wire format back into a user-defined type, the given user-defined type must be available on both the service and the client.</span></span> <span data-ttu-id="2a8db-104">Obvykle k tomu, <xref:System.Runtime.Serialization.DataContractAttribute> atribut bude použit pro tyto typy definované uživatelem a <xref:System.Runtime.Serialization.DataMemberAttribute> atributu se použije pro jejich členy.</span><span class="sxs-lookup"><span data-stu-id="2a8db-104">Usually to accomplish this, the <xref:System.Runtime.Serialization.DataContractAttribute> attribute is applied to these user-defined types and the <xref:System.Runtime.Serialization.DataMemberAttribute> attribute is applied to their members.</span></span> <span data-ttu-id="2a8db-105">Tento mechanismus platí i v případě práce s objekty JavaScript Object Notation (JSON), jak je popsáno v tématu [jak: Serializace a deserializace dat protokolu JSON](../../../../docs/framework/wcf/feature-details/how-to-serialize-and-deserialize-json-data.md).</span><span class="sxs-lookup"><span data-stu-id="2a8db-105">This mechanism also applies when working with JavaScript Object Notation (JSON) objects, as described in the topic [How to: Serialize and Deserialize JSON Data](../../../../docs/framework/wcf/feature-details/how-to-serialize-and-deserialize-json-data.md).</span></span>  
  
 <span data-ttu-id="2a8db-106">V některých případech musí přístup služby Windows Communication Foundation (WCF) ani klienta vygeneruje služba nebo klient, který je mimo kontrolu vývojáře objekty JSON.</span><span class="sxs-lookup"><span data-stu-id="2a8db-106">In some scenarios, a Windows Communication Foundation (WCF) service or client must access JSON objects generated by a service or client that is outside of the control of the developer.</span></span> <span data-ttu-id="2a8db-107">Jako další webové služby vystavují veřejně JSON rozhraní API, může být nepraktické pro vývojáře WCF k vytvoření místního typy definované uživatelem, do které se má deserializovat libovolného objekty JSON.</span><span class="sxs-lookup"><span data-stu-id="2a8db-107">As more Web services publicly expose JSON APIs, it can become impractical for the WCF developer to construct local user-defined types into which to deserialize arbitrary JSON objects.</span></span> <span data-ttu-id="2a8db-108">Tato ukázka poskytuje mechanismus, který umožňuje vývojářům WCF pro práci s objekty JSON deserializovat, libovolného bez vytvoření uživatelem definovaných typů.</span><span class="sxs-lookup"><span data-stu-id="2a8db-108">This sample provides a mechanism that enables WCF developers to work with deserialized, arbitrary JSON objects, without creating user-defined types.</span></span> <span data-ttu-id="2a8db-109">To se označuje jako *slabě typovaná serializace* objektů JSON, protože typ, do kterého se deserializuje objekt JSON není známý v době kompilace.</span><span class="sxs-lookup"><span data-stu-id="2a8db-109">This is known as *weakly-typed serialization* of JSON objects, because the type into which a JSON object deserializes is not known at compile time.</span></span>  
  
> [!NOTE]
>  <span data-ttu-id="2a8db-110">Postup a sestavení pokynů pro tuto ukázku se nachází na konci tohoto tématu.</span><span class="sxs-lookup"><span data-stu-id="2a8db-110">The setup procedure and build instructions for this sample are located at the end of this topic.</span></span>  
  
 <span data-ttu-id="2a8db-111">Například veřejné rozhraní API webové služby vrátí následující objekt JSON, který popisuje některé informace o uživateli služby.</span><span class="sxs-lookup"><span data-stu-id="2a8db-111">For example, a public Web service API returns the following JSON object, which describes some information about a user of the service.</span></span>  
  
```json  
{"personal": {"name": "Paul", "age": 23, "height": 1.7, "isSingle": true, "luckyNumbers": [5,17,21]}, "favoriteBands": ["Band ABC", "Band XYZ"]}  
```  
  
 <span data-ttu-id="2a8db-112">K deserializaci tohoto objektu, musí implementovat klienta WCF uživatelem definované typy.</span><span class="sxs-lookup"><span data-stu-id="2a8db-112">To deserialize this object, a WCF client must implement the following user-defined types.</span></span>  
  
```  
[DataContract]  
 public class MemberProfile  
 {  
     [DataMember]  
     public PersonalInfo personal;  
  
     [DataMember]  
     public string[] favoriteBands;  
 }  
  
 [DataContract]  
 public class PersonalInfo  
 {  
     [DataMember]  
     public string name;  
  
     [DataMember]  
     public int age;  
  
     [DataMember]  
     public double height;  
  
     [DataMember]  
     public bool isSingle;  
  
     [DataMember]  
     public int[] luckyNumbers;  
 }  
```  
  
 <span data-ttu-id="2a8db-113">To může být náročné, zejména v případě, že klient má pro zpracování více než jeden typ objektu JSON.</span><span class="sxs-lookup"><span data-stu-id="2a8db-113">This can be cumbersome, especially if the client has to handle more than one type of JSON object.</span></span>  
  
 <span data-ttu-id="2a8db-114">`JsonObject` Typ zadaný v tomto příkladu představuje slabě typovaná reprezentace deserializovaný objekt JSON.</span><span class="sxs-lookup"><span data-stu-id="2a8db-114">The `JsonObject` type provided by this sample introduces a weakly-typed representation of the deserialized JSON object.</span></span> <span data-ttu-id="2a8db-115">`JsonObject` spoléhá na fyzické mapování mezi objekty JSON a [!INCLUDE[dnprdnshort](../../../../includes/dnprdnshort-md.md)] slovníky a mapování mezi poli JSON a [!INCLUDE[dnprdnshort](../../../../includes/dnprdnshort-md.md)] pole.</span><span class="sxs-lookup"><span data-stu-id="2a8db-115">`JsonObject` relies on the natural mapping between JSON objects and [!INCLUDE[dnprdnshort](../../../../includes/dnprdnshort-md.md)] dictionaries, and the mapping between JSON arrays and [!INCLUDE[dnprdnshort](../../../../includes/dnprdnshort-md.md)] arrays.</span></span> <span data-ttu-id="2a8db-116">Následující kód ukazuje `JsonObject` typu.</span><span class="sxs-lookup"><span data-stu-id="2a8db-116">The following code shows the `JsonObject` type.</span></span>  
  
```  
// Instantiation of JsonObject json omitted  
  
string name = json["root"]["personal"]["name"];  
int age = json["root"]["personal"]["age"];  
double height = json["root"]["personal"]["height"];  
bool isSingle = json["root"]["personal"]["isSingle"];  
int[] luckyNumbers = {  
                                     json["root"]["personal"]["luckyNumbers"][0],  
                                     json["root"]["personal"]["luckyNumbers"][1],  
                                     json["root"]["personal"]["luckyNumbers"][2]   
                                 };  
string[] favoriteBands = {  
                                        json["root"]["favoriteBands"][0],  
                                        json["root"]["favoriteBands"][1]  
                                    };  
```  
  
 <span data-ttu-id="2a8db-117">Všimněte si, že můžete "Vyhledat" objekty JSON a pole bez nutnosti pro deklaraci typu v době kompilace.</span><span class="sxs-lookup"><span data-stu-id="2a8db-117">Note that you can "browse" JSON objects and arrays without the need to declare their type at compile time.</span></span> <span data-ttu-id="2a8db-118">Vysvětlení požadavek na nejvyšší úrovni `["root"]` objektu, naleznete v tématu [mapování mezi JSON a XML](../../../../docs/framework/wcf/feature-details/mapping-between-json-and-xml.md).</span><span class="sxs-lookup"><span data-stu-id="2a8db-118">For an explanation of the requirement for the top-level `["root"]` object, see the topic [Mapping Between JSON and XML](../../../../docs/framework/wcf/feature-details/mapping-between-json-and-xml.md).</span></span>  
  
> [!NOTE]
>  <span data-ttu-id="2a8db-119">`JsonObject` Třídy je k dispozici pouze jako příklad.</span><span class="sxs-lookup"><span data-stu-id="2a8db-119">The `JsonObject` class is provided as an example only.</span></span> <span data-ttu-id="2a8db-120">Neprošla dostatečně důkladnými a neměli byste používat v produkčním prostředí.</span><span class="sxs-lookup"><span data-stu-id="2a8db-120">It has not been thoroughly tested, and should not be used in production environments.</span></span> <span data-ttu-id="2a8db-121">Zřejmé nepřímo slabě typovaná serializace JSON je nedostatek bezpečnost typů, pokud pracujete s `JsonObject`.</span><span class="sxs-lookup"><span data-stu-id="2a8db-121">An obvious implication of weakly-typed JSON serialization is the lack of type-safety when working with `JsonObject`.</span></span>  
  
 <span data-ttu-id="2a8db-122">Použít `JsonObject` typu, musíte použít klienta kontrakt <xref:System.ServiceModel.Channels.Message> jako její typ vrácené hodnoty.</span><span class="sxs-lookup"><span data-stu-id="2a8db-122">To use the `JsonObject` type, the client operation contract must use <xref:System.ServiceModel.Channels.Message> as its return type.</span></span>  
  
```

    interface IClientSideProfileService  
    {  
        // There is no need to write a DataContract for the complex type returned by the service.  
        // The client will use a JsonObject to browse the JSON in the received message.  
  
        [OperationContract]  
        [WebGet(ResponseFormat = WebMessageFormat.Json)]  
        Message GetMemberProfile();  
    }  
```  
  
 <span data-ttu-id="2a8db-123">`JsonObject` Je pak instance, jak je znázorněno v následujícím kódu.</span><span class="sxs-lookup"><span data-stu-id="2a8db-123">The `JsonObject` is then instantiated as shown in the following code.</span></span>

```  
// Code to instantiate IClientSideProfileService channel omitted…  
  
// Make a request to the service and obtain the Json response

// Go through the Json as though it is a dictionary. There is no need to map it to a .NET CLR type.

```  
  
 <span data-ttu-id="2a8db-124">`JsonObject` Přebírá konstruktor <xref:System.Xml.XmlDictionaryReader>, kterého se získává <xref:System.ServiceModel.Channels.Message.GetReaderAtBodyContents%2A> metoda.</span><span class="sxs-lookup"><span data-stu-id="2a8db-124">The `JsonObject` constructor takes a <xref:System.Xml.XmlDictionaryReader>, which is obtained through the <xref:System.ServiceModel.Channels.Message.GetReaderAtBodyContents%2A> method.</span></span> <span data-ttu-id="2a8db-125">Čtecí modul obsahuje reprezentaci v jazyce XML JSON zprávu přijatou klientem.</span><span class="sxs-lookup"><span data-stu-id="2a8db-125">The reader contains an XML representation of the JSON message received by the client.</span></span> <span data-ttu-id="2a8db-126">Další informace naleznete v tématu [mapování mezi JSON a XML](../../../../docs/framework/wcf/feature-details/mapping-between-json-and-xml.md).</span><span class="sxs-lookup"><span data-stu-id="2a8db-126">For more information, see the topic [Mapping Between JSON and XML](../../../../docs/framework/wcf/feature-details/mapping-between-json-and-xml.md).</span></span>  
  
 <span data-ttu-id="2a8db-127">Program vygeneruje následující výstup:</span><span class="sxs-lookup"><span data-stu-id="2a8db-127">The program produces the following output:</span></span>
