## <a name="tracing"></a><span data-ttu-id="2959e-107">Trasování</span><span class="sxs-lookup"><span data-stu-id="2959e-107">Tracing</span></span>  
 <span data-ttu-id="2959e-108">Windows Communication Foundation (WCF) používá mechanismus trasování, které jsou definovány v <xref:System.Diagnostics> oboru názvů.</span><span class="sxs-lookup"><span data-stu-id="2959e-108">Windows Communication Foundation (WCF) uses the tracing mechanism defined in the <xref:System.Diagnostics> namespace.</span></span> <span data-ttu-id="2959e-109">V tomto modelu trasování je dat trasování vyprodukované zdroji trasování, které implementují aplikace.</span><span class="sxs-lookup"><span data-stu-id="2959e-109">In this tracing model, trace data is produced by trace sources that applications implement.</span></span> <span data-ttu-id="2959e-110">Každý zdroj je identifikována názvem.</span><span class="sxs-lookup"><span data-stu-id="2959e-110">Each source is identified by a name.</span></span> <span data-ttu-id="2959e-111">Trasování uživatelé vytváří naslouchacích procesů trasování pro trasování zdrojů, které chtějí získat informace.</span><span class="sxs-lookup"><span data-stu-id="2959e-111">Trace consumers create trace listeners for the trace sources for which they want to retrieve information.</span></span> <span data-ttu-id="2959e-112">Pokud chcete přijímat data trasování, musíte vytvořit naslouchací proces pro zdroj trasování.</span><span class="sxs-lookup"><span data-stu-id="2959e-112">To receive trace data, you must create a listener for the trace source.</span></span> <span data-ttu-id="2959e-113">Ve službě WCF, to lze provést přidáním následujícího kódu do konfiguračního souboru služby nebo klienta tak, že nastavíte zdroj trasování modelu služby `switchValue`:</span><span class="sxs-lookup"><span data-stu-id="2959e-113">In WCF, this can be done by adding the following code to either the service’s or client’s configuration file by setting the Service Model trace source `switchValue`:</span></span>  
  
```xml  
<system.diagnostics>  
    <sources>  
      <source name="System.ServiceModel" switchValue="Information,ActivityTracing"  
        propagateActivity="true">  
        <listeners>  
          <add name="xml" />  
        </listeners>  
      </source>  
      <source name="System.ServiceModel.MessageLogging">  
        <listeners>  
          <add name="xml" />  
        </listeners>  
      </source>  
    </sources>  
    <sharedListeners>  
      <add initializeData="C:\logs\TracingAndLogging-service.svclog" type="System.Diagnostics.XmlWriterTraceListener"  
        name="xml" />  
    </sharedListeners>  
    <trace autoflush="true" />  
</system.diagnostics>  
```  
  
 <span data-ttu-id="2959e-114">Další informace o zdrojů trasování, naleznete v části Zdroj trasování v [Konfigurace trasování](../../../../docs/framework/wcf/diagnostics/tracing/configuring-tracing.md) tématu.</span><span class="sxs-lookup"><span data-stu-id="2959e-114">For more information about trace sources, see the Trace Source section in the [Configuring Tracing](../../../../docs/framework/wcf/diagnostics/tracing/configuring-tracing.md) topic.</span></span>  
  
## <a name="activity-tracing-and-propagation"></a><span data-ttu-id="2959e-115">Trasování aktivit a šíření</span><span class="sxs-lookup"><span data-stu-id="2959e-115">Activity Tracing and Propagation</span></span>  
 <span data-ttu-id="2959e-116">S `ActivityTracing` povolené a `propagateActivity` nastavena na `true` v `system.ServiceModel` zdroje trasování klienta a služby poskytovat korelace trasování v rámci logické jednotky (aktivity), zpracování napříč aktivity v rámci koncových bodů ( pomocí aktivity převody) a také napříč aktivity pokrývající několik koncových bodů (prostřednictvím šíření ID aktivity).</span><span class="sxs-lookup"><span data-stu-id="2959e-116">Having `ActivityTracing` enabled and `propagateActivity` set to `true` in the `system.ServiceModel` trace sources for both the client and service provide correlation of traces within logical units of processing (activities), across activities within endpoints (through activity transfers), and across activities spanning multiple endpoints (through activity ID propagation).</span></span>  
  
 <span data-ttu-id="2959e-117">Tyto tři mechanismy (aktivity, přenosy a šíření) vám pomůže najít hlavní příčinu chyby více rychle pomocí nástroje prohlížeče trasování služeb.</span><span class="sxs-lookup"><span data-stu-id="2959e-117">These three mechanisms (activities, transfers, and propagation) can help you locate the root cause of an error more quickly using the Service Trace Viewer tool.</span></span> <span data-ttu-id="2959e-118">Další informace najdete v tématu [pomocí prohlížeče trasování služeb k zobrazení korelovaných trasování a řešení potíží](../../../../docs/framework/wcf/diagnostics/tracing/using-service-trace-viewer-for-viewing-correlated-traces-and-troubleshooting.md).</span><span class="sxs-lookup"><span data-stu-id="2959e-118">For more information, see [Using Service Trace Viewer for Viewing Correlated Traces and Troubleshooting](../../../../docs/framework/wcf/diagnostics/tracing/using-service-trace-viewer-for-viewing-correlated-traces-and-troubleshooting.md).</span></span>  
  
 <span data-ttu-id="2959e-119">Je možné rozšířit trasování, která je poskytována ServiceModel tak, že vytvoříte trasy definované uživatelem aktivity.</span><span class="sxs-lookup"><span data-stu-id="2959e-119">It is possible to extend the tracing that is provided by the ServiceModel by creating user-defined activity traces.</span></span> <span data-ttu-id="2959e-120">Trasování činnosti uživatelem definované umožňuje uživateli vytvořit trasování činnosti:</span><span class="sxs-lookup"><span data-stu-id="2959e-120">User-defined activity tracing allows the user to create trace activities to:</span></span>

-   <span data-ttu-id="2959e-123">Snížit náklady na trasování WCF (například náklady místa na disku souboru protokolu).</span><span class="sxs-lookup"><span data-stu-id="2959e-123">Lessen the performance cost of WCF tracing (for example, the disk space cost of a log file).</span></span>

## <a name="message-logging"></a><span data-ttu-id="2959e-125">Protokolování zpráv</span><span class="sxs-lookup"><span data-stu-id="2959e-125">Message Logging</span></span>  
 <span data-ttu-id="2959e-126">Jak na klienta a služby z libovolné aplikace WCF je možné povolit protokolování zpráv.</span><span class="sxs-lookup"><span data-stu-id="2959e-126">Message logging can be enabled both on the client and service of any WCF application.</span></span> <span data-ttu-id="2959e-127">Povolit protokolování zpráv, musíte přidat následující kód k klienta nebo službě:</span><span class="sxs-lookup"><span data-stu-id="2959e-127">To enable message logging, you must add the following code to either the client or service:</span></span>  
  
```xml  
<configuration>  
  <system.serviceModel>  
    <diagnostics>  
      <!-- Enable Message Logging here. -->  
      <!-- log all messages received or sent at the transport or service model levels -->  
      <messageLogging logEntireMessage="true"  
                      maxMessagesToLog="300"  
                      logMessagesAtServiceLevel="true"  
                      logMalformedMessages="true"  
                      logMessagesAtTransportLevel="true" />  
    </diagnostics>  
  </system.serviceModel>  
</configuration>  
```  
  
 <span data-ttu-id="2959e-128">Když se zaznamená zprávu, typ trasování závisí na tom, jestli jsou trasovány na klienta nebo serveru.</span><span class="sxs-lookup"><span data-stu-id="2959e-128">When a message is recorded, the trace type depends on whether it is being traced at the client or the server.</span></span> <span data-ttu-id="2959e-129">Například zpráva "Add" odeslaný klientem je sledována v kategorii "TransportWrite" u klienta, že stejné zprávy je sledována v kategorii "TransportRead" na službu.</span><span class="sxs-lookup"><span data-stu-id="2959e-129">For example, an "Add" message that is sent to a client is traced under the "TransportWrite" category at the client, whereas the same message is traced under the "TransportRead" category at the service.</span></span>  
  
 <span data-ttu-id="2959e-130">Konfigurace naslouchacího procesu trasování přidáním následujícího kódu <xref:System.Diagnostics> část souboru App.config klienta nebo v souboru Web.config služby:</span><span class="sxs-lookup"><span data-stu-id="2959e-130">Configure the trace listener by adding the following code to the <xref:System.Diagnostics> section of the client's App.config file or the service's Web.config file:</span></span>  
  
```xml  
<system.diagnostics>  
    <sources>  
      <source name="System.ServiceModel" switchValue="Information,ActivityTracing"  
        propagateActivity="true">  
        <listeners>  
          <add name="xml" />  
        </listeners>  
      </source>  
      <source name="System.ServiceModel.MessageLogging">  
        <listeners>  
          <add name="xml" />  
        </listeners>  
      </source>  
    </sources>  
    <sharedListeners>  
      <add initializeData="C:\logs\TracingAndLogging-client.svclog" type="System.Diagnostics.XmlWriterTraceListener"  
        name="xml" />  
    </sharedListeners>  
    <trace autoflush="true" />  
  </system.diagnostics>  
```  
  
 <span data-ttu-id="2959e-131">Zprávy jsou protokolovány ve formátu XML v cílovém adresáři zadané v konfiguračním souboru.</span><span class="sxs-lookup"><span data-stu-id="2959e-131">Messages are logged in XML format in the target directory specified in the configuration file.</span></span>  
  
> [!NOTE]
>  <span data-ttu-id="2959e-132">Trasovací soubory nejsou vytvořeny bez vytvoření původně adresář protokolu.</span><span class="sxs-lookup"><span data-stu-id="2959e-132">Trace files are not created without initially creating the log directory.</span></span> <span data-ttu-id="2959e-133">Ujistěte se, že existuje adresář C:\logs\ nebo určit alternativní protokolování adresář v konfiguraci naslouchacího procesu.</span><span class="sxs-lookup"><span data-stu-id="2959e-133">Make sure that the directory C:\logs\ exists, or specify an alternate logging directory in the listener configuration.</span></span> <span data-ttu-id="2959e-134">Přečtěte si pokyny počáteční nastavení na konci tohoto dokumentu pro další informace.</span><span class="sxs-lookup"><span data-stu-id="2959e-134">See the initial setup instructions at the end of this document for more information.</span></span>  
  
 <span data-ttu-id="2959e-135">Další informace o protokolování zpráv, najdete v článku [konfigurace protokolování zpráv](../../../../docs/framework/wcf/diagnostics/configuring-message-logging.md) tématu.</span><span class="sxs-lookup"><span data-stu-id="2959e-135">For more information about message logging, see the [Configuring Message Logging](../../../../docs/framework/wcf/diagnostics/configuring-message-logging.md) topic.</span></span>
