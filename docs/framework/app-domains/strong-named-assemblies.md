 <span data-ttu-id="c8e53-110">Když je vytvořeno sestavení se silným názvem, obsahuje jednoduchý textový název sestavení, číslo verze, volitelné informace o jazykové verzi, digitální podpis a veřejný klíč, který odpovídá privátnímu klíči používanému k podepisování.</span><span class="sxs-lookup"><span data-stu-id="c8e53-110">When a strong-named assembly is created, it contains the simple text name of the assembly, the version number, optional culture information, a digital signature, and the public key that corresponds to the private key used for signing.</span></span>

## <a name="why-strong-name-your-assemblies"></a><span data-ttu-id="c8e53-113">Proč je sestavení silným názvem?</span><span class="sxs-lookup"><span data-stu-id="c8e53-113">Why strong-name your assemblies?</span></span>  
 <span data-ttu-id="c8e53-114">Když odkazujete na sestavení se silným názvem, můžete očekávat určité výhody, jako je například Správa verzí a ochrana názvů.</span><span class="sxs-lookup"><span data-stu-id="c8e53-114">When you reference a strong-named assembly, you can expect certain benefits, such as versioning and naming protection.</span></span> <span data-ttu-id="c8e53-115">Sestavení se silným názvem může být instalována v globální mezipaměti sestavení (GAC), která je nutná k povolení některých scénářů.</span><span class="sxs-lookup"><span data-stu-id="c8e53-115">Strong-named assemblies can be installed in the Global Assembly Cache, which is required to enable some scenarios.</span></span>  
  
 <span data-ttu-id="c8e53-116">Sestavení se silným názvem jsou užitečná v následujících situacích:</span><span class="sxs-lookup"><span data-stu-id="c8e53-116">Strong-named assemblies are useful in the following scenarios:</span></span>  
  
- <span data-ttu-id="c8e53-117">Chcete povolit odkazování na vaše sestavení pomocí sestavení se silným názvem, nebo chcete udělit `friend` přístup k sestavením z jiných sestavení se silným názvem.</span><span class="sxs-lookup"><span data-stu-id="c8e53-117">You want to enable your assemblies to be referenced by strong-named assemblies, or you want to give `friend` access to your assemblies from other strong-named assemblies.</span></span>  
  
- <span data-ttu-id="c8e53-118">Aplikace potřebuje přístup k různým verzím stejného sestavení.</span><span class="sxs-lookup"><span data-stu-id="c8e53-118">An app needs access to different versions of the same assembly.</span></span> <span data-ttu-id="c8e53-119">To znamená, že budete potřebovat různé verze sestavení pro zatížení souběžně ve stejné doméně aplikace bez konfliktu.</span><span class="sxs-lookup"><span data-stu-id="c8e53-119">This means  you need different versions of an assembly to load side by side in the same app domain without conflict.</span></span> <span data-ttu-id="c8e53-120">Například pokud různá rozšíření rozhraní API existují v sestaveních se stejným jednoduchým názvem, silné názvy poskytují jedinečnou identitu pro každou verzi sestavení.</span><span class="sxs-lookup"><span data-stu-id="c8e53-120">For example, if different extensions of an API exist in assemblies that have the same simple name, strong-naming provides a unique identity for each version of the assembly.</span></span>  
  
- <span data-ttu-id="c8e53-121">Nechcete negativně ovlivnit výkon aplikací, které používají sestavení, takže chcete, aby bylo sestavení v doméně neutrální.</span><span class="sxs-lookup"><span data-stu-id="c8e53-121">You do not want to negatively affect performance of apps using your assembly, so you want the assembly to be domain neutral.</span></span> <span data-ttu-id="c8e53-122">To vyžaduje silné pojmenovávání, protože v globální mezipaměti sestavení (GAC) musí být nainstalována doménově neutrální sestavení.</span><span class="sxs-lookup"><span data-stu-id="c8e53-122">This requires strong-naming because a domain-neutral assembly must be installed in the global assembly cache.</span></span>  
  
- <span data-ttu-id="c8e53-123">Pokud chcete centralizovat obsluhu pro vaši aplikaci pomocí zásad vydavatele, což znamená, že sestavení musí být nainstalováno v globální mezipaměti sestavení (GAC).</span><span class="sxs-lookup"><span data-stu-id="c8e53-123">When you want to centralize servicing for your app by applying publisher policy, which means the assembly must be installed in the  global assembly cache.</span></span>  
  
 <span data-ttu-id="c8e53-124">Pokud jste Open Source vývojář a chcete mít výhody identity sestavení se silným názvem, zvažte vrácení privátního klíče přidruženého k sestavení do systému správy zdrojového kódu.</span><span class="sxs-lookup"><span data-stu-id="c8e53-124">If you are an open-source developer and you want the identity benefits of a strong-named assembly, consider checking in the private key associated with an assembly into your source control system.</span></span>
