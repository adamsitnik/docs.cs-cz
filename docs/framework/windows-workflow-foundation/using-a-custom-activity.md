# <a name="using-a-custom-activity"></a><span data-ttu-id="4700b-102">Použití vlastní aktivity</span><span class="sxs-lookup"><span data-stu-id="4700b-102">Using a custom activity</span></span>
<span data-ttu-id="4700b-103">Aktivity, které jsou odvozeny z <xref:System.Activities.Activity> nebo jejích podtříd složený do větší pracovních postupů, nebo přímo v kódu.</span><span class="sxs-lookup"><span data-stu-id="4700b-103">Activities that derive from <xref:System.Activities.Activity> or its subclasses can be composed into larger workflows, or created directly in code.</span></span> <span data-ttu-id="4700b-104">Toto téma popisuje použití vlastních aktivit v vytvořených v kódu nebo v Návrháři pracovních postupů.</span><span class="sxs-lookup"><span data-stu-id="4700b-104">This topic describes how to use custom activities in workflows created either in code or in the designer.</span></span>  
  
> [!NOTE]
>  <span data-ttu-id="4700b-105">Můžete použít vlastní aktivity ve stejném projektu, ve kterém jsou definovány, tak dlouho, dokud vlastní aktivity a aktivity, která ji používá jsou zkompilovány (například načíst konkretizujete typem generovaných proces sestavení) Pokud je načtena odkazující aktivita dynamicky (např. při použití služba ActivityXAMLServices), pak odkazované sestavení umístit do jiného projektu nebo generovaný návrhářem XAML je třeba ručně upravit a povolit.</span><span class="sxs-lookup"><span data-stu-id="4700b-105">Custom activities can be used in the same project in which they are defined, as long as both the custom activity and the activity that uses it are compiled (i.e. loaded by an instantiating type generated by the build process) If the referencing activity is loaded dynamically (e.g. using ActivityXAMLServices), then the referenced assembly should be placed in a different project, or the designer-generated XAML needs to be hand-edited to enable this.</span></span>  
  
#### <a name="using-a-custom-activity-to-a-workflow-project"></a><span data-ttu-id="4700b-106">Použití vlastní aktivity do projektu pracovního postupu</span><span class="sxs-lookup"><span data-stu-id="4700b-106">Using a custom activity to a workflow project</span></span>  
  
1. <span data-ttu-id="4700b-107">Přidáte odkaz na projekt knihovny aktivit obsahující vlastní aktivitu z projektu hostitel.</span><span class="sxs-lookup"><span data-stu-id="4700b-107">Add a reference from the host project to the activity library project containing the custom activity.</span></span>  
  
2. <span data-ttu-id="4700b-108">Sestavte řešení.</span><span class="sxs-lookup"><span data-stu-id="4700b-108">Build the solution.</span></span>  
  
3. <span data-ttu-id="4700b-109">Chcete-li použít vlastní aktivity v návrháři, vyhledejte vlastní aktivity v sadě nástrojů a přetažením aktivity na plochu návrháře.</span><span class="sxs-lookup"><span data-stu-id="4700b-109">To use the custom activity in the designer, locate the custom activity in the toolbox, and drag the activity onto the designer surface.</span></span>  
  
4. <span data-ttu-id="4700b-110">Použití vlastní aktivity v kódu, přidejte příkaz Using, který odkazuje na projekt vlastní aktivity a předat novou instanci třídy aktivity na <xref:System.Activities.WorkflowInvoker.Invoke%2A>.</span><span class="sxs-lookup"><span data-stu-id="4700b-110">To use the custom activity in code, add a Using statement that refers to the custom activity project, and pass a new instance of the activity to <xref:System.Activities.WorkflowInvoker.Invoke%2A>.</span></span>
