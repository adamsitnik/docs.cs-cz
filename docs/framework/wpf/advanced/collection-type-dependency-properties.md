 <span data-ttu-id="85abf-105">Pro vlastnost závislosti obecně implementaci vzoru, který sledujete je, že definujete [!INCLUDE[TLA2#tla_clr](../../../../includes/tla2sharptla-clr-md.md)] vlastnost obálku, ve kterém je zajištěná tuto vlastnost <xref:System.Windows.DependencyProperty> identifikátor spíše než pole nebo jiné konstrukce.</span><span class="sxs-lookup"><span data-stu-id="85abf-105">For a dependency property in general, the implementation pattern that you follow is that you define a [!INCLUDE[TLA2#tla_clr](../../../../includes/tla2sharptla-clr-md.md)] property wrapper, where that property is backed by a <xref:System.Windows.DependencyProperty> identifier rather than a field or other construct.</span></span> <span data-ttu-id="85abf-106">Můžete sledovat tento stejný vzor při implementaci vlastnosti typu kolekce.</span><span class="sxs-lookup"><span data-stu-id="85abf-106">You follow this same pattern when you implement a collection-type property.</span></span> <span data-ttu-id="85abf-107">Však vlastnost typ kolekce zavádí některé složitost se vzorem pokaždé, když je typ, který je součástí kolekce <xref:System.Windows.DependencyObject> nebo <xref:System.Windows.Freezable> odvozené třídy.</span><span class="sxs-lookup"><span data-stu-id="85abf-107">However, a collection-type property introduces some complexity to the pattern whenever the type that is contained within the collection is itself a <xref:System.Windows.DependencyObject> or <xref:System.Windows.Freezable> derived class.</span></span>

 <span data-ttu-id="85abf-130">Vlastnost kolekce, který je sám vlastnost závislosti automaticky nehlásí změny k jeho objektu třídy subproperties.</span><span class="sxs-lookup"><span data-stu-id="85abf-130">A collection property that is itself a dependency property does not automatically report changes to its subproperties.</span></span> <span data-ttu-id="85abf-131">Při vytváření vazby do kolekce, můžou zabránit vazbě z hlášení změn, tedy zrušení platnosti některých scénáře datových vazeb.</span><span class="sxs-lookup"><span data-stu-id="85abf-131">If you are creating bindings into a collection, this can prevent the binding from reporting changes, thus invalidating some data binding scenarios.</span></span> <span data-ttu-id="85abf-132">Nicméně pokud použijete typ kolekce <xref:System.Windows.FreezableCollection%601> jako typ kolekce, pak podvlastností změny jeho prvky v kolekci správně označené a vazby funguje podle očekávání.</span><span class="sxs-lookup"><span data-stu-id="85abf-132">However, if you use the collection type <xref:System.Windows.FreezableCollection%601> as your collection type, then subproperty changes to contained elements in the collection are properly reported, and binding works as expected.</span></span>
