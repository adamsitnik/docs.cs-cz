## <a name="implementation-guidelines-and-conventions"></a><span data-ttu-id="0451f-110">Pokyny a konvence implementace</span><span class="sxs-lookup"><span data-stu-id="0451f-110">Implementation Guidelines and Conventions</span></span>

  - <span data-ttu-id="0451f-119">Akce vyžaduje zásah uživatele.</span><span class="sxs-lookup"><span data-stu-id="0451f-119">The action requires user interaction.</span></span>

  - <span data-ttu-id="0451f-120">Akce je časově náročná a způsobí, že volající klient bude po významné době blokovat.</span><span class="sxs-lookup"><span data-stu-id="0451f-120">The action is time-consuming and will cause the calling client to block for a significant amount of time.</span></span>

- <span data-ttu-id="0451f-121">Pokud vyvolání ovládacího prvku má významné vedlejší účinky, tyto vedlejší účinky by měly být vystaveny prostřednictvím <xref:System.Windows.Automation.AutomationElement.AutomationElementInformation.HelpText%2A> vlastnosti.</span><span class="sxs-lookup"><span data-stu-id="0451f-121">If invoking the control has significant side-effects, those side-effects should be exposed through the <xref:System.Windows.Automation.AutomationElement.AutomationElementInformation.HelpText%2A> property.</span></span> <span data-ttu-id="0451f-122">Například, i když <xref:System.Windows.Automation.Provider.IInvokeProvider.Invoke%2A> není přidružen k výběru, může způsobit, <xref:System.Windows.Automation.Provider.IInvokeProvider.Invoke%2A> že se vybere jiný ovládací prvek.</span><span class="sxs-lookup"><span data-stu-id="0451f-122">For example, even though <xref:System.Windows.Automation.Provider.IInvokeProvider.Invoke%2A> is not associated with selection, <xref:System.Windows.Automation.Provider.IInvokeProvider.Invoke%2A> may cause another control to become selected.</span></span>
