# <a name="xfieldmodifier-directive"></a><span data-ttu-id="a59ad-102">x:FieldModifier – direktiva</span><span class="sxs-lookup"><span data-stu-id="a59ad-102">x:FieldModifier Directive</span></span>
<span data-ttu-id="a59ad-103">Upravuje chování sestavení XAML, takže pole pro odkazy na pojmenované objekty jsou definovány pomocí <xref:System.Reflection.TypeAttributes.Public?displayProperty=nameWithType> přístup místo <xref:System.Reflection.TypeAttributes.NotPublic?displayProperty=nameWithType> výchozí chování.</span><span class="sxs-lookup"><span data-stu-id="a59ad-103">Modifies XAML compilation behavior so that fields for named object references are defined with <xref:System.Reflection.TypeAttributes.Public?displayProperty=nameWithType> access instead of the <xref:System.Reflection.TypeAttributes.NotPublic?displayProperty=nameWithType> default behavior.</span></span>


 <span data-ttu-id="a59ad-122"><xref:System.Reflection.TypeAttributes.NotPublic?displayProperty=nameWithType> je výchozí chování, protože je zřídka, že kód mimo sestavení, který zkompiluje XAML potřebuje přístup k prvku XAML vytvořit.</span><span class="sxs-lookup"><span data-stu-id="a59ad-122"><xref:System.Reflection.TypeAttributes.NotPublic?displayProperty=nameWithType> is the default behavior because it is infrequent that code outside the assembly that compiled the XAML needs access to a XAML-created element.</span></span> <span data-ttu-id="a59ad-123">Architektura WPF zabezpečení spolu s chování kompilace XAML nebude deklarovat pole, které ukládají instance elementu jako veřejné, pokud je výslovně nastavit `x:FieldModifier` umožní veřejný přístup.</span><span class="sxs-lookup"><span data-stu-id="a59ad-123">WPF security architecture together with XAML compilation behavior will not declare fields that store element instances as public, unless you specifically set the `x:FieldModifier` to allow public access.</span></span>
