## <a name="deferred-execution"></a><span data-ttu-id="2a353-105">Odložené provedení</span><span class="sxs-lookup"><span data-stu-id="2a353-105">Deferred Execution</span></span>  
 <span data-ttu-id="2a353-106">Odložené provedení znamená, že vyhodnocení výrazu je odložena až do jeho *realizované* hodnota je skutečně nutná.</span><span class="sxs-lookup"><span data-stu-id="2a353-106">Deferred execution means that the evaluation of an expression is delayed until its *realized* value is actually required.</span></span> <span data-ttu-id="2a353-107">Odložené provedení může výrazně zlepšit výkon, když máte k manipulaci s kolekcí velkých objemů dat, zejména v aplikacích, které obsahují řadu zřetězených dotazů nebo manipulace.</span><span class="sxs-lookup"><span data-stu-id="2a353-107">Deferred execution can greatly improve performance when you have to manipulate large data collections, especially in programs that contain a series of chained queries or manipulations.</span></span> <span data-ttu-id="2a353-108">Odložené provedení v nejlepším případě umožňuje pouze jednu iterace přes zdrojové kolekce.</span><span class="sxs-lookup"><span data-stu-id="2a353-108">In the best case, deferred execution enables only a single iteration through the source collection.</span></span>  
  
 <span data-ttu-id="2a353-109">Technologie LINQ využívat rozsáhlé odložené provedení v členy základní <xref:System.Linq?displayProperty=nameWithType> třídy a metody rozšíření v různých oborech názvů LINQ, jako například <xref:System.Xml.Linq.Extensions?displayProperty=nameWithType>.</span><span class="sxs-lookup"><span data-stu-id="2a353-109">The LINQ technologies make extensive use of deferred execution in both the members of core <xref:System.Linq?displayProperty=nameWithType> classes and in the extension methods in the various LINQ namespaces, such as <xref:System.Xml.Linq.Extensions?displayProperty=nameWithType>.</span></span>  
  
## <a name="eager-vs-lazy-evaluation"></a><span data-ttu-id="2a353-110">Nemůžou dočkat, až vs. Opožděné vyhodnocení</span><span class="sxs-lookup"><span data-stu-id="2a353-110">Eager vs. Lazy Evaluation</span></span>  
 <span data-ttu-id="2a353-111">Když napíšete metodu, která implementuje odložené provedení, máte také rozhodnout, jestli se má implementovat metodu pomocí opožděné vyhodnocení nebo nemůžou dočkat, až hodnocení.</span><span class="sxs-lookup"><span data-stu-id="2a353-111">When you write a method that implements deferred execution, you also have to decide whether to implement the method using lazy evaluation or eager evaluation.</span></span>  
  
-   <span data-ttu-id="2a353-112">V *opožděné vyhodnocení*, jeden element zdrojové kolekce je zpracovaných za každé volání iterátoru.</span><span class="sxs-lookup"><span data-stu-id="2a353-112">In *lazy evaluation*, a single element of the source collection is processed during each call to the iterator.</span></span> <span data-ttu-id="2a353-113">Toto je typická způsob, ve kterém jsou implementovány iterátory.</span><span class="sxs-lookup"><span data-stu-id="2a353-113">This is the typical way in which iterators are implemented.</span></span>  
  
-   <span data-ttu-id="2a353-114">V *nemůžou dočkat, až hodnocení*, první volání iterátoru způsobí celou kolekci právě zpracovává.</span><span class="sxs-lookup"><span data-stu-id="2a353-114">In *eager evaluation*, the first call to the iterator will result in the entire collection being processed.</span></span> <span data-ttu-id="2a353-115">Dočasná kopie zdrojové kolekce může být také potřeba.</span><span class="sxs-lookup"><span data-stu-id="2a353-115">A temporary copy of the source collection might also be required.</span></span> <span data-ttu-id="2a353-116">Například <xref:System.Linq.Enumerable.OrderBy%2A> metoda se má seřadit celou kolekci dříve, než vrátí první prvek.</span><span class="sxs-lookup"><span data-stu-id="2a353-116">For example, the <xref:System.Linq.Enumerable.OrderBy%2A> method has to sort the entire collection before it returns the first element.</span></span>  
  
 <span data-ttu-id="2a353-117">Opožděné vyhodnocení obvykle poskytuje lepší výkon, protože distribuuje nároky na zpracování rovnoměrně během vyhodnocení kolekce a minimalizuje využití dočasná data.</span><span class="sxs-lookup"><span data-stu-id="2a353-117">Lazy evaluation usually yields better performance because it distributes overhead processing evenly throughout the evaluation of the collection and minimizes the use of temporary data.</span></span> <span data-ttu-id="2a353-118">Samozřejmě pro některé operace, neexistuje žádná možnost než a materializovat mezilehlých výsledků.</span><span class="sxs-lookup"><span data-stu-id="2a353-118">Of course, for some operations, there is no other option than to materialize intermediate results.</span></span>
