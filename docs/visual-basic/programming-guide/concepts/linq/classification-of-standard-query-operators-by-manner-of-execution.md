<span data-ttu-id="4039d-103">LINQ to Objects implementace standardních metod operátoru dotazu se spouštějí jedním ze dvou hlavních způsobů: Immediate nebo odloženo.</span><span class="sxs-lookup"><span data-stu-id="4039d-103">The LINQ to Objects implementations of the standard query operator methods execute in one of two main ways: immediate or deferred.</span></span> <span data-ttu-id="4039d-104">Operátory dotazů, které používají odložené provádění, lze dále rozdělit do dvou kategorií: streaming a non-stream.</span><span class="sxs-lookup"><span data-stu-id="4039d-104">The query operators that use deferred execution can be additionally divided into two categories: streaming and non-streaming.</span></span> <span data-ttu-id="4039d-105">Pokud víte, jak se spouštějí různé operátory dotazu, může vám porozumět výsledkům, které získáte z daného dotazu.</span><span class="sxs-lookup"><span data-stu-id="4039d-105">If you know how the different query operators execute, it may help you understand the results that you get from a given query.</span></span> <span data-ttu-id="4039d-106">To platí zejména v případě, že se změnil zdroj dat nebo pokud vytváříte dotaz nad jiným dotazem.</span><span class="sxs-lookup"><span data-stu-id="4039d-106">This is especially true if the data source is changing or if you are building a query on top of another query.</span></span> <span data-ttu-id="4039d-107">Toto téma klasifikuje standardní operátory dotazu podle jejich způsobu provádění.</span><span class="sxs-lookup"><span data-stu-id="4039d-107">This topic classifies the standard query operators according to their manner of execution.</span></span>  
  
## <a name="manners-of-execution"></a><span data-ttu-id="4039d-108">Způsoby spuštění</span><span class="sxs-lookup"><span data-stu-id="4039d-108">Manners of Execution</span></span>  
  
### <a name="immediate"></a><span data-ttu-id="4039d-109">Okamžité</span><span class="sxs-lookup"><span data-stu-id="4039d-109">Immediate</span></span>  
 <span data-ttu-id="4039d-110">Okamžité provedení znamená, že zdroj dat je čten a operace je provedena v místě v kódu, kde je dotaz deklarován.</span><span class="sxs-lookup"><span data-stu-id="4039d-110">Immediate execution means that the data source is read and the operation is performed at the point in the code where the query is declared.</span></span> <span data-ttu-id="4039d-111">Všechny standardní operátory dotazu, které vracejí jeden nevyčíslitelné výsledek, se spustí okamžitě.</span><span class="sxs-lookup"><span data-stu-id="4039d-111">All the standard query operators that return a single, non-enumerable result execute immediately.</span></span>  
  
### <a name="deferred"></a><span data-ttu-id="4039d-112">Odložené</span><span class="sxs-lookup"><span data-stu-id="4039d-112">Deferred</span></span>  
 <span data-ttu-id="4039d-113">Odložené spuštění znamená, že operace není provedena v místě v kódu, kde je dotaz deklarován.</span><span class="sxs-lookup"><span data-stu-id="4039d-113">Deferred execution means that the operation is not performed at the point in the code where the query is declared.</span></span> <span data-ttu-id="4039d-114">Operace se provádí pouze v případě, že je proměnná dotazu vyhodnocena, například pomocí `For Each` příkazu.</span><span class="sxs-lookup"><span data-stu-id="4039d-114">The operation is performed only when the query variable is enumerated, for example by using a `For Each` statement.</span></span> <span data-ttu-id="4039d-115">To znamená, že výsledky spuštění dotazu závisí na obsahu zdroje dat při spuštění dotazu, nikoli při definování dotazu.</span><span class="sxs-lookup"><span data-stu-id="4039d-115">This means that the results of executing the query depend on the contents of the data source when the query is executed rather than when the query is defined.</span></span> <span data-ttu-id="4039d-116">Pokud je proměnná dotazu vyčíslovaná víckrát, výsledky se můžou pokaždé lišit.</span><span class="sxs-lookup"><span data-stu-id="4039d-116">If the query variable is enumerated multiple times, the results might differ every time.</span></span> <span data-ttu-id="4039d-117">Skoro všechny standardní operátory dotazu, jejichž návratový typ je <xref:System.Collections.Generic.IEnumerable%601> nebo <xref:System.Linq.IOrderedEnumerable%601> se má provést odvoditelné způsobem.</span><span class="sxs-lookup"><span data-stu-id="4039d-117">Almost all the standard query operators whose return type is <xref:System.Collections.Generic.IEnumerable%601> or <xref:System.Linq.IOrderedEnumerable%601> execute in a deferred manner.</span></span>  
  
 <span data-ttu-id="4039d-118">Operátory dotazů, které používají odložené provádění, se dají klasifikovat taky jako streamování nebo nestreamování.</span><span class="sxs-lookup"><span data-stu-id="4039d-118">Query operators that use deferred execution can be additionally classified as streaming or non-streaming.</span></span>  
  
#### <a name="streaming"></a><span data-ttu-id="4039d-119">Streamování</span><span class="sxs-lookup"><span data-stu-id="4039d-119">Streaming</span></span>  
 <span data-ttu-id="4039d-120">Operátoři streamování nemusí číst všechna zdrojová data před tím, než budou vracet prvky.</span><span class="sxs-lookup"><span data-stu-id="4039d-120">Streaming operators do not have to read all the source data before they yield elements.</span></span> <span data-ttu-id="4039d-121">V době spuštění operátor streamování provede svoji operaci na každém zdrojovém prvku, protože je čten a v případě potřeby vytvoří prvek.</span><span class="sxs-lookup"><span data-stu-id="4039d-121">At the time of execution, a streaming operator performs its operation on each source element as it is read and yields the element if appropriate.</span></span> <span data-ttu-id="4039d-122">Operátor streamování bude nadále číst zdrojové prvky, dokud nebude možné vytvořit element výsledku.</span><span class="sxs-lookup"><span data-stu-id="4039d-122">A streaming operator continues to read source elements until a result element can be produced.</span></span> <span data-ttu-id="4039d-123">To znamená, že je možné načíst více než jeden zdrojový prvek pro získání jednoho prvku výsledku.</span><span class="sxs-lookup"><span data-stu-id="4039d-123">This means that more than one source element might be read to produce one result element.</span></span>  
  
#### <a name="non-streaming"></a><span data-ttu-id="4039d-124">Bez streamování</span><span class="sxs-lookup"><span data-stu-id="4039d-124">Non-Streaming</span></span>  
 <span data-ttu-id="4039d-125">Nestreamované operátory musí číst všechna zdrojová data předtím, než mohou získat výsledný element.</span><span class="sxs-lookup"><span data-stu-id="4039d-125">Non-streaming operators must read all the source data before they can yield a result element.</span></span> <span data-ttu-id="4039d-126">Do této kategorie patří operace, jako je řazení nebo seskupování.</span><span class="sxs-lookup"><span data-stu-id="4039d-126">Operations such as sorting or grouping fall into this category.</span></span> <span data-ttu-id="4039d-127">V době spuštění nestreamující operátory dotazů čtou všechna zdrojová data, umístí je do struktury dat, provede operaci a vrátí výsledné prvky.</span><span class="sxs-lookup"><span data-stu-id="4039d-127">At the time of execution, non-streaming query operators read all the source data, put it into a data structure, perform the operation, and yield the resulting elements.</span></span>
