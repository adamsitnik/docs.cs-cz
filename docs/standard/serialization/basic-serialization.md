# <a name="basic-serialization"></a><span data-ttu-id="df021-102">Základní serializace</span><span class="sxs-lookup"><span data-stu-id="df021-102">Basic serialization</span></span>

[!INCLUDE [binary-serialization-warning](../../../includes/binary-serialization-warning.md)]

<span data-ttu-id="df021-103">Nejjednodušší způsob, jak vytvořit třídu serializovatelný je označit pomocí <xref:System.SerializableAttribute> následujícím způsobem.</span><span class="sxs-lookup"><span data-stu-id="df021-103">The easiest way to make a class serializable is to mark it with the <xref:System.SerializableAttribute> as follows.</span></span>  
  
```csharp  
[Serializable]  
public class MyObject {  
  public int n1 = 0;  
  public int n2 = 0;  
  public String str = null;  
}  
```  
  
<span data-ttu-id="df021-104">Následující příklad kódu ukazuje, jak lze serializovat instance této třídy do souboru.</span><span class="sxs-lookup"><span data-stu-id="df021-104">The following code example shows how an instance of this class can be serialized to a file.</span></span>  
  
```csharp  
MyObject obj = new MyObject();  
obj.n1 = 1;  
obj.n2 = 24;  
obj.str = "Some String";  
IFormatter formatter = new BinaryFormatter();  
Stream stream = new FileStream("MyFile.bin", FileMode.Create, FileAccess.Write, FileShare.None);  
formatter.Serialize(stream, obj);  
stream.Close();  
```  
  
<span data-ttu-id="df021-105">V tomto příkladu binární formátovací modul použije k serializaci.</span><span class="sxs-lookup"><span data-stu-id="df021-105">This example uses a binary formatter to do the serialization.</span></span> <span data-ttu-id="df021-106">Všechno, co potřebujete udělat, je vytvořit instanci datového proudu a formátovací modul, který máte v úmyslu použít a poté zavolejte **serializace** metodu na formátování.</span><span class="sxs-lookup"><span data-stu-id="df021-106">All you need to do is create an instance of the stream and the formatter you intend to use, and then call the **Serialize** method on the formatter.</span></span> <span data-ttu-id="df021-107">Datový proud a objekt určený k serializaci jsou poskytovány jako parametry pro toto volání.</span><span class="sxs-lookup"><span data-stu-id="df021-107">The stream and the object to serialize are provided as parameters to this call.</span></span> <span data-ttu-id="df021-108">I když není ukázáno explicitně v tomto příkladu, bude serializována všechny proměnné členů třídy – i proměnné, které jsou označeny jako soukromé.</span><span class="sxs-lookup"><span data-stu-id="df021-108">Although it is not explicitly demonstrated in this example, all member variables of a class will be serialized—even variables marked as private.</span></span> <span data-ttu-id="df021-109">V této aspekt binární serializace se liší od <xref:System.Xml.Serialization.XmlSerializer> třídu, která serializuje pouze veřejná pole.</span><span class="sxs-lookup"><span data-stu-id="df021-109">In this aspect, binary serialization differs from the <xref:System.Xml.Serialization.XmlSerializer> class, which only serializes public fields.</span></span> <span data-ttu-id="df021-110">Informace o vyloučení proměnné členů z binární serializace naleznete v tématu [selektivní serializace](selective-serialization.md).</span><span class="sxs-lookup"><span data-stu-id="df021-110">For information on excluding member variables from binary serialization, see [Selective Serialization](selective-serialization.md).</span></span>  
  
<span data-ttu-id="df021-111">Obnovení objektu zpět do původní stavu je stejně jednoduché.</span><span class="sxs-lookup"><span data-stu-id="df021-111">Restoring the object back to its former state is just as easy.</span></span> <span data-ttu-id="df021-112">Nejprve vytvořte datového proudu pro čtení a <xref:System.Runtime.Serialization.Formatter>, a potom pokyn formátovací modul k deserializaci objektu.</span><span class="sxs-lookup"><span data-stu-id="df021-112">First, create a stream for reading and a <xref:System.Runtime.Serialization.Formatter>, and then instruct the formatter to deserialize the object.</span></span> <span data-ttu-id="df021-113">Následující příklad kódu ukazuje, jak to lze provést.</span><span class="sxs-lookup"><span data-stu-id="df021-113">The code example below shows how this is done.</span></span>  
  
```csharp  
IFormatter formatter = new BinaryFormatter();  
Stream stream = new FileStream("MyFile.bin", FileMode.Open, FileAccess.Read, FileShare.Read);  
MyObject obj = (MyObject) formatter.Deserialize(stream);  
stream.Close();  
  
// Here's the proof.  
Console.WriteLine("n1: {0}", obj.n1);  
Console.WriteLine("n2: {0}", obj.n2);  
Console.WriteLine("str: {0}", obj.str);  
```  
  
<span data-ttu-id="df021-114"><xref:System.Runtime.Serialization.Formatters.Binary.BinaryFormatter> Používá výše, je velmi efektivní a vytváří tok compact bajtů.</span><span class="sxs-lookup"><span data-stu-id="df021-114">The <xref:System.Runtime.Serialization.Formatters.Binary.BinaryFormatter> used above is very efficient and produces a compact byte stream.</span></span> <span data-ttu-id="df021-115">Všechny objekty serializované pomocí tohoto formátování lze deserializovat také s ním, díky čemuž je ideální nástroj pro serializaci objektů, které k deserializaci na rozhraní .NET Framework.</span><span class="sxs-lookup"><span data-stu-id="df021-115">All objects serialized with this formatter can also be deserialized with it, which makes it an ideal tool for serializing objects that will be deserialized on the .NET Framework.</span></span> <span data-ttu-id="df021-116">Je důležité si všimnout, že konstruktory nejsou volána, když je objekt deserializován.</span><span class="sxs-lookup"><span data-stu-id="df021-116">It is important to note that constructors are not called when an object is deserialized.</span></span> <span data-ttu-id="df021-117">Toto omezení je umístěn na deserializace z hlediska výkonu.</span><span class="sxs-lookup"><span data-stu-id="df021-117">This constraint is placed on deserialization for performance reasons.</span></span> <span data-ttu-id="df021-118">Však to porušuje některé obvyklé smluv, které modul runtime provede pomocí objektu zapisovače a vývojáři se ujistěte, že znají důsledky při označení objektu jako serializovatelný.</span><span class="sxs-lookup"><span data-stu-id="df021-118">However, this violates some of the usual contracts the runtime makes with the object writer, and developers should ensure that they understand the ramifications when marking an object as serializable.</span></span>  
  
<span data-ttu-id="df021-119">Je-li přenositelnost je vyžadováno, použijte <xref:System.Runtime.Serialization.Formatters.Soap.SoapFormatter> místo.</span><span class="sxs-lookup"><span data-stu-id="df021-119">If portability is a requirement, use the <xref:System.Runtime.Serialization.Formatters.Soap.SoapFormatter> instead.</span></span> <span data-ttu-id="df021-120">Jednoduše nahradit **BinaryFormatter** v kódu výše s **SoapFormatter,** a volat **serializace** a **Deserialize** stejně jako předtím.</span><span class="sxs-lookup"><span data-stu-id="df021-120">Simply replace the **BinaryFormatter** in the code above with **SoapFormatter,** and call **Serialize** and **Deserialize** as before.</span></span> <span data-ttu-id="df021-121">Tento formátovací modul vytváří následující výstup například využité nad.</span><span class="sxs-lookup"><span data-stu-id="df021-121">This formatter produces the following output for the example used above.</span></span>

```xml  
      <str id="ref-3">Some String</str>  
    </a1:MyObject>  
  </SOAP-ENV:Body>  
</SOAP-ENV:Envelope>  
```  
  
<span data-ttu-id="df021-122">Je důležité si uvědomit, že [Serializable](xref:System.SerializableAttribute) atribut nemůže být odvozeny.</span><span class="sxs-lookup"><span data-stu-id="df021-122">It's important to note that the [Serializable](xref:System.SerializableAttribute) attribute cannot be inherited.</span></span> <span data-ttu-id="df021-123">Je-li odvodit novou třídu z `MyObject`, musí být označen nové třídy s atributem také nebo nemůže být serializován.</span><span class="sxs-lookup"><span data-stu-id="df021-123">If you derive a new class from `MyObject`, the new class must be marked with the attribute as well, or it cannot be serialized.</span></span> <span data-ttu-id="df021-124">Například při pokusu o serializaci instancí třídy níže, získáte <xref:System.Runtime.Serialization.SerializationException> informací, které `MyStuff` typ není označen jako serializovatelný.</span><span class="sxs-lookup"><span data-stu-id="df021-124">For example, when you attempt to serialize an instance of the class below, you'll get a <xref:System.Runtime.Serialization.SerializationException> informing you that the `MyStuff` type is not marked as serializable.</span></span>

```csharp  
  public int n3;
```  
