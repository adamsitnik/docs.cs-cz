-   <span data-ttu-id="72d03-122">Vyhněte se úniku obou vzor založený na událostech a <xref:System.IAsyncResult> vzor ve stejné třídě.</span><span class="sxs-lookup"><span data-stu-id="72d03-122">Avoid exposing both the event-based pattern and the <xref:System.IAsyncResult> pattern on the same class.</span></span> <span data-ttu-id="72d03-123">Vystavení vzor založený na událostech pro "na vyšší úrovni" třídy a <xref:System.IAsyncResult> vzor na "snížit úroveň" třídy.</span><span class="sxs-lookup"><span data-stu-id="72d03-123">Expose the event-based pattern on "higher level" classes and the <xref:System.IAsyncResult> pattern on "lower level" classes.</span></span> <span data-ttu-id="72d03-124">Například porovnat na model založený na událostech <xref:System.Net.WebClient> komponentu s <xref:System.IAsyncResult> vzor na <xref:System.Web.HttpRequest> třídy.</span><span class="sxs-lookup"><span data-stu-id="72d03-124">For example, compare the event-based pattern on the <xref:System.Net.WebClient> component with the <xref:System.IAsyncResult> pattern on the <xref:System.Web.HttpRequest> class.</span></span>  
  
    -   <span data-ttu-id="72d03-125">Vystavení vzor založený na událostech a <xref:System.IAsyncResult> vzor na stejné třídy, pokud to vyžaduje kompatibility.</span><span class="sxs-lookup"><span data-stu-id="72d03-125">Expose the event-based pattern and the <xref:System.IAsyncResult> pattern on the same class when compatibility requires it.</span></span> <span data-ttu-id="72d03-126">Například, pokud jste už vydali rozhraní API, které používá <xref:System.IAsyncResult> vzor, je třeba zachovat <xref:System.IAsyncResult> vzoru z důvodu zpětné kompatibility.</span><span class="sxs-lookup"><span data-stu-id="72d03-126">For example, if you have already released an API that uses the <xref:System.IAsyncResult> pattern, you would need to retain the <xref:System.IAsyncResult> pattern for backward compatibility.</span></span>  
  
    -   <span data-ttu-id="72d03-127">Vystavení vzor založený na událostech a <xref:System.IAsyncResult> vzorku ve stejné třídě, pokud výsledný složitost modelu objektu převažuje získané výhody oddělení implementace.</span><span class="sxs-lookup"><span data-stu-id="72d03-127">Expose the event-based pattern and the <xref:System.IAsyncResult> pattern on the same class if the resulting object model complexity outweighs the benefit of separating the implementations.</span></span> <span data-ttu-id="72d03-128">Je lepší vystavit obou vzory na jednu třídu než zabráníte tak vystavení vzor založený na událostech.</span><span class="sxs-lookup"><span data-stu-id="72d03-128">It is better to expose both patterns on a single class than to avoid exposing the event-based pattern.</span></span>

## <a name="criteria-for-exposing-the-iasyncresult-pattern-in-addition-to-the-event-based-pattern"></a><span data-ttu-id="72d03-132">Kritéria pro zveřejnění vzor IAsyncResult kromě vzor založený na událostech</span><span class="sxs-lookup"><span data-stu-id="72d03-132">Criteria for Exposing the IAsyncResult Pattern in Addition to the Event-based Pattern</span></span>  
 <span data-ttu-id="72d03-133">Během asynchronního vzoru založeného na událostech má mnoho výhod podle výše uvedených scénářů, má některé nevýhody, které byste měli vědět, pokud je výkon vašich nejdůležitějších požadavek.</span><span class="sxs-lookup"><span data-stu-id="72d03-133">While the Event-based Asynchronous Pattern has many benefits under the previously mentioned scenarios, it does have some drawbacks, which you should be aware of if performance is your most important requirement.</span></span>  
  
 <span data-ttu-id="72d03-134">Existují tři scénáře, které model založený na událostech neřeší i na <xref:System.IAsyncResult> vzoru:</span><span class="sxs-lookup"><span data-stu-id="72d03-134">There are three scenarios that the event-based pattern does not address as well as the <xref:System.IAsyncResult> pattern:</span></span>  
  
-   <span data-ttu-id="72d03-135">Blokovat čekání na jednom <xref:System.IAsyncResult></span><span class="sxs-lookup"><span data-stu-id="72d03-135">Blocking wait on one <xref:System.IAsyncResult></span></span>  
  
-   <span data-ttu-id="72d03-136">Blokovat čekání na mnoha <xref:System.IAsyncResult> objekty</span><span class="sxs-lookup"><span data-stu-id="72d03-136">Blocking wait on many <xref:System.IAsyncResult> objects</span></span>  
  
-   <span data-ttu-id="72d03-137">Dotazovat se na dokončení <xref:System.IAsyncResult></span><span class="sxs-lookup"><span data-stu-id="72d03-137">Polling for completion on the <xref:System.IAsyncResult></span></span>  
  
 <span data-ttu-id="72d03-138">Tyto scénáře můžete řešit pomocí vzoru založeného na událostech, ale to je náročnější než při použití <xref:System.IAsyncResult> vzor.</span><span class="sxs-lookup"><span data-stu-id="72d03-138">You can address these scenarios by using the event-based pattern, but doing so is more cumbersome than using the <xref:System.IAsyncResult> pattern.</span></span>  
  
 <span data-ttu-id="72d03-139">Vývojáři často používají <xref:System.IAsyncResult> modelu pro služby, které mají obvykle velmi vysoký výkon požadavky.</span><span class="sxs-lookup"><span data-stu-id="72d03-139">Developers often use the <xref:System.IAsyncResult> pattern for services that typically have very high performance requirements.</span></span> <span data-ttu-id="72d03-140">Například dotazování pro dokončení scénář je technika výkonné serveru.</span><span class="sxs-lookup"><span data-stu-id="72d03-140">For example, the polling for completion scenario is a high-performance server technique.</span></span>  
  
 <span data-ttu-id="72d03-141">Kromě toho je méně efektivní než vzor založený na událostech <xref:System.IAsyncResult> vzorek, protože vytváří více objektů, zejména <xref:System.EventArgs>, a protože synchronizuje přes více vláken.</span><span class="sxs-lookup"><span data-stu-id="72d03-141">Additionally, the event-based pattern is less efficient than the <xref:System.IAsyncResult> pattern because it creates more objects, especially <xref:System.EventArgs>, and because it synchronizes across threads.</span></span>  
  
 <span data-ttu-id="72d03-142">Následující seznam uvádí několik doporučení, které řídí, pokud se rozhodnete použít <xref:System.IAsyncResult> vzoru:</span><span class="sxs-lookup"><span data-stu-id="72d03-142">The following list shows some recommendations to follow if you decide to use the <xref:System.IAsyncResult> pattern:</span></span>  
  
-   <span data-ttu-id="72d03-143">Pouze vystavit <xref:System.IAsyncResult> vzor, pokud výslovně nepotřebujete podporu <xref:System.Threading.WaitHandle> nebo <xref:System.IAsyncResult> objekty.</span><span class="sxs-lookup"><span data-stu-id="72d03-143">Only expose the <xref:System.IAsyncResult> pattern when you specifically require support for <xref:System.Threading.WaitHandle> or <xref:System.IAsyncResult> objects.</span></span>  
  
-   <span data-ttu-id="72d03-144">Pouze vystavit <xref:System.IAsyncResult> vzor, pokud máte existující rozhraní API, které používá <xref:System.IAsyncResult> vzor.</span><span class="sxs-lookup"><span data-stu-id="72d03-144">Only expose the <xref:System.IAsyncResult> pattern when you have an existing API that uses the <xref:System.IAsyncResult> pattern.</span></span>  
  
-   <span data-ttu-id="72d03-145">Pokud máte existující rozhraní API na základě <xref:System.IAsyncResult> vzorku, zvažte také zveřejnění vzor založený na událostech v další vydané verzi.</span><span class="sxs-lookup"><span data-stu-id="72d03-145">If you have an existing API based on the <xref:System.IAsyncResult> pattern, consider also exposing the event-based pattern in your next release.</span></span>  
  
-   <span data-ttu-id="72d03-146">Pouze vystavit <xref:System.IAsyncResult> vzor, pokud mají požadavky na vysoký výkon, které si ověříte nelze splnit třídou vzor založený na událostech, ale mohou být splněny <xref:System.IAsyncResult> vzor.</span><span class="sxs-lookup"><span data-stu-id="72d03-146">Only expose <xref:System.IAsyncResult> pattern if you have high performance requirements which you have verified cannot be met by the event-based pattern but can be met by the <xref:System.IAsyncResult> pattern.</span></span>
