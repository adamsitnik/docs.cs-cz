# <a name="weak-references"></a><span data-ttu-id="cfc6a-102">Slabé odkazy</span><span class="sxs-lookup"><span data-stu-id="cfc6a-102">Weak References</span></span>
<span data-ttu-id="cfc6a-103">Systém uvolňování paměti nemůže shromáždit objekt, který je používán aplikací, zatímco kód aplikace může dosáhnout tohoto objektu.</span><span class="sxs-lookup"><span data-stu-id="cfc6a-103">The garbage collector cannot collect an object in use by an application while the application's code can reach that object.</span></span> <span data-ttu-id="cfc6a-104">Aplikace je označována jako, že má silný odkaz na objekt.</span><span class="sxs-lookup"><span data-stu-id="cfc6a-104">The application is said to have a strong reference to the object.</span></span>  
  
 <span data-ttu-id="cfc6a-105">Slabý odkaz umožňuje, aby systém uvolňování paměti shromáždil objekt, zatímco aplikace stále umožňuje přístup k objektu.</span><span class="sxs-lookup"><span data-stu-id="cfc6a-105">A weak reference permits the garbage collector to collect the object while still allowing the application to access the object.</span></span> <span data-ttu-id="cfc6a-106">Slabý odkaz je platný pouze během neurčitého množství času, dokud není objekt shromážděn, když neexistují žádné silné odkazy.</span><span class="sxs-lookup"><span data-stu-id="cfc6a-106">A weak reference is valid only during the indeterminate amount of time until the object is collected when no strong references exist.</span></span> <span data-ttu-id="cfc6a-107">Když použijete slabý odkaz, aplikace může stále získat silný odkaz na objekt, který brání jeho shromažďování.</span><span class="sxs-lookup"><span data-stu-id="cfc6a-107">When you use a weak reference, the application can still obtain a strong reference to the object, which prevents it from being collected.</span></span> <span data-ttu-id="cfc6a-108">Nicméně vždy hrozí riziko, že systém uvolňování paměti získá nejprve před tím, než se silný odkaz znovu vytvoří.</span><span class="sxs-lookup"><span data-stu-id="cfc6a-108">However, there is always the risk that the garbage collector will get to the object first before a strong reference is reestablished.</span></span>  
  
 <span data-ttu-id="cfc6a-109">Slabé odkazy jsou užitečné pro objekty, které využívají hodně paměti, ale je možné je znovu vytvořit, pokud jsou uvolněny uvolňováním paměti.</span><span class="sxs-lookup"><span data-stu-id="cfc6a-109">Weak references are useful for objects that use a lot of memory, but can be recreated easily if they are reclaimed by garbage collection.</span></span>  
  
 <span data-ttu-id="cfc6a-110">Předpokládejme, že se stromové zobrazení v aplikaci model Windows Forms zobrazí uživateli komplexní hierarchická volba možností.</span><span class="sxs-lookup"><span data-stu-id="cfc6a-110">Suppose a tree view in a Windows Forms application displays a complex hierarchical choice of options to the user.</span></span> <span data-ttu-id="cfc6a-111">Pokud je podkladová data velká, udržování stromu paměti je neefektivní, pokud je uživatel zapojen do aplikace v nějakém jiném.</span><span class="sxs-lookup"><span data-stu-id="cfc6a-111">If the underlying data is large, keeping the tree in memory is inefficient when the user is involved with something else in the application.</span></span>  
  
 <span data-ttu-id="cfc6a-112">Když uživatel přepne jinam do jiné části aplikace, můžete použít třídu <xref:System.WeakReference> k vytvoření slabého odkazu na strom a zničení všech silných odkazů.</span><span class="sxs-lookup"><span data-stu-id="cfc6a-112">When the user switches away to another part of the application, you can use the <xref:System.WeakReference> class to create a weak reference to the tree and destroy all strong references.</span></span> <span data-ttu-id="cfc6a-113">Když uživatel přepne zpět do stromu, aplikace se pokusí získat silný odkaz na strom a v případě úspěchu se vyhne opětovné konstrukci stromu.</span><span class="sxs-lookup"><span data-stu-id="cfc6a-113">When the user switches back to the tree, the application attempts to obtain a strong reference to the tree and, if successful, avoids reconstructing the tree.</span></span>  
  
 <span data-ttu-id="cfc6a-114">Chcete-li vytvořit slabý odkaz na objekt, vytvoříte <xref:System.WeakReference> pomocí instance objektu, který chcete sledovat.</span><span class="sxs-lookup"><span data-stu-id="cfc6a-114">To establish a weak reference with an object, you create a <xref:System.WeakReference> using the instance of the object to be tracked.</span></span> <span data-ttu-id="cfc6a-115">Potom nastavíte vlastnost <xref:System.WeakReference.Target%2A> na tento objekt a nastavíte původní odkaz na objekt, který se má `null`.</span><span class="sxs-lookup"><span data-stu-id="cfc6a-115">You then set the <xref:System.WeakReference.Target%2A> property to that object and set the original reference to the object to `null`.</span></span> <span data-ttu-id="cfc6a-116">Příklad kódu naleznete v tématu <xref:System.WeakReference> v knihovně tříd.</span><span class="sxs-lookup"><span data-stu-id="cfc6a-116">For a code example, see <xref:System.WeakReference> in the class library.</span></span>  
  
## <a name="short-and-long-weak-references"></a><span data-ttu-id="cfc6a-117">Krátké a dlouhé slabé odkazy</span><span class="sxs-lookup"><span data-stu-id="cfc6a-117">Short and Long Weak References</span></span>  
 <span data-ttu-id="cfc6a-118">Můžete vytvořit krátký slabý odkaz nebo dlouhý slabý odkaz:</span><span class="sxs-lookup"><span data-stu-id="cfc6a-118">You can create a short weak reference or a long weak reference:</span></span>  
  
- <span data-ttu-id="cfc6a-119">Dostatečná</span><span class="sxs-lookup"><span data-stu-id="cfc6a-119">Short</span></span>  
  
     <span data-ttu-id="cfc6a-120">Cíl krátkého slabého odkazu se bude `null`, když se objekt uvolní uvolňováním paměti.</span><span class="sxs-lookup"><span data-stu-id="cfc6a-120">The target of a short weak reference becomes `null` when the object is reclaimed by garbage collection.</span></span> <span data-ttu-id="cfc6a-121">Slabý odkaz je sám o sobě spravovaným objektem a podléhá uvolňování paměti stejně jako jakýkoli jiný spravovaný objekt.</span><span class="sxs-lookup"><span data-stu-id="cfc6a-121">The weak reference is itself a managed object, and is subject to garbage collection just like any other managed object.</span></span>  <span data-ttu-id="cfc6a-122">Krátký slabý odkaz je konstruktor bez parametrů pro <xref:System.WeakReference>.</span><span class="sxs-lookup"><span data-stu-id="cfc6a-122">A short weak reference is the parameterless constructor for <xref:System.WeakReference>.</span></span>  
  
- <span data-ttu-id="cfc6a-123">Dlouhou</span><span class="sxs-lookup"><span data-stu-id="cfc6a-123">Long</span></span>  
  
     <span data-ttu-id="cfc6a-124">Po volání metody <xref:System.Object.Finalize%2A> objektu se zachová dlouhý slabý odkaz.</span><span class="sxs-lookup"><span data-stu-id="cfc6a-124">A long weak reference is retained after the object's <xref:System.Object.Finalize%2A> method has been called.</span></span> <span data-ttu-id="cfc6a-125">To umožňuje, aby byl objekt znovu vytvořen, ale stav objektu zůstane nepředvídatelné.</span><span class="sxs-lookup"><span data-stu-id="cfc6a-125">This allows the object to be recreated, but the state of the object remains unpredictable.</span></span> <span data-ttu-id="cfc6a-126">Chcete-li použít dlouhý odkaz, zadejte `true` v konstruktoru <xref:System.WeakReference>.</span><span class="sxs-lookup"><span data-stu-id="cfc6a-126">To use a long reference, specify `true` in the <xref:System.WeakReference> constructor.</span></span>  
  
     <span data-ttu-id="cfc6a-127">Pokud typ objektu nemá metodu <xref:System.Object.Finalize%2A>, je použita krátká slabá odkazová funkce a slabý odkaz je platný pouze do okamžiku, kdy je cíl shromážděn, což může nastat kdykoli po spuštění finalizační metody.</span><span class="sxs-lookup"><span data-stu-id="cfc6a-127">If the object's type does not have a <xref:System.Object.Finalize%2A> method, the short weak reference functionality applies and the weak reference is valid only until the target is collected, which can occur anytime after the finalizer is run.</span></span>  
  
 <span data-ttu-id="cfc6a-128">Chcete-li vytvořit silný odkaz a znovu použít objekt, přetypujte vlastnost <xref:System.WeakReference.Target%2A> <xref:System.WeakReference> na typ objektu.</span><span class="sxs-lookup"><span data-stu-id="cfc6a-128">To establish a strong reference and use the object again, cast the <xref:System.WeakReference.Target%2A> property of a <xref:System.WeakReference> to the type of the object.</span></span> <span data-ttu-id="cfc6a-129">Pokud vlastnost <xref:System.WeakReference.Target%2A> vrátí `null`, objekt byl shromážděn; v opačném případě můžete objekt nadále používat, protože aplikace znovu získala silný odkaz na něj.</span><span class="sxs-lookup"><span data-stu-id="cfc6a-129">If the <xref:System.WeakReference.Target%2A> property returns `null`, the object was collected; otherwise, you can continue to use the object because the application has regained a strong reference to it.</span></span>
