<span data-ttu-id="b94e4-103">.NET Framework poskytuje běhové prostředí označované jako modul CLR (Common Language Runtime), který spouští kód a poskytuje služby, které usnadňují proces vývoje.</span><span class="sxs-lookup"><span data-stu-id="b94e4-103">The .NET Framework provides a run-time environment called the common language runtime, which runs the code and provides services that make the development process easier.</span></span>

<span data-ttu-id="b94e4-104">Kompilátory a nástroje zpřístupňují funkce modulu CLR (Common Language Runtime) a umožňují psát kód, který je výhodou z tohoto spravovaného prováděcího prostředí.</span><span class="sxs-lookup"><span data-stu-id="b94e4-104">Compilers and tools expose the common language runtime's functionality and enable you to write code that benefits from this managed execution environment.</span></span> <span data-ttu-id="b94e4-105">Kód, který vyvíjíte pomocí kompilátoru jazyka, který cílí na modul runtime, se nazývá spravovaný kód; využívá výhody funkcí, jako je například integrace mezi jazyky, zpracování výjimek v různých jazycích, vylepšené zabezpečení, Správa verzí a podpora nasazení, zjednodušený model pro interakci s komponentami a ladění a profilování služeb.</span><span class="sxs-lookup"><span data-stu-id="b94e4-105">Code that you develop with a language compiler that targets the runtime is called managed code; it benefits from features such as cross-language integration, cross-language exception handling, enhanced security, versioning and deployment support, a simplified model for component interaction, and debugging and profiling services.</span></span>

> [!NOTE]
> <span data-ttu-id="b94e4-106">Kompilátory a nástroje jsou schopny vytvářet výstupy, které může modul CLR (Common Language Runtime) spotřebovat, protože systém typů, formát metadat a běhové prostředí (virtuální spouštěcí systém) jsou definovány veřejným standardem, společný jazyk ECMA Specifikace infrastruktury</span><span class="sxs-lookup"><span data-stu-id="b94e4-106">Compilers and tools are able to produce output that the common language runtime can consume because the type system, the format of metadata, and the runtime environment (the virtual execution system) are all defined by a public standard, the ECMA Common Language Infrastructure specification.</span></span> <span data-ttu-id="b94e4-107">Další informace naleznete v tématu [specifikace C# ECMA a Common Language Infrastructure](https://visualstudio.microsoft.com/license-terms/ecma-c-common-language-infrastructure-standards/).</span><span class="sxs-lookup"><span data-stu-id="b94e4-107">For more information, see [ECMA C# and Common Language Infrastructure Specifications](https://visualstudio.microsoft.com/license-terms/ecma-c-common-language-infrastructure-standards/).</span></span>

<span data-ttu-id="b94e4-121">V rámci svých metadat všechny spravované součásti přenášejí informace o komponentách a prostředcích, na které byly vytvořeny.</span><span class="sxs-lookup"><span data-stu-id="b94e4-121">As part of their metadata, all managed components carry information about the components and resources they were built against.</span></span> <span data-ttu-id="b94e4-122">Modul runtime používá tyto informace k zajištění toho, že vaše komponenta nebo aplikace má specifikované verze všeho, co IT potřebuje, což způsobí, že váš kód bude méně pravděpodobný v důsledku nesplnění závislosti.</span><span class="sxs-lookup"><span data-stu-id="b94e4-122">The runtime uses this information to ensure that your component or application has the specified versions of everything it needs, which makes your code less likely to break because of some unmet dependency.</span></span> <span data-ttu-id="b94e4-123">Informace o registraci a stavová data se už neukládají v registru, kde se můžou obtížně nakládat a udržovat.</span><span class="sxs-lookup"><span data-stu-id="b94e4-123">Registration information and state data are no longer stored in the registry where they can be difficult to establish and maintain.</span></span> <span data-ttu-id="b94e4-124">Místo toho jsou informace o typech, které definujete (a jejich závislosti), uloženy spolu s kódem jako metadata, takže úkoly replikace komponent a odebírání jsou mnohem méně komplikované.</span><span class="sxs-lookup"><span data-stu-id="b94e4-124">Instead, information about the types you define (and their dependencies) is stored with the code as metadata, making the tasks of component replication and removal much less complicated.</span></span>
