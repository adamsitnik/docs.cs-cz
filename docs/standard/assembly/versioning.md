## <a name="version-information"></a><span data-ttu-id="9792e-116">Informace o verzi</span><span class="sxs-lookup"><span data-stu-id="9792e-116">Version information</span></span>

### <a name="assembly-version-number"></a><span data-ttu-id="9792e-121">Číslo verze sestavení</span><span class="sxs-lookup"><span data-stu-id="9792e-121">Assembly version number</span></span>  
 <span data-ttu-id="9792e-122">Každé sestavení má v rámci své identity číslo verze.</span><span class="sxs-lookup"><span data-stu-id="9792e-122">Each assembly has a version number as part of its identity.</span></span> <span data-ttu-id="9792e-123">V takovém případě se dvě sestavení, která se liší číslem verze, považují za modul runtime za zcela odlišná sestavení.</span><span class="sxs-lookup"><span data-stu-id="9792e-123">As such, two assemblies that differ by version number are considered by the runtime to be completely different assemblies.</span></span> <span data-ttu-id="9792e-124">Toto číslo verze je fyzicky reprezentované jako řetězec o čtyřech částech v následujícím formátu:</span><span class="sxs-lookup"><span data-stu-id="9792e-124">This version number is physically represented as a four-part string with the following format:</span></span>

 <span data-ttu-id="9792e-126">Například verze 1.5.1254.0 označuje 1 jako hlavní verzi, 5 jako dílčí verzi, 1254 jako číslo sestavení a 0 jako číslo revize.</span><span class="sxs-lookup"><span data-stu-id="9792e-126">For example, version 1.5.1254.0 indicates 1 as the major version, 5 as the minor version, 1254 as the build number, and 0 as the revision number.</span></span>  
  
 <span data-ttu-id="9792e-127">Číslo verze je uloženo v manifestu sestavení spolu s dalšími informacemi o identitě, včetně názvu sestavení a veřejného klíče, a také informace o relacích a identitách jiných sestavení připojených k aplikaci.</span><span class="sxs-lookup"><span data-stu-id="9792e-127">The version number is stored in the assembly manifest along with other identity information, including the assembly name and public key, as well as information on relationships and identities of other assemblies connected with the application.</span></span>  
  
 <span data-ttu-id="9792e-128">Při sestavování sestavení Nástroj pro vývoj zaznamenává informace o závislostech pro každé sestavení, na které je odkazováno v manifestu sestavení.</span><span class="sxs-lookup"><span data-stu-id="9792e-128">When an assembly is built, the development tool records dependency information for each assembly that is referenced in the assembly manifest.</span></span> <span data-ttu-id="9792e-129">Modul runtime používá tato čísla verzí ve spojení s konfiguračními informacemi nastavenými správcem, aplikací nebo vydavatelem k načtení správné verze odkazovaného sestavení.</span><span class="sxs-lookup"><span data-stu-id="9792e-129">The runtime uses these version numbers, in conjunction with configuration information set by an administrator, an application, or a publisher, to load the proper version of a referenced assembly.</span></span>
