# <a name="assembly-names"></a><span data-ttu-id="37f7e-102">Názvy sestavení</span><span class="sxs-lookup"><span data-stu-id="37f7e-102">Assembly names</span></span>
<span data-ttu-id="37f7e-103">Název sestavení je uložen v metadatech a má významný dopad na rozsah sestavení a použití aplikací.</span><span class="sxs-lookup"><span data-stu-id="37f7e-103">An assembly's name is stored in metadata and has a significant impact on the assembly's scope and use by an application.</span></span> <span data-ttu-id="37f7e-104">Sestavení se silným názvem má plně kvalifikovaný název, který obsahuje název, jazykovou verzi, veřejný klíč a číslo verze sestavení.</span><span class="sxs-lookup"><span data-stu-id="37f7e-104">A strong-named assembly has a fully qualified name that includes the assembly's name, culture, public key, and version number.</span></span> <span data-ttu-id="37f7e-105">Tato možnost je často označována jako zobrazované jméno a pro načtená sestavení lze získat pomocí vlastnosti <xref:System.Reflection.Assembly.FullName%2A>.</span><span class="sxs-lookup"><span data-stu-id="37f7e-105">This is frequently referred to as the display name, and for loaded assemblies can be obtained by using the <xref:System.Reflection.Assembly.FullName%2A> property.</span></span>  
  
 <span data-ttu-id="37f7e-106">Modul runtime používá tyto informace k vyhledání sestavení a jeho odlišení od ostatních sestavení se stejným názvem.</span><span class="sxs-lookup"><span data-stu-id="37f7e-106">The runtime uses this information to locate the assembly and differentiate it from other assemblies with the same name.</span></span> <span data-ttu-id="37f7e-107">Například sestavení se silným názvem s názvem `myTypes` může mít následující plně kvalifikovaný název:</span><span class="sxs-lookup"><span data-stu-id="37f7e-107">For example, a strong-named assembly called `myTypes` could have the following fully qualified name:</span></span>  
  
```  
myTypes, Version=1.0.1234.0, Culture=en-US, PublicKeyToken=b77a5c561934e089c, ProcessorArchitecture=msil  
```  
  
> [!NOTE]
> <span data-ttu-id="37f7e-108">Do identity sestavení v .NET Framework verze 2,0 se přidá architektura procesoru, aby se povolily verze sestavení specifické pro procesor.</span><span class="sxs-lookup"><span data-stu-id="37f7e-108">Processor architecture is added to the assembly identity in the .NET Framework version 2.0, to allow processor-specific versions of assemblies.</span></span> <span data-ttu-id="37f7e-109">Můžete vytvořit verze sestavení, jehož identita se liší pouze architekturou procesoru, například 32-bit a 64-bit specifických verzí procesoru.</span><span class="sxs-lookup"><span data-stu-id="37f7e-109">You can create versions of an assembly whose identity differs only by processor architecture, for example 32-bit and 64-bit processor-specific versions.</span></span> <span data-ttu-id="37f7e-110">Architektura procesoru není pro silné názvy nutná.</span><span class="sxs-lookup"><span data-stu-id="37f7e-110">Processor architecture is not required for strong names.</span></span> <span data-ttu-id="37f7e-111">Další informace najdete v tématu <xref:System.Reflection.AssemblyName.ProcessorArchitecture%2A?displayProperty=nameWithType>.</span><span class="sxs-lookup"><span data-stu-id="37f7e-111">For more information, see <xref:System.Reflection.AssemblyName.ProcessorArchitecture%2A?displayProperty=nameWithType>.</span></span>  
  
 <span data-ttu-id="37f7e-112">V tomto příkladu plně kvalifikovaný název označuje, že `myTypes` sestavení má silný název s tokenem veřejného klíče, má hodnotu jazykové verze pro jazyk US English a má číslo verze 1.0.1234.0.</span><span class="sxs-lookup"><span data-stu-id="37f7e-112">In this example, the fully qualified name indicates that the `myTypes` assembly has a strong name with a public key token, has the culture value for US English, and has a version number of 1.0.1234.0.</span></span> <span data-ttu-id="37f7e-113">Jeho architektura procesoru je "MSIL", což znamená, že bude za běhu (JIT) zkompilován do 32 bitového kódu nebo 64 kódu v závislosti na operačním systému a procesoru.</span><span class="sxs-lookup"><span data-stu-id="37f7e-113">Its processor architecture is "msil", which means that it will be just-in-time (JIT)-compiled to 32-bit code or 64-bit code depending on the operating system and processor.</span></span>  
  
 <span data-ttu-id="37f7e-114">Kód, který požaduje typy v sestavení, musí používat plně kvalifikovaný název sestavení.</span><span class="sxs-lookup"><span data-stu-id="37f7e-114">Code that requests types in an assembly must use a fully qualified assembly name.</span></span> <span data-ttu-id="37f7e-115">Tato metoda se nazývá plně kvalifikovaná vazba.</span><span class="sxs-lookup"><span data-stu-id="37f7e-115">This is called fully qualified binding.</span></span> <span data-ttu-id="37f7e-116">Částečná vazba, která určuje pouze název sestavení, není povolena při odkazování na sestavení v .NET Framework.</span><span class="sxs-lookup"><span data-stu-id="37f7e-116">Partial binding, which specifies only an assembly name, is not permitted when referencing assemblies in the .NET Framework.</span></span>  
  
 <span data-ttu-id="37f7e-117">Všechny odkazy na sestavení, které tvoří .NET Framework, musí také obsahovat plně kvalifikovaný název sestavení.</span><span class="sxs-lookup"><span data-stu-id="37f7e-117">All assembly references to assemblies that make up the .NET Framework must also contain the fully qualified name of the assembly.</span></span> <span data-ttu-id="37f7e-118">Například odkaz na sestavení System. data .NET Framework pro verzi 1,0 by zahrnoval:</span><span class="sxs-lookup"><span data-stu-id="37f7e-118">For example, a reference to the System.Data .NET Framework assembly for version 1.0 would include:</span></span>  
  
```  
System.data, version=1.0.3300.0, Culture=neutral, PublicKeyToken=b77a5c561934e089  
```  
  
 <span data-ttu-id="37f7e-119">Všimněte si, že verze odpovídá číslu verze všech .NET Framework sestavení, která byla dodávána s .NET Framework verze 1,0.</span><span class="sxs-lookup"><span data-stu-id="37f7e-119">Note that the version corresponds to the version number of all .NET Framework assemblies that shipped with .NET Framework version 1.0.</span></span> <span data-ttu-id="37f7e-120">Pro .NET Framework sestavení je hodnota jazykové verze vždycky neutrální a veřejný klíč je stejný, jak je uvedeno v předchozím příkladu.</span><span class="sxs-lookup"><span data-stu-id="37f7e-120">For .NET Framework assemblies, the culture value is always neutral, and the public key is the same as shown in the above example.</span></span>  
  
 <span data-ttu-id="37f7e-121">Chcete-li například přidat odkaz na sestavení do konfiguračního souboru pro nastavení naslouchacího procesu trasování, zahrnete plně kvalifikovaný název systémového .NET Framework sestavení:</span><span class="sxs-lookup"><span data-stu-id="37f7e-121">For example, to add an assembly reference in a configuration file to set up a trace listener, you would include the fully qualified name of the system .NET Framework assembly:</span></span>  
  
```xml  
<add name="myListener" type="System.Diagnostics.TextWriterTraceListener, System, Version=1.0.3300.0, Culture=neutral, PublicKeyToken=b77a5c561934e089" initializeData="c:\myListener.log" />  
```  
  
> [!NOTE]
> <span data-ttu-id="37f7e-122">Modul runtime považuje názvy sestavení za nerozlišování velkých a malých písmen při vytváření vazby na sestavení, ale zachová bez ohledu na případ použití v názvu sestavení.</span><span class="sxs-lookup"><span data-stu-id="37f7e-122">The runtime treats assembly names as case-insensitive when binding to an assembly, but preserves whatever case is used in an assembly name.</span></span> <span data-ttu-id="37f7e-123">Několik nástrojů v Windows SDK zpracovává názvy sestavení při rozlišování velkých a malých písmen.</span><span class="sxs-lookup"><span data-stu-id="37f7e-123">Several tools in the Windows SDK handle assembly names as case-sensitive.</span></span> <span data-ttu-id="37f7e-124">Pro dosažení nejlepších výsledků spravujte názvy sestavení, jako by se jednalo o velká a malá písmena.</span><span class="sxs-lookup"><span data-stu-id="37f7e-124">For best results, manage assembly names as though they were case-sensitive.</span></span>  
  
## <a name="name-application-components"></a><span data-ttu-id="37f7e-125">Pojmenovat součásti aplikace</span><span class="sxs-lookup"><span data-stu-id="37f7e-125">Name application components</span></span>  
 <span data-ttu-id="37f7e-126">Modul runtime při určování identity sestavení nebere v úvahu název souboru.</span><span class="sxs-lookup"><span data-stu-id="37f7e-126">The runtime does not consider the file name when determining an assembly's identity.</span></span> <span data-ttu-id="37f7e-127">Identita sestavení, která se skládá z názvu sestavení, verze, jazykové verze a silného názvu, musí být pro modul runtime nejasná.</span><span class="sxs-lookup"><span data-stu-id="37f7e-127">The assembly identity, which consists of the assembly name, version, culture, and strong name, must be clear to the runtime.</span></span>  
  
 <span data-ttu-id="37f7e-128">Například pokud máte sestavení s názvem *MyAssembly. exe* , které odkazuje na sestavení s názvem *MyAssembly. dll*, vazba proběhne správně, pokud spustíte *MyAssembly. exe*.</span><span class="sxs-lookup"><span data-stu-id="37f7e-128">For example, if you have an assembly called *myAssembly.exe* that references an assembly called *myAssembly.dll*, binding occurs correctly if you execute *myAssembly.exe*.</span></span> <span data-ttu-id="37f7e-129">Pokud však jiná aplikace provádí *MyAssembly. exe* pomocí metody <xref:System.AppDomain.ExecuteAssembly%2A?displayProperty=nameWithType>, modul runtime určí, že `myAssembly` již načtena, když *MyAssembly. exe* požaduje vazbu na `myAssembly`.</span><span class="sxs-lookup"><span data-stu-id="37f7e-129">However, if another application executes *myAssembly.exe* using the method <xref:System.AppDomain.ExecuteAssembly%2A?displayProperty=nameWithType>, the runtime determines that `myAssembly` is already loaded when *myAssembly.exe* requests binding to `myAssembly`.</span></span> <span data-ttu-id="37f7e-130">V tomto případě není *MyAssembly. dll* nikdy načtena.</span><span class="sxs-lookup"><span data-stu-id="37f7e-130">In this case, *myAssembly.dll* is never loaded.</span></span> <span data-ttu-id="37f7e-131">Protože *MyAssembly. exe* neobsahuje požadovaný typ, dojde k <xref:System.TypeLoadException>.</span><span class="sxs-lookup"><span data-stu-id="37f7e-131">Because *myAssembly.exe* does not contain the requested type, a <xref:System.TypeLoadException> occurs.</span></span>  
  
 <span data-ttu-id="37f7e-132">Chcete-li se tomuto problému vyhnout, ujistěte se, že sestavení, která tvoří vaši aplikaci, nemají stejný název sestavení nebo umísťují sestavení se stejným názvem do různých adresářů.</span><span class="sxs-lookup"><span data-stu-id="37f7e-132">To avoid this problem, make sure the assemblies that make up your application do not have the same assembly name or place assemblies with the same name in different directories.</span></span>  
  
> [!NOTE]
> <span data-ttu-id="37f7e-133">Pokud v .NET Framework umístíte sestavení se silným názvem do globální mezipaměti sestavení (GAC), název souboru sestavení musí odpovídat názvu sestavení, včetně přípony názvu souboru, například *. exe* nebo *. dll*.</span><span class="sxs-lookup"><span data-stu-id="37f7e-133">In the .NET Framework, if you put a strong-named assembly in the global assembly cache, the assembly's file name must match the assembly name, not including the file name extension, such as *.exe* or *.dll*.</span></span> <span data-ttu-id="37f7e-134">Například pokud název souboru sestavení je *MyAssembly. dll*, název sestavení musí být `myAssembly`.</span><span class="sxs-lookup"><span data-stu-id="37f7e-134">For example, if the file name of an assembly is *myAssembly.dll*, the assembly name must be `myAssembly`.</span></span> <span data-ttu-id="37f7e-135">Soukromá sestavení nasazená pouze v kořenovém adresáři aplikace mohou mít název sestavení, který se liší od názvu souboru.</span><span class="sxs-lookup"><span data-stu-id="37f7e-135">Private assemblies deployed only in the root application directory can have an assembly name that is different from the file name.</span></span>
