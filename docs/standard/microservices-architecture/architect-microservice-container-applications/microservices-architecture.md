<span data-ttu-id="b9cd6-104">Jak již název napovídá, architekturu mikroslužeb je takový přístup k sestavení aplikace serveru jako sada malých služeb.</span><span class="sxs-lookup"><span data-stu-id="b9cd6-104">As the name implies, a microservices architecture is an approach to building a server application as a set of small services.</span></span> <span data-ttu-id="b9cd6-105">To znamená, že architektura mikroslužeb je především orientované do back endu, přestože tento přístup se také používá pro front-endu.</span><span class="sxs-lookup"><span data-stu-id="b9cd6-105">That means a microservices architecture is mainly oriented to the back-end, although the approach is also being used for the front end.</span></span> <span data-ttu-id="b9cd6-106">Každá služba běží ve vlastním procesu a komunikuje s dalšími procesy pomocí protokolů, jako je například HTTP/HTTPS, protokoly Websocket, nebo [AMQP](https://en.wikipedia.org/wiki/Advanced_Message_Queuing_Protocol).</span><span class="sxs-lookup"><span data-stu-id="b9cd6-106">Each service runs in its own process and communicates with other implementuje konkrétní začátku do konce domény nebo obchodní funkci v rámci určité hranici kontextu a každý musí vyvinout samostatně a možno nasadit nezávisle na sobě.</span><span class="sxs-lookup"><span data-stu-id="b9cd6-107">Each microservice implements a specific end-to-end domain or business capability within a certain context boundary, and each must be developed autonomously and be deployable independently.</span></span> <span data-ttu-id="b9cd6-108">Nakonec každá mikroslužba by měl vlastní jeho související domény datový model a logiku domény (suverenitu a microservice should own its related domain data model and domain logic (sovereignty and decentralized data management) and could be based on different data storage technologies (SQL, NoSQL) and different programming languages.</span></span>

<span data-ttu-id="b9cd6-109">Jaké velikosti má být mikroslužby?</span><span class="sxs-lookup"><span data-stu-id="b9cd6-109">What size should a microservice be?</span></span> <span data-ttu-id="b9cd6-110">Při vývoji mikroslužby, velikost nesmí být důležité.</span><span class="sxs-lookup"><span data-stu-id="b9cd6-110">When developing a microservice, size shouldn't be the important point.</span></span> <span data-ttu-id="b9cd6-111">Místo toho by měl být důležité, k vytvoření volně s velkou provázaností, služby, máte autonomie vývoje, nasazení a škálování pro každou službu.</span><span class="sxs-lookup"><span data-stu-id="b9cd6-111">Instead, the important point should be to create loosely coupled services so you have autonomy of development, deployment, and scale, for each service.</span></span> <span data-ttu-id="b9cd6-112">Samozřejmě při identifikaci a návrh mikroslužeb, snažte tak, aby byly co nejmenší, dokud není nutné příliš mnoho přímých závislostí s další mikroslužeb.</span><span class="sxs-lookup"><span data-stu-id="b9cd6-112">Of course, when identifying and designing microservices, you should try to make them as small as possible as long as you don't have too many direct dependencies with other microservices.</span></span> <span data-ttu-id="b9cd6-113">Důležitější než velikost mikroslužby je interní soudržnosti, musí mít a jeho nezávislost od ostatních služeb.</span><span class="sxs-lookup"><span data-stu-id="b9cd6-113">More important than the size of the microservice is the internal cohesion it must have and its independence from other services.</span></span>

<span data-ttu-id="b9cd6-114">Proč architektury mikroslužeb?</span><span class="sxs-lookup"><span data-stu-id="b9cd6-114">Why a microservices architecture?</span></span> <span data-ttu-id="b9cd6-115">Stručně řečeno umožňuje dlouhodobé flexibilitu.</span><span class="sxs-lookup"><span data-stu-id="b9cd6-115">In short, it provides long-term agility.</span></span> <span data-ttu-id="b9cd6-116">Mikroslužby povolit lepší udržovatelnosti komplexní, velký a vysoce škálovatelné systémy umožňují vytvářet aplikace založené na mnoho samostatně nasaditelných služeb, že obsahují svazky detailní a autonomní životního cyklu.</span><span class="sxs-lookup"><span data-stu-id="b9cd6-116">Microservices enable better maintainability in complex, large, and highly-scalable systems by letting you create applications based on many independently deployable services that each have granular and autonomous lifecycles.</span></span>

<span data-ttu-id="b9cd6-117">Další výhodou je mikroslužby můžete škálovat nezávisle.</span><span class="sxs-lookup"><span data-stu-id="b9cd6-117">As an additional benefit, microservices can scale out independently.</span></span> <span data-ttu-id="b9cd6-118">Namísto toho, aby jedna monolitické aplikace musí horizontální navýšení kapacity jako celek, můžete místo toho horizontální navýšení kapacity konkrétní mikroslužeb.</span><span class="sxs-lookup"><span data-stu-id="b9cd6-118">Instead of having a single monolithic application that you must scale out as a unit, you can instead scale out specific microservices.</span></span> <span data-ttu-id="b9cd6-119">Tímto způsobem je možné škálovat jenom funkční oblast, která potřebuje další zpracování napájení nebo sítě šířky pásma pro podporu poptávky, nikoli horizontální navýšení kapacity dalších oblastí aplikace, které není potřeba škálovat.</span><span class="sxs-lookup"><span data-stu-id="b9cd6-119">That way, you can scale just the functional area that needs more processing power or network bandwidth to support demand, rather than scaling out other areas of the application that don't need to be scaled.</span></span> <span data-ttu-id="b9cd6-120">To znamená, že úspory nákladů, které vzhledem k tomu, že budete potřebovat méně hardwaru.</span><span class="sxs-lookup"><span data-stu-id="b9cd6-120">That means cost savings because you need less hardware.</span></span>

<span data-ttu-id="b9cd6-126">Aplikační architektura založená na jemně aplikací založených na mikroslužbách umožňuje průběžnou integraci a průběžné doručování postupy.</span><span class="sxs-lookup"><span data-stu-id="b9cd6-126">Architecting fine-grained microservices-based applications enables continuous integration and continuous delivery practices.</span></span> <span data-ttu-id="b9cd6-127">Také zrychluje doplňování nových funkcí do aplikace.</span><span class="sxs-lookup"><span data-stu-id="b9cd6-127">It also accelerates delivery of new functions into the application.</span></span> <span data-ttu-id="b9cd6-128">Detailní složení aplikace také umožňuje spustit a otestovat mikroslužeb v izolaci a autonomně rozvíjet při zachování vymazat kontraktů mezi nimi.</span><span class="sxs-lookup"><span data-stu-id="b9cd6-128">Fine-grained composition of applications also allows you to run and test microservices in isolation, and to evolve them autonomously while maintaining clear contracts between them.</span></span> <span data-ttu-id="b9cd6-129">Dokud nezměníte rozhraní nebo smlouvy, můžete změnit vnitřní implementace všechny mikroslužby nebo přidat nové funkce bez narušení další mikroslužeb.</span><span class="sxs-lookup"><span data-stu-id="b9cd6-129">As long as you don't change the interfaces or contracts, you can change the internal implementation of any microservice or add new functionality without breaking other microservices.</span></span>

- <span data-ttu-id="b9cd6-132">Škálovatelná infrastruktura služby (to znamená cloud a orchestrátorů).</span><span class="sxs-lookup"><span data-stu-id="b9cd6-132">Scalable infrastructure for the services (that is, cloud and orchestrators).</span></span>

- <span data-ttu-id="b9cd6-133">Zabezpečení návrhu a implementace na různých úrovních: ověřování, autorizace, správu tajných kódů, zabezpečenou komunikaci, atd.</span><span class="sxs-lookup"><span data-stu-id="b9cd6-133">Security design and implementation at multiple levels: authentication, authorization, secrets management, secure communication, etc.</span></span>

- <span data-ttu-id="b9cd6-134">Doručování aplikací rychle, obvykle s různými týmy, zaměřuje se na různé mikroslužeb.</span><span class="sxs-lookup"><span data-stu-id="b9cd6-134">Rapid application delivery, usually with different teams focusing on different microservices.</span></span>
