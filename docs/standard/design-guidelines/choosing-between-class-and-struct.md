# <a name="choosing-between-class-and-struct"></a><span data-ttu-id="2c8d1-102">Volba mezi třídou a strukturou</span><span class="sxs-lookup"><span data-stu-id="2c8d1-102">Choosing Between Class and Struct</span></span>
<span data-ttu-id="2c8d1-103">Jeden základní rozhodnutí o návrhu, který čelí každý framework designer je, zda navrhnout typu třídy (odkaz na typ) nebo jako – struktura (typu hodnoty).</span><span class="sxs-lookup"><span data-stu-id="2c8d1-103">One of the basic design decisions every framework designer faces is whether to design a type as a class (a reference type) or as a struct (a value type).</span></span> <span data-ttu-id="2c8d1-104">Je velmi důležité při vytvoření tato volba dostatečné povědomí o rozdíly v chování typy odkazů a typy hodnot.</span><span class="sxs-lookup"><span data-stu-id="2c8d1-104">Good understanding of the differences in the behavior of reference types and value types is crucial in making this choice.</span></span>  
  
 <span data-ttu-id="2c8d1-105">První rozdíl mezi typy odkazů a typy hodnot, které posoudíme je, že jsou odkazové typy přiděleného v haldě a prováděno uvolnění paměti, že typy hodnot jsou přiděleny na zásobníku nebo vložené v obsahující typy a uvolněna při zásobníku unwinds nebo při jejich nadřazeným typem uvolnění.</span><span class="sxs-lookup"><span data-stu-id="2c8d1-105">The first difference between reference types and value types we will consider is that reference types are allocated on the heap and garbage-collected, whereas value types are allocated either on the stack or inline in containing types and deallocated when the stack unwinds or when their containing type gets deallocated.</span></span> <span data-ttu-id="2c8d1-106">Přidělování a navrácení typů hodnot, proto jsou obecně levnější než přidělování a navrácení odkazových typů.</span><span class="sxs-lookup"><span data-stu-id="2c8d1-106">Therefore, allocations and deallocations of value types are in general cheaper than allocations and deallocations of reference types.</span></span>  
  
 <span data-ttu-id="2c8d1-107">V dalším kroku pole odkazu na typy jsou přidělené mimo řádek, což znamená pole, které prvky jsou pouze odkazy na instance typu odkazu, které se nacházejí na haldě.</span><span class="sxs-lookup"><span data-stu-id="2c8d1-107">Next, arrays of reference types are allocated out-of-line, meaning the array elements are just references to instances of the reference type residing on the heap.</span></span> <span data-ttu-id="2c8d1-108">Pole typu hodnota přidělují vloženě, což znamená, že prvky pole jsou přímo instance typu hodnoty.</span><span class="sxs-lookup"><span data-stu-id="2c8d1-108">Value type arrays are allocated inline, meaning that the array elements are the actual instances of the value type.</span></span> <span data-ttu-id="2c8d1-109">Přidělování a navrácení pole typu hodnota jsou tedy mnohem levnější než přidělování a navrácení pole typu odkazu.</span><span class="sxs-lookup"><span data-stu-id="2c8d1-109">Therefore, allocations and deallocations of value type arrays are much cheaper than allocations and deallocations of reference type arrays.</span></span> <span data-ttu-id="2c8d1-110">Kromě toho pole typu hodnota ve většině případů je mnohem lepší místo odkazu.</span><span class="sxs-lookup"><span data-stu-id="2c8d1-110">In addition, in a majority of cases value type arrays exhibit much better locality of reference.</span></span>  
  
 <span data-ttu-id="2c8d1-111">Další rozdíl se týká využití paměti.</span><span class="sxs-lookup"><span data-stu-id="2c8d1-111">The next difference is related to memory usage.</span></span> <span data-ttu-id="2c8d1-112">Typy hodnot získat zabalená, pokud přetypovat na typ odkazu nebo jednoho z rozhraní, které implementují.</span><span class="sxs-lookup"><span data-stu-id="2c8d1-112">Value types get boxed when cast to a reference type or one of the interfaces they implement.</span></span> <span data-ttu-id="2c8d1-113">Vývojáři získají nezabalený Pokud přetypovat zpět na typ hodnoty.</span><span class="sxs-lookup"><span data-stu-id="2c8d1-113">They get unboxed when cast back to the value type.</span></span> <span data-ttu-id="2c8d1-114">Protože pole jsou objekty, které jsou přiděleny do haldy a jsou uvolněna, příliš mnoho zabalení a rozbalení může mít negativní dopad na haldě systému uvolňování paměti a nakonec výkon aplikace.</span><span class="sxs-lookup"><span data-stu-id="2c8d1-114">Because boxes are objects that are allocated on the heap and are garbage-collected, too much boxing and unboxing can have a negative impact on the heap, the garbage collector, and ultimately the performance of the application.</span></span>  <span data-ttu-id="2c8d1-115">Naproti tomu žádné takové zabalení dochází při typy odkazů jsou přetypovat.</span><span class="sxs-lookup"><span data-stu-id="2c8d1-115">In contrast, no such boxing occurs as reference types are cast.</span></span> <span data-ttu-id="2c8d1-116">(Další informace najdete v tématu [zabalení a rozbalení](../../csharp/programming-guide/types/boxing-and-unboxing.md)).</span><span class="sxs-lookup"><span data-stu-id="2c8d1-116">(For more information, see [Boxing and Unboxing](../../csharp/programming-guide/types/boxing-and-unboxing.md)).</span></span>
  
 <span data-ttu-id="2c8d1-117">Přiřazení typu odkazu v dalším kroku zkopírujte odkaz, zatímco přiřazení typu hodnoty zkopírujte celou hodnotu.</span><span class="sxs-lookup"><span data-stu-id="2c8d1-117">Next, reference type assignments copy the reference, whereas value type assignments copy the entire value.</span></span> <span data-ttu-id="2c8d1-118">Proto jsou levnější než přiřazení typů velkých hodnot přiřazení velké referenční typy.</span><span class="sxs-lookup"><span data-stu-id="2c8d1-118">Therefore, assignments of large reference types are cheaper than assignments of large value types.</span></span>  
  
 <span data-ttu-id="2c8d1-119">A konečně typy odkazů jsou předány podle odkazu, zatímco hodnotové typy jsou předávány hodnotou.</span><span class="sxs-lookup"><span data-stu-id="2c8d1-119">Finally, reference types are passed by reference, whereas value types are passed by value.</span></span> <span data-ttu-id="2c8d1-120">Na instanci typu odkazu ovlivní všechny odkazy odkazující na instanci.</span><span class="sxs-lookup"><span data-stu-id="2c8d1-120">Changes to an instance of a reference type affect all references pointing to the instance.</span></span> <span data-ttu-id="2c8d1-121">Instance typu hodnoty jsou zkopírovány, když jsou předány podle hodnoty.</span><span class="sxs-lookup"><span data-stu-id="2c8d1-121">Value type instances are copied when they are passed by value.</span></span> <span data-ttu-id="2c8d1-122">Při změně instance hodnotového typu samozřejmě neovlivňuje některý z jeho kopie.</span><span class="sxs-lookup"><span data-stu-id="2c8d1-122">When an instance of a value type is changed, it of course does not affect any of its copies.</span></span> <span data-ttu-id="2c8d1-123">Protože tyto kopie nejsou explicitně vytvořených uživatelem se implicitně vytvářejí, když jsou předány argumenty nebo návratové hodnoty jsou vráceny, může být matoucí pro mnoho uživatelů typů hodnot, které je možné změnit.</span><span class="sxs-lookup"><span data-stu-id="2c8d1-123">Because the copies are not created explicitly by the user but are implicitly created when arguments are passed or return values are returned, value types that can be changed can be confusing to many users.</span></span> <span data-ttu-id="2c8d1-124">Typy hodnot, proto by měl být neměnitelný.</span><span class="sxs-lookup"><span data-stu-id="2c8d1-124">Therefore, value types should be immutable.</span></span>  
  
 <span data-ttu-id="2c8d1-125">Jako říci by měla být většina typů v rámci třídy.</span><span class="sxs-lookup"><span data-stu-id="2c8d1-125">As a rule of thumb, the majority of types in a framework should be classes.</span></span> <span data-ttu-id="2c8d1-126">Existují však některé situace, ve kterých vlastnosti typu hodnoty usnadňují vhodnější použít struktury.</span><span class="sxs-lookup"><span data-stu-id="2c8d1-126">There are, however, some situations in which the characteristics of a value type make it more appropriate to use structs.</span></span>  
  
 <span data-ttu-id="2c8d1-127">**✓ CONSIDER** definice struktury místo třídu, pokud instance typu jsou malé a běžně krátkodobou nebo jsou běžně součástí jiné objekty.</span><span class="sxs-lookup"><span data-stu-id="2c8d1-127">**✓ CONSIDER** defining a struct instead of a class if instances of the type are small and commonly short-lived or are commonly embedded in other objects.</span></span>  
  
 <span data-ttu-id="2c8d1-128">**X AVOID** definice struktury, pokud má tento typ všechny následující vlastnosti:</span><span class="sxs-lookup"><span data-stu-id="2c8d1-128">**X AVOID** defining a struct unless the type has all of the following characteristics:</span></span>  
  
- <span data-ttu-id="2c8d1-129">Logicky reprezentuje hodnotu single, podobně jako primitivní typy (`int`, `double`atd.).</span><span class="sxs-lookup"><span data-stu-id="2c8d1-129">It logically represents a single value, similar to primitive types (`int`, `double`, etc.).</span></span>  
  
- <span data-ttu-id="2c8d1-130">To má velikost instance mladší 16 bajtů.</span><span class="sxs-lookup"><span data-stu-id="2c8d1-130">It has an instance size under 16 bytes.</span></span>  
  
- <span data-ttu-id="2c8d1-131">To se nedá změnit.</span><span class="sxs-lookup"><span data-stu-id="2c8d1-131">It is immutable.</span></span>  
  
- <span data-ttu-id="2c8d1-132">Nebudete muset použít boxing. často.</span><span class="sxs-lookup"><span data-stu-id="2c8d1-132">It will not have to be boxed frequently.</span></span>  
  
 <span data-ttu-id="2c8d1-133">Ve všech ostatních případech byste měli definovat vaše typy jako třídy.</span><span class="sxs-lookup"><span data-stu-id="2c8d1-133">In all other cases, you should define your types as classes.</span></span>
