# <a name="dependency-properties"></a><span data-ttu-id="eb097-102">Vlastnosti závislosti</span><span class="sxs-lookup"><span data-stu-id="eb097-102">Dependency Properties</span></span>
<span data-ttu-id="eb097-103">Vlastnost závislosti (DP) je regulární vlastnost, která uloží svou hodnotu v úložišti vlastností místo ukládání proměnná typu (pole), např.</span><span class="sxs-lookup"><span data-stu-id="eb097-103">A dependency property (DP) is a regular property that stores its value in a property store instead of storing it in a type variable (field), for example.</span></span>  
  
 <span data-ttu-id="eb097-104">Na vlastnost připojené závislostí je druh vlastnost závislosti nemodelují jako statické metody Get a Set představující "properties" popisující vztahy mezi objekty a jejich kontejnery (například pozici `Button` objektu na `Panel` kontejner).</span><span class="sxs-lookup"><span data-stu-id="eb097-104">An attached dependency property is a kind of dependency property modeled as static Get and Set methods representing "properties" describing relationships between objects and their containers (e.g., the position of a `Button` object on a `Panel` container).</span></span>  
  
 <span data-ttu-id="eb097-105">**✓ DO** zadejte vlastnosti závislosti, pokud budete potřebovat k podpoře funkce WPF například stylů, aktivační události, vazby dat, animací, dynamické prostředky a dědičnosti vlastností.</span><span class="sxs-lookup"><span data-stu-id="eb097-105">**✓ DO** provide the dependency properties, if you need the properties to support WPF features such as styling, triggers, data binding, animations, dynamic resources, and inheritance.</span></span>  
  
## <a name="dependency-property-design"></a><span data-ttu-id="eb097-106">Návrh vlastnosti závislosti</span><span class="sxs-lookup"><span data-stu-id="eb097-106">Dependency Property Design</span></span>  
 <span data-ttu-id="eb097-107">**✓ DO** dědí <xref:System.Windows.DependencyObject>, nebo jeden z jeho podtypech při implementaci vlastností závislostí.</span><span class="sxs-lookup"><span data-stu-id="eb097-107">**✓ DO** inherit from <xref:System.Windows.DependencyObject>, or one of its subtypes, when implementing dependency properties.</span></span> <span data-ttu-id="eb097-108">Typ poskytuje implementaci velmi efektivní úložiště vlastností a automaticky podporuje datové vazby WPF.</span><span class="sxs-lookup"><span data-stu-id="eb097-108">The type provides a very efficient implementation of a property store and automatically supports WPF data binding.</span></span>  
  
 <span data-ttu-id="eb097-109">**✓ DO** zadejte regulární vlastnost CLR a veřejné statické jen pro čtení pole ukládání instanci <xref:System.Windows.DependencyProperty?displayProperty=nameWithType> pro každou vlastnost závislosti.</span><span class="sxs-lookup"><span data-stu-id="eb097-109">**✓ DO** provide a regular CLR property and public static read-only field storing an instance of <xref:System.Windows.DependencyProperty?displayProperty=nameWithType> for each dependency property.</span></span>  
  
 <span data-ttu-id="eb097-110">**✓ DO** implementovat vlastností závislostí voláním metod instance <xref:System.Windows.DependencyObject.GetValue%2A?displayProperty=nameWithType> a <xref:System.Windows.DependencyObject.SetValue%2A?displayProperty=nameWithType>.</span><span class="sxs-lookup"><span data-stu-id="eb097-110">**✓ DO** implement dependency properties by calling instance methods <xref:System.Windows.DependencyObject.GetValue%2A?displayProperty=nameWithType> and <xref:System.Windows.DependencyObject.SetValue%2A?displayProperty=nameWithType>.</span></span>  
  
 <span data-ttu-id="eb097-111">**✓ DO** názvů statické pole vlastnosti závislosti suffixing název vlastnosti s "Vlastnost."</span><span class="sxs-lookup"><span data-stu-id="eb097-111">**✓ DO** name the dependency property static field by suffixing the name of the property with "Property."</span></span>  
  
 <span data-ttu-id="eb097-112">**X DO NOT** explicitně nastavit výchozí hodnoty vlastností závislostí v kódu; nastavte je v metadatech.</span><span class="sxs-lookup"><span data-stu-id="eb097-112">**X DO NOT** set default values of dependency properties explicitly in code; set them in metadata instead.</span></span>  
  
 <span data-ttu-id="eb097-113">Pokud nastavíte výchozí vlastnost explicitně, by mohly bránit tuto vlastnost z nastavování některé implicitní prostředky, například styl.</span><span class="sxs-lookup"><span data-stu-id="eb097-113">If you set a property default explicitly, you might prevent that property from being set by some implicit means, such as a styling.</span></span>  
  
 <span data-ttu-id="eb097-114">**X DO NOT** vkládat kód do vlastnosti přistupující objekty než standardní kód pro přístup k statické pole.</span><span class="sxs-lookup"><span data-stu-id="eb097-114">**X DO NOT** put code in the property accessors other than the standard code to access the static field.</span></span>  
  
 <span data-ttu-id="eb097-115">Statické pole, že kód nebude spuštěno Pokud je nastavena implicitní prostředky, například styl, protože práce se styly používá přímo.</span><span class="sxs-lookup"><span data-stu-id="eb097-115">That code won’t execute if the property is set by implicit means, such as a styling, because styling uses the static field directly.</span></span>  
  
 <span data-ttu-id="eb097-116">**X DO NOT** zabezpečené data pomocí vlastností závislostí.</span><span class="sxs-lookup"><span data-stu-id="eb097-116">**X DO NOT** use dependency properties to store secure data.</span></span> <span data-ttu-id="eb097-117">Vlastnosti závislostí i privátní přístupná veřejně.</span><span class="sxs-lookup"><span data-stu-id="eb097-117">Even private dependency properties can be accessed publicly.</span></span>  
  
## <a name="attached-dependency-property-design"></a><span data-ttu-id="eb097-118">Návrh vlastnosti připojené závislostí</span><span class="sxs-lookup"><span data-stu-id="eb097-118">Attached Dependency Property Design</span></span>  
 <span data-ttu-id="eb097-119">Vlastnosti závislostí popsaných v předchozí části představují vnitřní vlastnosti deklarujícího typu; například `Text` vlastností je vlastnost `TextButton`, který deklaruje.</span><span class="sxs-lookup"><span data-stu-id="eb097-119">Dependency properties described in the preceding section represent intrinsic properties of the declaring type; for example, the `Text` property is a property of `TextButton`, which declares it.</span></span> <span data-ttu-id="eb097-120">Zvláštní druh vlastnost závislosti je vlastnost připojené závislosti.</span><span class="sxs-lookup"><span data-stu-id="eb097-120">A special kind of dependency property is the attached dependency property.</span></span>  
  
 <span data-ttu-id="eb097-121">Klasické příkladem připojené vlastnosti je <xref:System.Windows.Controls.Grid.Column%2A?displayProperty=nameWithType> vlastnost.</span><span class="sxs-lookup"><span data-stu-id="eb097-121">A classic example of an attached property is the <xref:System.Windows.Controls.Grid.Column%2A?displayProperty=nameWithType> property.</span></span> <span data-ttu-id="eb097-122">Vlastnost představuje pozici sloupce tlačítka (ne mřížky), ale je pouze relevantní tlačítko je součástí do mřížky, takže je "připojeno" k tlačítka pomocí mřížky.</span><span class="sxs-lookup"><span data-stu-id="eb097-122">The property represents Button’s (not Grid’s) column position, but it is only relevant if the Button is contained in a Grid, and so it's "attached" to Buttons by Grids.</span></span>  
  
```xaml
<Grid>  
    <Grid.ColumnDefinitions>  
        <ColumnDefinition />  
        <ColumnDefinition />  
    </Grid.ColumnDefinitions>  
  
    <Button Grid.Column="0">Click</Button>  
    <Button Grid.Column="1">Clack</Button>  
</Grid>  
```  
  
 <span data-ttu-id="eb097-123">Definice připojené vlastnosti vypadá převážně u vlastnosti regulárního závislosti, s tím rozdílem, přístupové objekty jsou reprezentovány statické metody Get a Set:</span><span class="sxs-lookup"><span data-stu-id="eb097-123">The definition of an attached property looks mostly like that of a regular dependency property, except that the accessors are represented by static Get and Set methods:</span></span>  
  
```csharp
public class Grid {  
  
    public static int GetColumn(DependencyObject obj) {  
        return (int)obj.GetValue(ColumnProperty);  
    }  
  
    public static void SetColumn(DependencyObject obj, int value) {  
        obj.SetValue(ColumnProperty,value);  
    }  
  
    public static readonly DependencyProperty ColumnProperty =  
        DependencyProperty.RegisterAttached(  
            "Column",  
            typeof(int),  
            typeof(Grid)  
    );  
}  
```  
  
## <a name="dependency-property-validation"></a><span data-ttu-id="eb097-124">Ověření vlastností závislostí</span><span class="sxs-lookup"><span data-stu-id="eb097-124">Dependency Property Validation</span></span>  
 <span data-ttu-id="eb097-125">Vlastnosti často implementovat, co se nazývá ověření.</span><span class="sxs-lookup"><span data-stu-id="eb097-125">Properties often implement what is called validation.</span></span> <span data-ttu-id="eb097-126">Logiku ověřování provede při pokusu o změnit hodnotu vlastnosti.</span><span class="sxs-lookup"><span data-stu-id="eb097-126">Validation logic executes when an attempt is made to change the value of a property.</span></span>  
  
 <span data-ttu-id="eb097-127">Bohužel přistupující objekty vlastnosti závislost nemůže obsahovat libovolný ověřovací kód.</span><span class="sxs-lookup"><span data-stu-id="eb097-127">Unfortunately dependency property accessors cannot contain arbitrary validation code.</span></span> <span data-ttu-id="eb097-128">Místo toho logiku ověření závislostí vlastnost je nutné zadat během registrace vlastnost.</span><span class="sxs-lookup"><span data-stu-id="eb097-128">Instead, dependency property validation logic needs to be specified during property registration.</span></span>  
  
 <span data-ttu-id="eb097-129">**X DO NOT** chápat logiku ověření pro vlastnost závislosti vlastnosti přistupující objekty.</span><span class="sxs-lookup"><span data-stu-id="eb097-129">**X DO NOT** put dependency property validation logic in the property’s accessors.</span></span> <span data-ttu-id="eb097-130">Místo toho předat zpětné volání pro ověření do `DependencyProperty.Register` metody.</span><span class="sxs-lookup"><span data-stu-id="eb097-130">Instead, pass a validation callback to `DependencyProperty.Register` method.</span></span>  
  
## <a name="dependency-property-change-notifications"></a><span data-ttu-id="eb097-131">Oznámení změn vlastností závislosti</span><span class="sxs-lookup"><span data-stu-id="eb097-131">Dependency Property Change Notifications</span></span>

## <a name="dependency-property-value-coercion"></a><span data-ttu-id="eb097-134">Vynucení hodnotu vlastnosti závislosti</span><span class="sxs-lookup"><span data-stu-id="eb097-134">Dependency Property Value Coercion</span></span>  
 <span data-ttu-id="eb097-135">Vlastnost převod se provádí při změně hodnoty zadané pro vlastnost setter podle setter předtím, než se skutečně upraví úložiště vlastností.</span><span class="sxs-lookup"><span data-stu-id="eb097-135">Property coercion takes place when the value given to a property setter is modified by the setter before the property store is actually modified.</span></span>

 <span data-ttu-id="eb097-137">Vlastnosti závislosti mají integrovanou vynucení funkce a je možné poskytnutím zpětné volání vynucení, aby `PropertyMetadata`.</span><span class="sxs-lookup"><span data-stu-id="eb097-137">Dependency properties have a built-in coercion feature, and it can be used by supplying a coercion callback to the `PropertyMetadata`.</span></span>
