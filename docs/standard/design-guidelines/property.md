# <a name="property-design"></a><span data-ttu-id="05dad-102">Návrh vlastnosti</span><span class="sxs-lookup"><span data-stu-id="05dad-102">Property Design</span></span>
<span data-ttu-id="05dad-103">Přestože jsou vlastnosti technicky velmi podobné metody, jsou značně odlišná z hlediska jejich scénáře použití.</span><span class="sxs-lookup"><span data-stu-id="05dad-103">Although properties are technically very similar to methods, they are quite different in terms of their usage scenarios.</span></span> <span data-ttu-id="05dad-104">Měla by se zobrazit jako inteligentní pole.</span><span class="sxs-lookup"><span data-stu-id="05dad-104">They should be seen as smart fields.</span></span> <span data-ttu-id="05dad-105">Mají volání syntaxe pole a flexibilitu metody.</span><span class="sxs-lookup"><span data-stu-id="05dad-105">They have the calling syntax of fields, and the flexibility of methods.</span></span>  
  
 <span data-ttu-id="05dad-106">**✓ DO** vytvořit vlastnosti jen pro get, pokud má volající neměli mít možnost ke změně hodnoty vlastnosti.</span><span class="sxs-lookup"><span data-stu-id="05dad-106">**✓ DO** create get-only properties if the caller should not be able to change the value of the property.</span></span>  
  
 <span data-ttu-id="05dad-107">Mějte na paměti, že pokud typ vlastnosti je proměnlivý odkazový typ, hodnoty vlastnosti lze změnit i v případě, že vlastnost je určená jenom pro.</span><span class="sxs-lookup"><span data-stu-id="05dad-107">Keep in mind that if the type of the property is a mutable reference type, the property value can be changed even if the property is get-only.</span></span>  
  
 <span data-ttu-id="05dad-108">**X DO NOT** poskytnout setter s širší usnadnění než metoda getter vlastnosti jen pro sadu nebo vlastnosti.</span><span class="sxs-lookup"><span data-stu-id="05dad-108">**X DO NOT** provide set-only properties or properties with the setter having broader accessibility than the getter.</span></span>  
  
 <span data-ttu-id="05dad-109">Například nepoužívejte vlastnosti veřejné metody setter a getter chráněné.</span><span class="sxs-lookup"><span data-stu-id="05dad-109">For example, do not use properties with a public setter and a protected getter.</span></span>  
  
 <span data-ttu-id="05dad-110">Pokud metoda getter vlastnosti nejde zadat, implementujte funkci jako metodu.</span><span class="sxs-lookup"><span data-stu-id="05dad-110">If the property getter cannot be provided, implement the functionality as a method instead.</span></span> <span data-ttu-id="05dad-111">Vezměte v úvahu od názvu metody název `Set` a postupujte podle pokynů s co by mít pojmenujete vlastnost.</span><span class="sxs-lookup"><span data-stu-id="05dad-111">Consider starting the method name with `Set` and follow with what you would have named the property.</span></span> <span data-ttu-id="05dad-112">Například <xref:System.AppDomain> obsahuje metodu nazvanou `SetCachePath` namísto toho, aby vlastnost jen pro sadu s názvem `CachePath`.</span><span class="sxs-lookup"><span data-stu-id="05dad-112">For example, <xref:System.AppDomain> has a method called `SetCachePath` instead of having a set-only property called `CachePath`.</span></span>  
  
 <span data-ttu-id="05dad-113">**✓ DO** zadejte rozumný výchozí hodnoty pro všechny vlastnosti, zajistíte, že výchozí hodnoty nezpůsobovalo neustálé bezpečnostní riziko nebo terribly neefektivní kód.</span><span class="sxs-lookup"><span data-stu-id="05dad-113">**✓ DO** provide sensible default values for all properties, ensuring that the defaults do not result in a security hole or terribly inefficient code.</span></span>  
  
 <span data-ttu-id="05dad-114">**✓ DO** povolit vlastnosti nastavit v libovolném pořadí, i když to vede k dočasné neplatný stav objektu.</span><span class="sxs-lookup"><span data-stu-id="05dad-114">**✓ DO** allow properties to be set in any order even if this results in a temporary invalid state of the object.</span></span>  
  
 <span data-ttu-id="05dad-115">Je běžné, že dva nebo více vlastností tak, aby vzájemně propojené do bodu, ve kterých některé hodnoty jednu vlastnost může být neplatný zadané hodnoty dalších vlastností na stejný objekt.</span><span class="sxs-lookup"><span data-stu-id="05dad-115">It is common for two or more properties to be interrelated to a point where some values of one property might be invalid given the values of other properties on the same object.</span></span> <span data-ttu-id="05dad-116">V takovém případě by měly být výjimky vyplývající z neplatný stav odložit, dokud vzájemně propojené vlastnosti jsou skutečně používané tímto objektem společně.</span><span class="sxs-lookup"><span data-stu-id="05dad-116">In such cases, exceptions resulting from the invalid state should be postponed until the interrelated properties are actually used together by the object.</span></span>  
  
 <span data-ttu-id="05dad-117">**✓ DO** zachovat předchozí hodnotu, pokud metoda setter vlastnosti vyvolá výjimku.</span><span class="sxs-lookup"><span data-stu-id="05dad-117">**✓ DO** preserve the previous value if a property setter throws an exception.</span></span>  
  
 <span data-ttu-id="05dad-118">**X AVOID** vyvolání výjimky z metody vlastnost getter.</span><span class="sxs-lookup"><span data-stu-id="05dad-118">**X AVOID** throwing exceptions from property getters.</span></span>  
  
 <span data-ttu-id="05dad-119">Gettery vlastností by měl být jednoduché operace a nemají žádné předpoklady.</span><span class="sxs-lookup"><span data-stu-id="05dad-119">Property getters should be simple operations and should not have any preconditions.</span></span> <span data-ttu-id="05dad-120">Pokud getter může vyvolat výjimku, by pravděpodobně upravena jako metodu.</span><span class="sxs-lookup"><span data-stu-id="05dad-120">If a getter can throw an exception, it should probably be redesigned to be a method.</span></span> <span data-ttu-id="05dad-121">Všimněte si, že toto pravidlo se nevztahuje na indexery, kde Očekáváme, že výjimky v důsledku ověřování argumentů.</span><span class="sxs-lookup"><span data-stu-id="05dad-121">Notice that this rule does not apply to indexers, where we do expect exceptions as a result of validating the arguments.</span></span>  
  
### <a name="indexed-property-design"></a><span data-ttu-id="05dad-122">Indexovaná vlastnost návrhu</span><span class="sxs-lookup"><span data-stu-id="05dad-122">Indexed Property Design</span></span>  
 <span data-ttu-id="05dad-123">Indexovaná vlastnost je speciální vlastnost, která může mít parametry a mohou být volány pomocí speciální syntaxe podobná indexování pole.</span><span class="sxs-lookup"><span data-stu-id="05dad-123">An indexed property is a special property that can have parameters and can be called with special syntax similar to array indexing.</span></span>  
  
 <span data-ttu-id="05dad-124">Indexované vlastnosti se běžně označují jako indexery.</span><span class="sxs-lookup"><span data-stu-id="05dad-124">Indexed properties are commonly referred to as indexers.</span></span> <span data-ttu-id="05dad-125">Indexery by měla sloužit pouze v rozhraní API, která poskytuje přístup k položkám v logické kolekce.</span><span class="sxs-lookup"><span data-stu-id="05dad-125">Indexers should be used only in APIs that provide access to items in a logical collection.</span></span> <span data-ttu-id="05dad-126">Například řetězec je kolekce znaků a indexer na <xref:System.String?displayProperty=nameWithType> byl přidán přístup k jeho znaků.</span><span class="sxs-lookup"><span data-stu-id="05dad-126">For example, a string is a collection of characters, and the indexer on <xref:System.String?displayProperty=nameWithType> was added to access its characters.</span></span>  
  
 <span data-ttu-id="05dad-127">**✓ CONSIDER** použití indexery pro poskytnutí přístupu k datům uloženým v interní pole.</span><span class="sxs-lookup"><span data-stu-id="05dad-127">**✓ CONSIDER** using indexers to provide access to data stored in an internal array.</span></span>  
  
 <span data-ttu-id="05dad-128">**✓ CONSIDER** poskytuje indexery na typy reprezentující kolekce položek.</span><span class="sxs-lookup"><span data-stu-id="05dad-128">**✓ CONSIDER** providing indexers on types representing collections of items.</span></span>  
  
 <span data-ttu-id="05dad-129">**X AVOID** použití indexovaných vlastností s více než jeden parametr.</span><span class="sxs-lookup"><span data-stu-id="05dad-129">**X AVOID** using indexed properties with more than one parameter.</span></span>  
  
 <span data-ttu-id="05dad-130">Pokud návrhu vyžaduje více parametrů, zvažte, jestli přistupující objekt vlastnosti ve skutečnosti představuje logické kolekce.</span><span class="sxs-lookup"><span data-stu-id="05dad-130">If the design requires multiple parameters, reconsider whether the property really represents an accessor to a logical collection.</span></span> <span data-ttu-id="05dad-131">Pokud tomu tak není, použijte metody.</span><span class="sxs-lookup"><span data-stu-id="05dad-131">If it does not, use methods instead.</span></span> <span data-ttu-id="05dad-132">Vezměte v úvahu od názvu metody název `Get` nebo `Set`.</span><span class="sxs-lookup"><span data-stu-id="05dad-132">Consider starting the method name with `Get` or `Set`.</span></span>  
  
 <span data-ttu-id="05dad-133">**X AVOID** indexery s typy parametrů jinak než <xref:System.Int32?displayProperty=nameWithType>, <xref:System.Int64?displayProperty=nameWithType>, <xref:System.String?displayProperty=nameWithType>, <xref:System.Object?displayProperty=nameWithType>, nebo výčet.</span><span class="sxs-lookup"><span data-stu-id="05dad-133">**X AVOID** indexers with parameter types other than <xref:System.Int32?displayProperty=nameWithType>, <xref:System.Int64?displayProperty=nameWithType>, <xref:System.String?displayProperty=nameWithType>, <xref:System.Object?displayProperty=nameWithType>, or an enum.</span></span>  
  
 <span data-ttu-id="05dad-134">Pokud návrhu vyžaduje jiné typy parametrů, důrazně opětovné vyhodnocení, jestli přistupující objekt rozhraní API ve skutečnosti představuje logické kolekce.</span><span class="sxs-lookup"><span data-stu-id="05dad-134">If the design requires other types of parameters, strongly reevaluate whether the API really represents an accessor to a logical collection.</span></span> <span data-ttu-id="05dad-135">Pokud tomu tak není, použijte metodu.</span><span class="sxs-lookup"><span data-stu-id="05dad-135">If it does not, use a method.</span></span> <span data-ttu-id="05dad-136">Vezměte v úvahu od názvu metody název `Get` nebo `Set`.</span><span class="sxs-lookup"><span data-stu-id="05dad-136">Consider starting the method name with `Get` or `Set`.</span></span>  
  
 <span data-ttu-id="05dad-137">**✓ DO** použijte název `Item` pro indexovaných vlastností, pokud je samozřejmě lepší název (například najdete v článku <xref:System.String.Chars%2A> vlastnost na `System.String`).</span><span class="sxs-lookup"><span data-stu-id="05dad-137">**✓ DO** use the name `Item` for indexed properties unless there is an obviously better name (e.g., see the <xref:System.String.Chars%2A> property on `System.String`).</span></span>  
  
 <span data-ttu-id="05dad-138">V jazyce C# indexery jsou ve výchozím nastavení s názvem položky.</span><span class="sxs-lookup"><span data-stu-id="05dad-138">In C#, indexers are by default named Item.</span></span> <span data-ttu-id="05dad-139"><xref:System.Runtime.CompilerServices.IndexerNameAttribute> Lze upravit tento název.</span><span class="sxs-lookup"><span data-stu-id="05dad-139">The <xref:System.Runtime.CompilerServices.IndexerNameAttribute> can be used to customize this name.</span></span>  
  
 <span data-ttu-id="05dad-140">**X DO NOT** zadejte indexer a metody, které jsou sémanticky ekvivalentní.</span><span class="sxs-lookup"><span data-stu-id="05dad-140">**X DO NOT** provide both an indexer and methods that are semantically equivalent.</span></span>  
  
 <span data-ttu-id="05dad-141">**X DO NOT** zadat více než jednu řadu přetížené indexery jednomu typu.</span><span class="sxs-lookup"><span data-stu-id="05dad-141">**X DO NOT** provide more than one family of overloaded indexers in one type.</span></span>  
  
 <span data-ttu-id="05dad-142">To je vynucena kompilátorem jazyka C#.</span><span class="sxs-lookup"><span data-stu-id="05dad-142">This is enforced by the C# compiler.</span></span>  
  
 <span data-ttu-id="05dad-143">**X DO NOT** nevýchozí použití indexovaných vlastností.</span><span class="sxs-lookup"><span data-stu-id="05dad-143">**X DO NOT** use nondefault indexed properties.</span></span>  
  
 <span data-ttu-id="05dad-144">To je vynucena kompilátorem jazyka C#.</span><span class="sxs-lookup"><span data-stu-id="05dad-144">This is enforced by the C# compiler.</span></span>  
  
### <a name="property-change-notification-events"></a><span data-ttu-id="05dad-145">Oznámení události změny vlastnosti</span><span class="sxs-lookup"><span data-stu-id="05dad-145">Property Change Notification Events</span></span>  
 <span data-ttu-id="05dad-146">Někdy je užitečné poskytnout událost upozornění uživatele změny hodnoty vlastnosti.</span><span class="sxs-lookup"><span data-stu-id="05dad-146">Sometimes it is useful to provide an event notifying the user of changes in a property value.</span></span> <span data-ttu-id="05dad-147">Například `System.Windows.Forms.Control` vyvolá `TextChanged` událostí po hodnota jeho `Text` je změněna vlastnost.</span><span class="sxs-lookup"><span data-stu-id="05dad-147">For example, `System.Windows.Forms.Control` raises a `TextChanged` event after the value of its `Text` property has changed.</span></span>  
  
 <span data-ttu-id="05dad-148">**✓ CONSIDER** vyvolání změnit události oznámení při změně hodnoty vlastností v vysoké úrovně rozhraní API (obvykle návrháře komponenty).</span><span class="sxs-lookup"><span data-stu-id="05dad-148">**✓ CONSIDER** raising change notification events when property values in high-level APIs (usually designer components) are modified.</span></span>  
  
 <span data-ttu-id="05dad-149">Pokud je dobré scénář pro uživatele potřebujete vědět, kdy je změna vlastnosti objektu, by měla vyvolat objekt událost oznámení o změně pro vlastnost.</span><span class="sxs-lookup"><span data-stu-id="05dad-149">If there is a good scenario for a user to know when a property of an object is changing, the object should raise a change notification event for the property.</span></span>  
  
 <span data-ttu-id="05dad-150">Je ale pravděpodobné, aby za režie potřebná k vyvolání těchto událostí pro rozhraní API nízké úrovně, jako je například základní typy nebo kolekce.</span><span class="sxs-lookup"><span data-stu-id="05dad-150">However, it is unlikely to be worth the overhead to raise such events for low-level APIs such as base types or collections.</span></span> <span data-ttu-id="05dad-151">Například <xref:System.Collections.Generic.List%601> by vyvolat tyto události při přidání nové položky do seznamu a `Count` změny vlastností.</span><span class="sxs-lookup"><span data-stu-id="05dad-151">For example, <xref:System.Collections.Generic.List%601> would not raise such events when a new item is added to the list and the `Count` property changes.</span></span>  
  
 <span data-ttu-id="05dad-152">**✓ CONSIDER** vyvolání události oznámení při změně hodnoty vlastnosti prostřednictvím externí vynutí změn.</span><span class="sxs-lookup"><span data-stu-id="05dad-152">**✓ CONSIDER** raising change notification events when the value of a property changes via external forces.</span></span>  
  
 <span data-ttu-id="05dad-153">Pokud se změní hodnota vlastnosti prostřednictvím některé externí platnost (způsobem jinak než pomocí volání metody na objektu), vyvolat události znamenat pro vývojáře, že hodnota se mění a došlo ke změně.</span><span class="sxs-lookup"><span data-stu-id="05dad-153">If a property value changes via some external force (in a way other than by calling methods on the object), raise events indicate to the developer that the value is changing and has changed.</span></span> <span data-ttu-id="05dad-154">Dobrým příkladem je `Text` vlastnost ovládacího prvku textového pole.</span><span class="sxs-lookup"><span data-stu-id="05dad-154">A good example is the `Text` property of a text box control.</span></span> <span data-ttu-id="05dad-155">Když uživatel zadá text `TextBox`, automaticky změní hodnotu vlastnosti.</span><span class="sxs-lookup"><span data-stu-id="05dad-155">When the user types text in a `TextBox`, the property value automatically changes.</span></span>
