# <a name="base-classes-for-implementing-abstractions"></a><span data-ttu-id="3a837-102">Základní třídy pro implementaci abstrakcí</span><span class="sxs-lookup"><span data-stu-id="3a837-102">Base Classes for Implementing Abstractions</span></span>
<span data-ttu-id="3a837-103">Přesněji řečeno třída změní základní třídy, pokud jiné třídy je odvozen z něj.</span><span class="sxs-lookup"><span data-stu-id="3a837-103">Strictly speaking, a class becomes a base class when another class is derived from it.</span></span> <span data-ttu-id="3a837-104">Pro účely této části ale základní třídy je třída navržená především pro poskytování běžné abstrakce nebo pro jiné třídy opakovaně používat některé výchozí implementace ale dědičnosti.</span><span class="sxs-lookup"><span data-stu-id="3a837-104">For the purpose of this section, however, a base class is a class designed mainly to provide a common abstraction or for other classes to reuse some default implementation though inheritance.</span></span> <span data-ttu-id="3a837-105">Základní třídy se obvykle nacházejí uprostřed hierarchie dědičnosti mezi několika vlastní implementace v dolní části a abstrakce na nejnižší úrovni hierarchie.</span><span class="sxs-lookup"><span data-stu-id="3a837-105">Base classes usually sit in the middle of inheritance hierarchies, between an abstraction at the root of a hierarchy and several custom implementations at the bottom.</span></span>  
  
 <span data-ttu-id="3a837-106">Pro implementaci abstrakcí slouží jako pomocné rutiny implementace.</span><span class="sxs-lookup"><span data-stu-id="3a837-106">They serve as implementation helpers for implementing abstractions.</span></span> <span data-ttu-id="3a837-107">Například jeden z rozhraní Framework abstrakce pro seřazené kolekce položek je <xref:System.Collections.Generic.IList%601> rozhraní.</span><span class="sxs-lookup"><span data-stu-id="3a837-107">For example, one of the Framework’s abstractions for ordered collections of items is the <xref:System.Collections.Generic.IList%601> interface.</span></span> <span data-ttu-id="3a837-108">Implementace <xref:System.Collections.Generic.IList%601> není triviální, a proto rozhraní zajišťuje několik základních tříd, jako například <xref:System.Collections.ObjectModel.Collection%601> a <xref:System.Collections.ObjectModel.KeyedCollection%602>, který slouží jako pomocné rutiny pro implementaci vlastní kolekce.</span><span class="sxs-lookup"><span data-stu-id="3a837-108">Implementing <xref:System.Collections.Generic.IList%601> is not trivial, and therefore the Framework provides several base classes, such as <xref:System.Collections.ObjectModel.Collection%601> and <xref:System.Collections.ObjectModel.KeyedCollection%602>, which serve as helpers for implementing custom collections.</span></span>  
  
 <span data-ttu-id="3a837-109">Základní třídy obvykle nejsou vhodné, která bude sloužit jako abstrakce samy, protože mají tendenci obsahují příliš mnoho implementací.</span><span class="sxs-lookup"><span data-stu-id="3a837-109">Base classes are usually not suited to serve as abstractions by themselves, because they tend to contain too much implementation.</span></span> <span data-ttu-id="3a837-110">Například `Collection<T>` základní třída obsahuje velké množství implementace týkající se skutečnosti, že implementuje neobecné `IList` rozhraní (aby lépe integrovat neobecné kolekce) a na skutečnost, že je kolekce položek uložených v paměti v jednom z jeho polí.</span><span class="sxs-lookup"><span data-stu-id="3a837-110">For example, the `Collection<T>` base class contains lots of implementation related to the fact that it implements the nongeneric `IList` interface (to integrate better with nongeneric collections) and to the fact that it is a collection of items stored in memory in one of its fields.</span></span>  
  
 <span data-ttu-id="3a837-111">Jak bylo uvedeno výše základních tříd může poskytnout nedocenitelné nápovědy pro uživatele, kteří potřebují k implementaci abstrakcí, ale ve stejnou dobu, může představovat i nevýhodu významné.</span><span class="sxs-lookup"><span data-stu-id="3a837-111">As previously discussed, base classes can provide invaluable help for users who need to implement abstractions, but at the same time they can be a significant liability.</span></span> <span data-ttu-id="3a837-112">Tyto přidat styčné plochy a zvětšit hloubku hierarchie dědičnosti a proto koncepčně zkomplikovat rozhraní framework.</span><span class="sxs-lookup"><span data-stu-id="3a837-112">They add surface area and increase the depth of inheritance hierarchies and so conceptually complicate the framework.</span></span> <span data-ttu-id="3a837-113">Základní třídy by proto použít pouze v případě, že poskytují uživatelům rozhraní výrazně zlepšit situaci.</span><span class="sxs-lookup"><span data-stu-id="3a837-113">Therefore, base classes should be used only if they provide significant value to the users of the framework.</span></span> <span data-ttu-id="3a837-114">Mělo by se vyhnout Pokud, zadejte hodnotu pouze pro implementátory rozhraní, ve kterém případu delegování na interní implementaci namísto dědičnosti ze základní třídy silného považovat za.</span><span class="sxs-lookup"><span data-stu-id="3a837-114">They should be avoided if they provide value only to the implementers of the framework, in which case delegation to an internal implementation instead of inheritance from a base class should be strongly considered.</span></span>  
  
 <span data-ttu-id="3a837-115">**✓ CONSIDER** provedení základní třídy abstraktní i v případě, že neobsahují žádné abstraktní členy.</span><span class="sxs-lookup"><span data-stu-id="3a837-115">**✓ CONSIDER** making base classes abstract even if they don’t contain any abstract members.</span></span> <span data-ttu-id="3a837-116">To jasně komunikuje uživatelům určené výhradně, aby se dědila z třídy.</span><span class="sxs-lookup"><span data-stu-id="3a837-116">This clearly communicates to the users that the class is designed solely to be inherited from.</span></span>  
  
 <span data-ttu-id="3a837-117">**✓ CONSIDER** umístění základních tříd do samostatného oboru názvů z nejdůležitějších scénář typů.</span><span class="sxs-lookup"><span data-stu-id="3a837-117">**✓ CONSIDER** placing base classes in a separate namespace from the mainline scenario types.</span></span> <span data-ttu-id="3a837-118">Podle definice základních tříd jsou určené pro scénáře pokročilých rozšíření a proto nejsou zajímavé pro většinu uživatelů.</span><span class="sxs-lookup"><span data-stu-id="3a837-118">By definition, base classes are intended for advanced extensibility scenarios and therefore are not interesting to the majority of users.</span></span>  
  
 <span data-ttu-id="3a837-119">**X AVOID** pojmenování základní třídy s příponou "Základní", pokud třída je určena pro použití v veřejná rozhraní API.</span><span class="sxs-lookup"><span data-stu-id="3a837-119">**X AVOID** naming base classes with a "Base" suffix if the class is intended for use in public APIs.</span></span>
